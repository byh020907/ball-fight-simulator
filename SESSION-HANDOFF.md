## 최근 현재 계약 요약

- 합성은 같은 등급 장비 3개만 사용하고, 강화는 단계별 파편 200/300/500/800/1200을 사용합니다.
- 강화 기본 스탯은 단계마다 2배가 되며 등급별 상한은 common +1부터 legendary +5까지입니다. 특수 옵션은 배율 대상이 아닙니다.
- 방어는 `피해 × 100 / (100 + 방어)`의 점감형 비율 감소를 사용하고 최소 1 피해를 유지합니다. 방어 스탯은 포인트당 +1이며 플레이어 기본 방어력은 10/15/20 구간입니다.
- 장비 속도는 장비 적용 전 기본 속도의 약 2배로 수렴하며 능력 고유 속도 배율과 사냥터 탭 가속은 이후 단계에서 적용됩니다.
- 장비 탭은 강화·확장 비용 판단에 필요한 현재 파편과 실패 복구용 강화석 잔액을 함께 표시합니다.
- `문서 현행화` 요청은 현재 코드·테스트·최근 L1/L2 결정을 기준으로 처리합니다. 부분 수정 가능한 문서는 `docs/`에 남기고 전면 재작성 대상만 `.legacy/docs/`로 이동합니다.

- 장기 원정에서 단계별 파편 소진 속도와 강화석 획득 체감은 추가 플레이 데이터가 필요합니다.

## 활성 결정

## [L1] 2026-07-24 — 최근접 적 기준 압박·이탈 조작으로 자동 전투에 숙련 개입을 더한다
- 맥락: 현재 전투는 자동 진행 비중이 높아 사용자가 숙련도를 쌓을 공통 조작 지점이 부족하다. 정확한 위치 이동·조이스틱·드래그는 모바일에서 손과 UI가 전장을 가리고 지속 조작 피로를 만들며, 사냥터의 기존 캔버스 탭 가속은 진행 방향의 속도만 올려 보스 패턴에 의도적으로 접근·이탈하는 선택을 제공하지 못한다.
- 결정: 플레이어가 참가하는 전투에 `압박`과 `이탈` 두 기본 조작을 제공한다. 압박은 발동 순간 가장 가까운 적 방향, 이탈은 그 반대 방향으로 볼의 운동을 한 번만 개입하며 정확한 목적지 지정·지속 추적·자동 유도는 사용하지 않는다. 두 조작은 HP를 소모하지 않고 각각 독립된 쿨다운과 연속 사용 단계를 가진다. 각 조작의 연속 쿨다운은 0.30·0.45·0.65·0.90초이며, 해당 조작을 마지막으로 사용한 뒤 1.2초가 지나면 그 조작만 첫 단계로 초기화한다. 두 입력 사이에는 동시 오입력 방지용 0.12초 공통 잠금만 둔다. 토너먼트의 선택형 클릭 액션은 함께 유지하고, 사냥터의 기존 캔버스 탭 가속은 이 조작으로 대체한다.
- 영향: 공통 플레이어 전투 입력 상태와 최근접 적 방향 운동, 사냥터 탭 가속 제거, 전투 캔버스 가장자리의 모바일 조작 UI·독립 쿨다운 피드백, 토너먼트 클릭 액션 공존, 전투·사냥터 도움말과 규칙·설계 문서, 패치노트와 회귀·UI 계약 테스트.

## [L1] 2026-07-24 — 영구 장비를 롤식 공개 레시피 조합 구조로 개편한다
- 맥락: 같은 등급 장비 3개를 무작위 상위 장비로 바꾸는 현재 합성은 원하는 완성 장비를 목표로 파밍하고 빌드를 설계하는 재미가 약하다. 롤처럼 재료와 상위 결과를 한눈에 추적하는 영구 조합 구조가 필요하다.
- 결정: 이름별 고정 성능 장비와 데이터 기반 가변 레시피 그래프를 사용한다. 기초 장비는 지정 사냥터 드롭과 비싼 파편 상점으로 얻고 직접 장착할 수 있으며, 중간·완성 조합에는 추가 파편 비용이 든다. 장착은 공용 6칸, 보관은 아이템 종류별 최대 100개 중첩이 가능한 4×4 확장형 격자 인벤토리로 관리하고 드래그 앤 드롭 없이 클릭·자동 정렬을 사용한다. 새 등급은 `basic`·`intermediate`·`completed`이며 레시피는 최대 3단계·직접 재료 3종으로 제한한다. 아이콘은 이미지 에셋 없이 Canvas 2D 코드로 실루엣을 먼저 설계한 뒤 그 그림에 맞춰 명칭을 정한다. 장비 데이터는 캐릭터 표현 태그처럼 `iconTag`만 소유하고 공통 템플릿 레지스트리와 Canvas UI 렌더러가 해석한다.
- 영향: 장비 데이터·드롭·상점·조합·장착·인벤토리·UI·저장 스키마 전면 개편. 기존 강화와 완성 장비 이후 성장은 후속 설계로 분리한다. 다음 실행 단위는 캐릭터 표현 태그 구조를 참고한 Canvas 아이콘 렌더러와 영구 개발자 갤러리로 한정하며, 이를 실제 크기에서 검수한 뒤 명칭·수치·레시피와 기존 저장 데이터 이전 인터뷰를 재개한다.
