# Ball Fight 시각 방향

> **CSS 파일 위치**: 모든 스타일은 `src/styles.css`에 정의되어 있습니다.
> **클릭 액션 시스템 설계**: [`docs/click-actions.md`](click-actions.md)

## 기본 스타일

- 광택 표현보다 납작하고 읽기 쉬운 형태를 우선합니다.
- 그라데이션, 그림자, 블러, 글로우 효과는 기본 UI와 전투 오브젝트에서 사용하지 않습니다.
- 전투 공은 강한 단색과 어두운 외곽선으로 구분합니다.
- 캐릭터 정체성은 먼저 중앙 얼굴에서 읽히고, 그다음 능력 이펙트로 보강되어야 합니다.
- 얼굴은 공 반지름에 맞춰 스케일되는 벡터/라인 기반의 이모지풍 표현을 사용합니다.
- 얼굴 애니메이션은 눈 깜빡임, 흔들림, 맥박감, 입 움직임, 시선 이동처럼 작고 명확하게 유지합니다.
- 능력 이펙트는 과감해도 되지만, 경기장과 기본 UI는 단순하고 대비가 높아야 합니다.
- 파티클은 투명 페이드보다 선명한 단색과 높은 채도를 우선합니다.
- 기본 이동 잔상은 눈 피로를 줄이기 위해 사용하지 않고, 능력 발동이나 충돌 순간의 짧은 파티클로만 속도감을 표현합니다.
- 대상·접점이 핵심인 효과는 전투원 위 레이어에서 원인과 결과가 함께 읽혀야 합니다. PC와 모바일은 같은 월드 크기를 사용하되 최소 선 두께와 핵심 문구 크기만 화면 배율 기반 의미 토큰으로 보정합니다.

## 기본 캐릭터 설계

- 캐릭터 능력은 빗나감이 반복될 때 사용자 불쾌감이 누적되지 않도록 설계합니다.
- 스킬이 계속 빗나가는 캐릭터는 다음 시도에서 보정, 충전, 쿨다운 완화, 범위 조정, 속도 조정처럼 실패를 회수하는 메커니즘을 가져야 합니다.
- 보정은 자동 승리처럼 느껴지지 않게 제한적으로 적용하되, 사용자가 "계속 헛치는 캐릭터"라고 느끼지 않도록 충분히 체감되어야 합니다.
- 실패 보정 규칙은 캐릭터 정체성과 연결되어야 합니다. 예를 들어 Dash Ball은 쿨다운 스택과 방향 보정, Grenade Ball은 빗나갈수록 더 맞추기 쉬운 투척 타이밍처럼 각 캐릭터의 플레이 감각 안에서 해결합니다.

## 캐릭터 얼굴 메모

- Archer Ball: 집중한 눈, 짧고 자신감 있는 입.
- Orbit Ball: 크고 호기심 있는 눈, 궤도/어지러움을 암시하는 표정.
- Trickster Ball: 장난스러운 이중 미소와 속임수 느낌.
- Grenade Ball: 긴장된 화난 눈, 폭발 직전의 이를 악문 입.
- Dash Ball: 방향을 잡고 돌진하는 집중한 눈, 적중 보상형 대시 느낌.
- Rage Ball: 두꺼운 눈썹, 충전된 포효 표정.
- Spin Ball: 소용돌이 눈과 단단히 다문 입, 자이로가 점점 빨라지는 긴장감.
- Eater Ball: 먹기 모드에서는 Pac-Man처럼 방향성이 있는 입, 그 외에는 배고픈 눈.

## 최근 표현 규칙

- Dash 레이저는 조준 중 가는 점선, 발사 중 굵은 적색 선으로 나눈다. 벽 반사점에는 흰 중심과 주황 전기 꼬리를 두고, Lv.9 레이저의 첫 실제 피해 순간부터 대상에 1초 동안 공통 점화 외곽선과 불꽃 오버레이를 유지한다.
- Grenade 점착은 회전하는 대상의 로컬 접점을 따라가며 빠른 적색 경고등으로 일반 수류탄과 구분한다. 화상 재폭발은 수축하는 흰 중심과 주황 충격 고리·파티클을 함께 사용해 최초 폭발과 읽히는 순서를 분리한다.
- Gunner 재사격은 청록 직선과 첫 반사 재조준 꺾임으로 일반 탄과 구분한다. 포탑은 고정 배치를 기본으로 사용하며, 설치 상승·HP 막대·8초 수명 고리·파괴 파편을 전경에서 표시한다. 이동형은 같은 전투 계약의 비교 모드로만 남긴다.
- Phantom 기본 Shadow Strike는 총공격력 ×1.50 성장 계수를 사용하되 그림자 단계·메아리·종결 표현은 유지한다. 메아리와 종결에 기본 추가 피해 연출을 중복하지 않는다.
- Hero 성장 스택은 5분할 금색 링, 성장 코어는 금색 육각 외곽선·속성색 중심·소유자 연결선, 저장 조각은 공전 육각형으로 구분한다. 공명은 0.1초 간격 곡선 궤적과 별 꼭짓점을 순서대로 그리고, 정확히 5조각일 때만 금빛 완성 별과 Heroic Burst 고리를 보여 준다.
- Orbit Ball은 주변 공이 피격되면 사라지고, 빠르게 회전한 뒤 본체에서 공이 하나씩 궤도로 재충전되는 흐름을 보여줍니다. Lv.3 협공은 첫 적중점의 고정 동기화 섬광과 각 탄의 실제 방향 보간을 따르는 곡선 잔상으로 읽히며, 이후 움직인 적에게 다시 조준하는 빔은 그리지 않습니다.
- Orbit Ball의 기본 궤도선은 배경과 본체에서 분리되도록 진한 파랑 단색을 사용하고, 위성은 노랑과 검은 외곽선으로 표시합니다. Lv.6 폭발은 실제 적중점에서만 70px 고리로 표시하고, Lv.9 본체 캐치는 원래 슬롯의 위성이 복원되는 것으로 피드백합니다.
- Trickster Lv.3 적대 씨앗 접촉은 대상 둘레의 덩굴 조임과 틱 맥동으로 표현합니다. Lv.6 표식은 대상 위 씨앗 문양, 폭발은 접점 파편 섬광으로 분리하고, Lv.9 후속 씨앗은 충돌 유예 동안 반투명 싹이었다가 활성 시 가시가 돋습니다.
- Eater Ball이 뱉은 공은 초록 원 이펙트 대신 표정이 빙글빙글 회전하는 상태로 표현합니다.
- Spin Ball은 충전량에 맞춰 둘레의 주황 12분할 링을 채웁니다. 만충 적중 뒤 실제 접점 절삭호·틱 불꽃/파편·한 개의 누적 숫자 ×1~×12·마지막 섬광으로 0.60초 표면 절단을 읽게 합니다. Lv.9 유체장은 고정 링이나 빔 대신 각 대상의 실제 접선 3:안쪽 1 유동 벡터를 따르는 전역 파티클로 표현합니다.
- Bat Ball Lv.3은 실제 각속도만 강화하며 별도 회전 오라를 만들지 않습니다. Lv.6 `HOME RUN ×n.nn`은 첫 정적 충돌점에 한 번, Lv.9 `RESET!`과 흰 방망이 섬광은 Bat Ball 본체에 표시해 서로 다른 사건으로 읽히게 합니다.
- Archer 투사체는 벽에 튕긴 뒤 탄환 방향도 실제 속도 벡터에 맞춰 보여야 합니다.
- 크리티컬 히트 발동 시 금색 "CRIT! N" 플로팅 텍스트(`CriticalNumber`)로 강한 한 방을 읽게 합니다.
- Eater Lv.9 벽 파열은 반원 섬광 이펙트로 충돌 지점에서 퍼지며, 대상과 주변의 피해 차이를 시각적으로 구분합니다.
- Rage 폭발은 같은 중심의 이중 불꽃 고리, 여진은 짧은 간격으로 이어지는 3파 고리로 충전 단계별 위력을 구분합니다. 점화 대상은 지속시간 동안 불꽃 외곽선으로 식별합니다.
- Eater 소화는 삼킨 대상의 축소된 대리 형상과 입 닫힘, 뱉기는 대상 궤적과 본체 반동, 벽 파열은 충돌점의 150px 반원과 잔해로 원인과 결과를 구분합니다.
- Archer Lv.3 이상 예측점은 조준 중 실제 갱신 위치를 따라가는 전경 십자 표식으로 표시하며, 발사 뒤에는 남기지 않습니다.

## 모바일 UI 규칙

- 모바일에서는 오른쪽 패널과 경기장을 가로로 압축하지 않고 세로로 쌓습니다.
- 공간 확보 우선순위는 전투 화면, 상태 UI, 토너먼트, 스탯 순서입니다.
- 초기 화면에서는 랜덤으로 선택된 유저 공을 경기장 중앙에 보여 사용자가 자신의 캐릭터를 바로 알 수 있게 하고, 표정 애니메이션이 계속 살아 있게 합니다.
- 스탯 배분 패널은 전투 시작 전 팝업 형태로 보여주고, 모바일에서는 구석이 아닌 하단 중앙에 배치해 선택 공 프리뷰를 가리지 않게 합니다.
- 시작 버튼은 스탯 배분 팝업 하단 액션 영역에 두어 배분 완료 후 바로 누를 수 있게 합니다.
- 스탯 배분이 완료되면 작은 세로 화면에서도 시작 버튼이 보이도록 스탯 패널을 하단 액션 영역으로 자동 스크롤합니다.
- 토너먼트표는 준비 단계에서는 숨기고, 스탯 배분 후 토너먼트가 시작되면 표시합니다.
- 세로가 더 긴 모바일 화면에서는 전체 프레임을 세로 flex로 구성하고, 경기장을 상단에서 가로폭 100% 기준으로 가능한 크게 표시합니다.
- 모바일 하단에는 스탯/상태/대진표 영역을 전투장 바로 아래에 붙여 배치하고, 부족한 내용은 각 패널 내부 스크롤로 처리합니다.
- 모바일 하단 패널 높이는 고정 픽셀 계산보다 flex의 남은 공간 분배를 우선합니다.
- 모바일 폰트는 뷰포트에 직접 크게 반응시키기보다 패널 기준 크기와 `em` 비율로 조절해 과도하게 커지지 않게 합니다.
- 전투 상단 타이틀 바는 화면 높이를 아끼기 위해 표시하지 않고, 대진은 시작 연출로 보여줍니다.
- 오버타임은 기존 중앙 연출 위치를 유지하되, 전투 화면을 가리지 않는 투명 배경과 반투명 글자로 한 번 보여줍니다.
- 100포인트 배분은 `+10`, `+1`, `-1`, `-10` 버튼을 함께 제공해 터치 횟수를 줄입니다.
- 터치 버튼은 최소 36px 높이를 확보하고, 좁은 화면에서는 설명 문구보다 조작값과 현재 배분을 우선합니다.
- 캐릭터 카드에는 원본 `hp`, `damage`, `speed` 수치를 직접 노출하지 않고 배분 퍼센트만 표시합니다.
- 모바일 화면은 페이지 전체 스크롤을 만들지 않고, 필요한 경우 스탯/브래킷 패널 안에서만 스크롤합니다.
- 모바일 경기장은 전투 카드가 잘리지 않도록 화면 높이에 맞춰 축소하되 정사각형 비율을 유지합니다.
- PC와 모바일 모두 뷰포트 위아래에 안전 여백을 남겨 UI 테두리와 전투 카드가 화면 끝에 붙거나 잘려 보이지 않게 합니다.
- 스탯 패널의 도움말, 컬렉션 등 보조 버튼은 첫 번째 줄의 전용 액션 영역에 모으고 오른쪽 정렬합니다. 이후 추가되는 보조 버튼도 같은 영역에 배치해 한 줄로 누적합니다.

## 토스트 알림

- 화면에는 현재 알림과 퇴장 중인 이전 알림만 최대 두 장까지 겹쳐 표시합니다. 새 알림은 전면으로 진입해 이전 알림을 덮습니다.
- 연속 호출 시 현재 알림은 최소 650ms를 표시한 뒤 다음 알림으로 전환합니다. 기본 표시 시간은 3.5초입니다.
- 같은 문구를 연속 호출하면 새 항목을 쌓지 않고 현재 또는 마지막 대기 항목의 횟수를 `×N`으로 합칩니다.
- 토스트는 게임 조작을 막지 않으며, 짧은 위쪽 진입·아래쪽 퇴장 전환을 사용합니다. 작은 화면에서는 문구를 줄바꿈해 화면 밖으로 넘치지 않게 합니다.

## 컬렉션 허브

- 도감, 캐릭터 연계, 업적은 하나의 중앙 팝업에서 탭으로 전환합니다.
- 팝업 헤더, 탭 바, 탭별 검색·필터·정렬 도구는 고정하고 카드 또는 목록 영역만 세로 스크롤합니다.
- 도감에서 캐릭터를 선택하면 상세 정보는 목록 스크롤 밖의 하단 고정 영역에 표시합니다.
- 모바일에서는 페이지 전체 스크롤을 만들지 않고 `100dvh` 안에서 팝업 내부만 스크롤합니다.
- 도감과 연계는 카드 grid, 업적은 진행도를 읽기 쉬운 세로 목록을 사용합니다.
- 미기록 캐릭터도 숨기지 않고 채도와 `미기록` 문구로 상태를 구분합니다.
- 연계 탭에는 장착, 해제, 슬롯 UI를 만들지 않습니다.
- 장비 시스템이 추가되면 같은 허브의 별도 `장비` 탭을 사용하되 연계 탭과 책임을 섞지 않습니다.
- 구현 상세는 [`collection-hub-ui.md`](collection-hub-ui.md)를 따릅니다.

## 앱 라이프사이클

`BattleApp`은 화면 전환을 토너먼트나 사냥 run의 내부 상태에서 추론하지 않는다. `AppLifecycle`이 아래 상태를 단일 소스로 관리한다.

| 상태                  | 설정 UI        | 의미                                                                                        |
| --------------------- | -------------- | ------------------------------------------------------------------------------------------- |
| `setup`               | 표시·조작 가능 | 모드 선택과 스탯 배분, 새 게임 시작 가능                                                    |
| `playing`             | 숨김·잠금      | 토너먼트 또는 사냥터 진행 중                                                                |
| `result_confirmation` | 숨김·잠금      | 우승·사냥터 결과·토너먼트 탈락 결과가 표시됐으며 확인 버튼으로만 `setup`으로 돌아갈 수 있음 |

라이프사이클 전이마다 세션 리비전이 증가한다. `BattleApp.startMatch()`의 비동기 단계는 리비전이 바뀌면 즉시 중단해, 결과 확인이나 초기화 뒤에 지연된 매치가 다시 화면을 덮지 못하게 한다.

`playing` 상태에서는 모바일 관전 중 자동 절전으로 화면이 어두워지지 않도록 Screen Wake Lock을 요청한다. 결과 확인 또는 `setup`으로 전환하면 잠금을 해제하며, 시스템이 잠금을 해제한 뒤 문서가 다시 보이면 아직 전투 중일 때만 다시 요청한다. 지원하지 않거나 잠금 요청을 거절한 브라우저도 전투를 정상 진행한다.

## 결과 확인 초기화

- `result_confirmation`에서 `setup`으로 돌아갈 때 `BattleApp.resetGameplayUiState()`가 전투 전용 UI 컴포넌트의 `reset()`을 호출합니다.
- 초기화 대상은 결과 오버레이와 보상, 전투 로그, 파이터 스트립, 대진표, 모드·시작 조작부, 토스트 및 플레이어 패널입니다.
- 초기화 뒤 `refreshPlayerSetup()`이 저장된 프로필, 장비, 경험치 정보를 다시 표시합니다. 영속 데이터는 비우지 않고 전투 중 일시 UI 상태만 제거합니다.
- 결과 오버레이는 재사용 가능한 `resultSequence` 단계 목록으로 구성합니다. 각 단계는 `label`·`text`·선택 보상 뷰를 가지며, 토너먼트·사냥터 등 결과를 내는 모든 모드가 같은 표시 경로를 사용합니다.
- 사냥터는 XP를 전장 오브 회수 시 즉시 반영하되, 패배·스테이지 클리어 결과에서는 해당 전투의 누적 XP 진행을 첫 `resultSequence` 단계로 다시 확인합니다. 일반 층 승리는 다음 선택을 막지 않도록 사냥터 오버레이 안에서 두 장의 전투 결과 카드로 나눕니다. 첫 카드는 XP 진행만, 오른쪽 `결과 1/2 》` 탭 뒤의 둘째 카드는 전투 완료·전리품·다음 층 선택만 표시합니다.
- 모바일 결과는 한 카드에 길게 쌓아 스크롤하지 않습니다. 카드는 화면의 수직·수평 정중앙에 고정하고, 카드 오른쪽 경계에 작게 겹친 세로 탭이 현재 단계와 `》`를 보여 줍니다. 마지막 요약 단계에서는 같은 탭이 `확인`으로 바뀌며, 탭은 카드 폭을 차지하지 않아 카드 중심을 움직이지 않습니다.
