# Ball Fight 시각 방향

## 기본 스타일

- 광택 표현보다 납작하고 읽기 쉬운 형태를 우선합니다.
- 그라데이션, 그림자, 블러, 글로우 효과는 기본 UI와 전투 오브젝트에서 사용하지 않습니다.
- 전투 공은 강한 단색과 어두운 외곽선으로 구분합니다.
- 캐릭터 정체성은 먼저 중앙 얼굴에서 읽히고, 그다음 능력 이펙트로 보강되어야 합니다.
- 얼굴은 공 반지름에 맞춰 스케일되는 벡터/라인 기반의 이모지풍 표현을 사용합니다.
- 얼굴 애니메이션은 눈 깜빡임, 흔들림, 맥박감, 입 움직임, 시선 이동처럼 작고 명확하게 유지합니다.
- 능력 이펙트는 과감해도 되지만, 경기장과 기본 UI는 단순하고 대비가 높아야 합니다.
- 파티클은 투명 페이드보다 선명한 단색과 높은 채도를 우선합니다.
- 기본 이동 잔상은 눈 피로를 줄이기 위해 사용하지 않고, 능력 발동이나 충돌 순간의 짧은 파티클로만 속도감을 표현합니다.

## 캐릭터 얼굴 메모

- Archer Ball: 집중한 눈, 짧고 자신감 있는 입.
- Orbit Ball: 크고 호기심 있는 눈, 궤도/어지러움을 암시하는 표정.
- Trickster Ball: 장난스러운 이중 미소와 속임수 느낌.
- Grenade Ball: 긴장된 화난 눈, 폭발 직전의 이를 악문 입.
- Frost Blade: 차분하고 날카로운 눈, 얼음 베기 느낌의 입.
- Rage Ball: 두꺼운 눈썹, 충전된 포효 표정.
- Eater Ball: 먹기 모드에서는 Pac-Man처럼 방향성이 있는 입, 그 외에는 배고픈 눈.

## 최근 표현 규칙

- Orbit Ball은 주변 공이 피격되면 사라지고, 빠르게 회전한 뒤 본체에서 공이 하나씩 궤도로 재충전되는 흐름을 보여줍니다.
- Orbit Ball의 기본 궤도선은 배경과 본체에서 분리되도록 진한 파랑 단색을 사용하고, 위성은 노랑과 검은 외곽선으로 표시합니다.
- Eater Ball이 뱉은 공은 초록 원 이펙트 대신 표정이 빙글빙글 회전하는 상태로 표현합니다.
- Archer 투사체는 벽에 튕긴 뒤 탄환 방향도 실제 속도 벡터에 맞춰 보여야 합니다.

## 모바일 UI 규칙

- 모바일에서는 오른쪽 패널과 경기장을 가로로 압축하지 않고 세로로 쌓습니다.
- 공간 확보 우선순위는 전투 화면, 상태 UI, 토너먼트, 스탯 순서입니다.
- 초기 화면에서는 랜덤으로 선택된 유저 공을 경기장 중앙에 보여 사용자가 자신의 캐릭터를 바로 알 수 있게 하고, 표정 애니메이션이 계속 살아 있게 합니다.
- 스탯 배분 패널은 전투 시작 전 팝업 형태로 보여주고, 모바일에서는 구석이 아닌 하단 중앙에 배치해 선택 공 프리뷰를 가리지 않게 합니다.
- 시작 버튼은 스탯 배분 팝업 하단 액션 영역에 두어 배분 완료 후 바로 누를 수 있게 합니다.
- 스탯 배분이 완료되면 작은 세로 화면에서도 시작 버튼이 보이도록 스탯 패널을 하단 액션 영역으로 자동 스크롤합니다.
- 토너먼트표는 준비 단계에서는 숨기고, 스탯 배분 후 토너먼트가 시작되면 표시합니다.
- 세로가 더 긴 모바일 화면에서는 전체 프레임을 세로 flex로 구성하고, 경기장을 상단에서 가로폭 100% 기준으로 가능한 크게 표시합니다.
- 모바일 하단에는 스탯/상태/대진표 영역을 전투장 바로 아래에 붙여 배치하고, 부족한 내용은 각 패널 내부 스크롤로 처리합니다.
- 모바일 하단 패널 높이는 고정 픽셀 계산보다 flex의 남은 공간 분배를 우선합니다.
- 모바일 폰트는 뷰포트에 직접 크게 반응시키기보다 패널 기준 크기와 `em` 비율로 조절해 과도하게 커지지 않게 합니다.
- 전투 상단 타이틀 바는 화면 높이를 아끼기 위해 표시하지 않고, 대진은 시작 연출로 보여줍니다.
- 오버타임은 기존 중앙 연출 위치를 유지하되, 전투 화면을 가리지 않는 투명 배경과 반투명 글자로 한 번 보여줍니다.
- 100포인트 배분은 `+10`, `+1`, `-1`, `-10` 버튼을 함께 제공해 터치 횟수를 줄입니다.
- 터치 버튼은 최소 36px 높이를 확보하고, 좁은 화면에서는 설명 문구보다 조작값과 현재 배분을 우선합니다.
- 캐릭터 카드에는 원본 `hp`, `damage`, `speed` 수치를 직접 노출하지 않고 배분 퍼센트만 표시합니다.
- 모바일 화면은 페이지 전체 스크롤을 만들지 않고, 필요한 경우 스탯/브래킷 패널 안에서만 스크롤합니다.
- 모바일 경기장은 전투 카드가 잘리지 않도록 화면 높이에 맞춰 축소하되 정사각형 비율을 유지합니다.
- PC와 모바일 모두 뷰포트 위아래에 안전 여백을 남겨 UI 테두리와 전투 카드가 화면 끝에 붙거나 잘려 보이지 않게 합니다.
