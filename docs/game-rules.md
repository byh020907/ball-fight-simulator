# 게임 규칙

> **클릭 액션 시스템**: [`docs/click-actions.md`](click-actions.md) (v0.9.0, 실패 피드백 추가)

이 문서는 Ball Fight Simulator의 게임성 규칙을 기록하는 기준 문서입니다. 전투 규칙, 스탯, 토너먼트 진행 방식이 바뀌면 구현과 함께 이 문서를 업데이트합니다.

## 현재 게임 흐름

1. 페이지가 열리면 현재 해금된 로스터 중 하나가 무작위로 사용자를 대표하는 캐릭터로 지정됩니다.
2. 사용자는 토너먼트 시작 전에 내 캐릭터의 스탯 포인트를 배분합니다.
3. 사용자가 모든 포인트를 배분하면 토너먼트를 시작할 수 있습니다.
4. 토너먼트 시작 시 나머지 캐릭터들도 같은 총량의 포인트를 무작위로 배분받습니다.
5. 첫 매치에서 클릭 액션 카드를 고르면, 현재 `첫 도전` 또는 숙련도 단계와 AI 시작 레벨을 잠깐 보여 준 뒤 VS 대진으로 넘어갑니다.
6. 대진표가 확정되고, 플레이어가 남아 있는 동안 전투는 자동으로 진행됩니다.
7. 사용자의 캐릭터가 우승하면 모바일 결과 순서에서 경험치 → 숙련도 승급 결과(보상 이름·실제 수치·다른 볼 적용 여부·다음 AI 시작 레벨) → 기존 축하 메시지를 차례로 표시합니다. 화면 중앙 카드의 오른쪽 경계에 겹친 작은 세로 탭의 `》`로 중간 단계를 넘기며, 마지막 축하 단계에서는 같은 탭의 `확인`을 눌러 초기 화면으로 돌아갑니다.
8. 사용자의 캐릭터가 중간에 탈락하면 경험치와 아쉬운 메시지·최종 등수를 같은 결과 순서로 표시한 뒤, 화면 중앙 카드의 마지막 탭 `확인`으로 초기 화면으로 돌아갑니다. 탈락 뒤 남은 AI 대진은 자동 진행하지 않습니다.

## 전투 팀 규칙

전투 참가자는 `teamId`를 가집니다. 명시적 팀이 없는 기존 토너먼트 참가자는 각자 다른 기본 팀을 받아 개인전처럼 동작합니다.

- 서로 다른 `teamId`끼리는 적입니다.
- 같은 `teamId`끼리는 아군입니다.
- 아군끼리는 겹쳤을 때 밀어내기와 충돌 물리는 유지되지만 충돌 피해, 투사체 피해, Dash 충돌 피해 같은 적대 효과를 주지 않습니다.
- 승패는 생존한 적대 팀 수로 판단합니다. 생존 fighter가 여러 명이어도 모두 같은 팀이면 그 팀의 승리로 처리합니다.
- 단일 대상 능력은 `simulation.getOpponent()`를 통해 가장 가까운 적을 고릅니다. 다수 대상 능력은 `simulation.getEnemiesOf()`를 사용해 아군을 제외해야 합니다.

이 규칙은 사냥터, 웨이브, 1대n, n대n 전투를 위한 기반입니다. 자세한 사냥터 설계는 [`docs/hunting-grounds-system.md`](hunting-grounds-system.md)를 참고합니다.

### 사냥터 시작 체크포인트

일반 사냥터 원정은 모든 해금 지역에서 기본적으로 **1층 첫 조우**부터 시작합니다. 지역을 고른 뒤 `1·20·40·60·80` 중 실제 첫 조우 층을 명시적으로 선택하며, 20층 이상은 해당 지역의 최근 도달 기록이 그 숫자 이상일 때만 선택할 수 있습니다. 이 기록은 캐릭터별이 아니라 플레이어 프로필 전체가 지역별로 공유하므로, 한 캐릭터로 연 체크포인트를 다른 우승 캐릭터도 사용할 수 있습니다. 예를 들어 그 지역에서 47층에 도달했다면 `1·20·40`만 열립니다. 귀환·패배·스테이지 클리어는 지역별 최근 도달 기록을 계속 갱신하지만, 자동 재개 위치를 정하지 않으며 업적용 최고 기록 `deepestFloor`도 별도의 전역 최대값으로 유지됩니다. 개발자 디버그 원정은 기존처럼 요청한 조우층을 그대로 사용합니다.

### 사냥터 적 전력

일반 적 전력은 1층 `1.0`에서 100층 `1.7`까지 완만하게 증가합니다. 정예 몹 습격의 구성원도 같은 현재 층 `normal` 전력을 사용하고, 챔피언만 역할 보정을 더해 합산 배율 `2.0` 상한을 적용합니다. 이 배율은 HP·공격력·방어력에만 적용하고 결과는 0.5 단위로 맞춥니다. 속도와 스킬은 층 전력으로 바뀌지 않으며, 몬스터 중간 보스의 HP 2배·공격/방어 1.5배 고유 보정은 별도로 유지됩니다.

일반 몬스터는 1층 추적·돌진, 10층 사수, 20층 방벽·힐러, 30층 전격·충격파, 40층 사슬·흡수·파편, 50층 부메랑·분열·도약·레이저 순으로 실제 해금됩니다. 해금 전에는 일반 조우와 중간 보스 후보에 포함되지 않습니다. 희귀도는 장비와 같은 `common`·`uncommon`·`rare`·`epic`을 사용하며 일반 몬스터에는 `legendary`가 없습니다.

정예 몹 습격은 10층부터 수동 7종 역할 조합 중 현재 층에 열린 조합을 같은 확률로 선택합니다. 힐러와 원거리 몬스터는 후열, 나머지는 전열에서 시작하고 정예 전용 군집 위치 보정으로 후열을 보호합니다. 개별 몬스터 행동은 유지하며, 보상은 XP 배분 1.2배·물리 XP 오브 +1·승리 XP +6만 추가합니다.

### 사냥터 XP 오브

사냥터의 처치 XP는 적이 떨어뜨린 여러 개의 둥근 황록색 오브를 실제로 주워 얻습니다. 등급·정예·보스일수록 오브 수와 XP 비중이 커지고, 한 알의 XP가 클수록 오브도 커져 가치 차이를 전장에서 읽을 수 있습니다. 전투마다 한 번 확정하는 기본 XP 보정은 -15~+15XP를 5XP 단위로 굴립니다. 따라서 최대 기여도 일반 승리는 45~75XP이며 평균은 기존 60XP로 유지되고, 상위 등급의 추가 프리미엄은 이 범위 뒤에 더해집니다. 승리하면 남은 오브를 전장 전체 자석으로 회수하고, 패배하면 이미 회수한 XP만 남습니다. 일반 승리 선택과 패배·스테이지 클리어 결과 모두 공통 XP 보상 패널의 진행 바로 이번 전투의 성장 결과를 보여 줍니다.

## 전투 능력 구성

각 fighter는 `AbilitySet` 하나를 소유합니다. `AbilitySet`에는 대표 행동을 담당하는 주 능력 1개와, 전투 연출·추가 효과를 맡을 수 있는 순서 보장 서브 능력 0개 이상이 들어갑니다.

- 능력의 update, 충돌 콜백, 처치/부활 콜백, 전투 오버레이 렌더링, UI 상태는 주 능력부터 서브 능력 등록 순서로 처리합니다.
- 이동 속도·충돌 피해·방어·충돌 충격·반지름·표정과 같은 fighter 물리/정체성 값은 주 능력만 결정합니다. 서브 능력은 현재 물리 보정에 참여하지 않습니다.
- 능력마다 고유 `abilityId`, role(`primary`/`sub`), 강화 tier, instance key, 표시 이름을 가집니다. 강화 보상은 소유 fighter의 대표 행동이 아니라 해당 능력 인스턴스의 ID와 tier를 기준으로 조회합니다.
- 투사체와 특수 효과는 생성한 능력 인스턴스를 출처로 보존합니다. 따라서 보조 Gunner의 귀환 탄환 쿨다운 감소와 보조 Hero Orb의 자석·수집 효과도 각각의 능력에만 적용됩니다.
- 전투 카드에는 능력 상태 목록을 순서대로 표시합니다. 각 행은 능력 이름·주/서브 구분·상태 문구·진행률·독립 쿨다운을 가지며, Rage·Spin처럼 충전 기반인 능력의 진행률도 그대로 표시합니다.

기본 roster는 주 능력 하나로 시작한다. Lv.10 환생으로 장착한 액션 카드는 이 `AbilitySet`에만 추가되며, 주 능력의 물리 정체성이나 XP 강화 tier를 바꾸지 않는다.

### 환생 보상

새 Lv.10 도달은 전투 결과를 `확인`해 준비 화면으로 복귀한 뒤 한 번만 알리고, 안내의 주 동작은 해당 캐릭터의 도감 상세 `환생` 탭을 연다. 이 탭에서 후보 3장을 만들거나 저장된 후보를 인라인으로 확인해 하나를 고른다. 후보는 선택 전에 저장되며 탭을 닫거나 새로고침해도 다시 열면 같은 세 장을 본다. 실제 선택에서만 자신의 XP와 토너먼트 도전 단계를 Lv.1 시작 상태로 초기화하고 환생 횟수를 1 올린다. 숙련도는 유지한다. 후보는 영구 기초 수치 보상, 다른 볼의 액션 카드, 패시브 카드가 같은 풀에서 비복원 추첨된다.

- 영구 기초 수치 보상은 HP·공격·속도·방어력 중 서로 다른 두 항목을 무작위로 즉시 누적한다. 수치는 희귀 장비의 스탯 범위와 항목별 가치 비율을 그대로 사용하며, 후보를 만들 때 정해진 두 수치가 저장되어 다시 열어도 바뀌지 않는다. 이 수치는 장착·rank 대상이 아니며, 도감 환생 탭의 `누적 기초 수치`에 합계로 표시한다. 쿨타임·반경·질량·충돌 판정은 영구 기초 수치가 아니다.
- 액션·패시브 카드는 캐릭터별로 수집하고 중복 획득은 rank만 올린다. 최대 rank는 10, 동시에 장착할 수 있는 카드는 3장이다.
- 액션 카드는 자기 자신의 대표 Ability를 제외한 기존 Ability를 `sub` 인스턴스로 재사용한다. 서브 인스턴스는 독립 쿨다운·update·draw·fighter card 상태를 가지며 XP tier는 0으로 시작한다.
- 초기 패시브 `냉각 회로`는 장착한 모든 능력의 쿨타임을 rank 1에서 30% 줄이고, rank마다 2%p씩 늘려 rank 10에서 최대 48%까지 줄인다.
- 누적 환생 횟수는 정지·전투 모두에서 오라, 외곽선, 사각 셀 불꽃으로 보인다. 불꽃은 곡선 또는 닫힌 실루엣이 아니라 회전하는 사각 셀의 밀집된 흐름으로만 표현한다. 셀은 공 외곽 전체의 표면 앵커에서 시작해 수명 초반에는 캐릭터를 감싸고, 새 셀은 각 앵커의 외곽 법선 방향으로 먼저 발사된다. 매 프레임 작용하며 수명에 따라 강해지는 위쪽 부력과 항력이 그 초기 궤적을 점차 위로 굽히며, 앵커 결합이 풀린 뒤에는 이동 방향과 무관하게 위로 타오른다. 공이 이동하면 결합이 느슨해진 셀이 이전 위치에 남아 자연스러운 후류가 생긴다. 셀은 밝은 중심에서 주황 몸통, 적색 끝으로 바뀌고 회전·크기·투명도가 수명과 흔들림에 따라 변한다. 셀 수는 환생 1회의 56개에서 10환생의 92개까지 정수 반올림 선형 보간하며, 크기와 흔들림도 같은 1~10 구간을 선형으로 강화한다. `source-over` 알파 합성을 사용해 과포화된 가산 발광을 피하며, 이 비주얼은 물리 수치나 저장 데이터에 영향을 주지 않는다.

## 스탯 배분 규칙

현재 배분 포인트는 총 100점입니다. 각 스탯은 캐릭터의 종족값을 기준으로 개선됩니다.

| 스탯     | 코드 값          | 1포인트 효과                     | 설명                                                   |
| -------- | ---------------- | -------------------------------- | ------------------------------------------------------ |
| 체력     | `hp`             | 종족값 x `1 + 포인트/100`        | 오래 버티는 능력입니다.                                |
| 공격     | `damage`         | 종족값 x `1 + 포인트/100`        | 충돌 최대 피해량이 증가합니다.                         |
| 속도     | `speed`          | 종족값 x `1 + 포인트/100`        | 기본 이동 속도를 올립니다.                             |
| 쿨타임   | `skill`          | 스킬 쿨타임 × `100/(100+포인트)` | 스킬 쿨타임이 체감 감소합니다. (100포인트 시 50% 단축) |
| 방어력   | `defense`        | 종족값 x `1 + 포인트/100`        | 받는 피해에서 방어력만큼 차감됩니다.                   |
| 크리티컬 | `criticalChance` | 기본 5% + 포인트당 +1%p          | 발동 시 원본 피해 ×2.                                  |

스탯 하나에 최대 50점까지 투자할 수 있습니다.

모든 플레이어 캐릭터의 종족값 속도는 로스터의 기존 상대 차이를 유지한 공통 `1.5배` 기준이다. 이 기준 뒤에 스탯 배분·레벨·장비·숙련도·환생·능력 modifier가 기존 순서대로 적용된다.

게임 UI에서는 캐릭터의 원본 값을 직접 보여주지 않고, `체력 +30% · 공격 +40% · 속도 +30% · 쿨타임 +30%`처럼 배분 결과만 표시합니다.

### 스탯 배분 계산 방식

`applyStatAllocation()`에서 모든 스탯은 **pre-compute**됩니다 (전투 중 재계산 없음).

```
hp      = baseHP      × (1 + pts/100)
damage  = baseDamage  × (1 + pts/100)
speed   = baseSpeed   × (1 + pts/100)
defense = baseDefense × (1 + pts/100)
skill   = pts         (별도 저장, Ability.cooldown에서 100/(100+pts)로 계산)
```

`skill`만 포인트를 그대로 저장하고, `Ability` 베이스 클래스의 `cooldown` getter에서 `100/(100+스탯)` 공식으로 동적 계산합니다.

### 충돌 피해 계산 (3단계)

```
1단계 — calculateCollisionDamage()
  ㄴ round(baseDamage × efficiency × getDamageMultiplier())
     - efficiency = speedEff × dirEff × glancingPenalty (모두 상한 없음)
     - speedEff = 현재 속도 / baseSpeed (스탯 보정 완료된 기준, 1=기본)
     - getDamageMultiplier(): 오버타임 전용, 평소 1

2단계 — aModifiers.damage (곱)
  ㄴ Ability.getStatModifiers().damage (능력별 피해 보정)

3단계 — takeDamage() 방어력 차감
  ㄴ actual = max(1, round(amount - totalDefense))
     - totalDefense = round(baseDefense × abilityDefMult)
     - abilityDefMult: Ability.getStatModifiers().defense (기본 ×1, Eater 피스트 ×1.5)
```

예: `damage=10, efficiency=0.7` → 1단계: 7 → 2단계: `×1.0` → 3단계: 방어력 4면 `7-4=3` 실제 피해.

### 충격 기반 충돌 물리

볼끼리 충돌하면 `BattleSimulation`이 두 볼의 상대 속도, 질량(`mass`), 반발 계수(`COLLISION_RESTITUTION`)를 기준으로 impulse를 계산해 양쪽 `velocity`에 더합니다. 이 충격 속도는 다음 틱에 즉시 사라지지 않고 유지되며, `BattleBall`은 매 틱 현재 속도를 기본 주행 속도 쪽으로 일정 비율만 보정합니다.

즉, 충돌 직후에는 튕겨 나가는 속도가 실제 전투에 남고, 시간이 지나면서 각 캐릭터의 기본 속도/스탯/이동 효과 기준으로 자연스럽게 복귀합니다. 이 구조의 목적은 공이 겹친 상태에서 매 프레임 최소 피해(`1`)가 반복으로 들어가는 현상을 줄이고, Dash/Rage/넉백처럼 빠른 이동 상태가 충돌 물리와 같은 방식으로 섞이게 하는 것입니다.

### 넉백 시스템

모든 공격은 명중 시 `applyKnockback(velocity, duration)`으로 넉백 impulse를 즉시 더하고, duration 동안 `forceHeading`으로 방향만 고정합니다. `forceHeading`은 더 이상 고정 속도(`overrideVelocity`)를 소유하지 않습니다. 실제 `velocity`는 충돌/넉백 impulse를 보존한 채 기본 주행 속도 쪽으로 보정됩니다. 벽에 닿으면 넉백 방향 고정은 종료됩니다.

| 공격                | 지속시간 | 속도                          |
| ------------------- | -------- | ----------------------------- |
| Arrow               | 0.2초    | 화살속도 × 0.6                |
| Orbit 근접 위성     | 0.3초    | targetSpeed × 2.5             |
| Orbit 원거리 발사체 | 0.15초   | 발사체속도 × 0.4              |
| Grenade 폭발        | 0.35초   | 400px/s (폭발 중심→대상 방향) |

### 방어력(defense) 상세

받는 피해 계산은 **뺄셈** 기반이며, 각 캐릭터의 기본 방어력에 능력 보정(곱)이 적용됩니다.

```
받는 피해 = max(1, 들어온 피해 - totalDefense)

totalDefense = round(baseDefense × abilityDefMult)
  - baseDefense: roster.js에 정의된 캐릭터별 기본 방어력 (스탯 배분으로 증가)
  - abilityDefMult: 각 Ability의 getStatModifiers().defense
    - 기본 ×1 (변화 없음)
    - Eater 피스트 중: ×1.5 (방어력 1.5배)
```

예: 방어력 2의 Eater가 10 피해를 받으면 `10 - 2 = 8`만 실제로 입습니다. 피스트 중에는 `10 - 3`으로 7 피해를 입습니다.

### 크리티컬 히트 (v0.26.0+)

크리티컬은 `criticalChance` 스탯으로 발동률이 결정됩니다. 기본 5% + 포인트당 +1%p입니다.

```
크리티컬 피해 = 들어온 피해 × 2 (방어력 차감 전에 2배)
```

- 적대 피해 이벤트(아군/자기 자신 제외)마다 독립적으로 roll.
- 크리티컬 roll 통과 시 방어력 차감 전 원본 피해를 2배로 만든 후 방어력 차감.
- 발동 시 금색 "CRIT! N" 플로팅 텍스트 표시.
- Archer Lv.9: 50% 확률로 critChance × 2(최대 100%)의 독립 roll.

## 스탯으로 열지 않은 값

아래 값은 시스템 스탯이지만 현재 사용자 배분 대상에서는 제외합니다.

- `radius`: 캐릭터 크기와 피격 판정, 시각 정체성을 동시에 바꿉니다.
- `mass`: 충돌 물리의 개성을 크게 바꿉니다.
- `force`: 이전 초안의 충격 스탯이었지만 실제 전투 계산에 연결되지 않아 제거했습니다.

`radius`와 `mass`는 캐릭터별 고유성에 가깝기 때문에 당장은 고정합니다. 추후 장비, 난이도, 특수 모드가 생기면 별도 규칙으로 다시 검토합니다.

## Spin Ball

Spin Ball은 충돌하지 않은 시간 동안 `3.5초`를 기준으로 회전 충전을 연속으로 쌓는다. 충전 상태는 목표 각속도 `20πrad/s`, 즉 초당 10회전으로 변환되고, 실제 회전은 `applyTorque()`로 그 목표를 따라간다. 부분 충전 충돌은 기존 공통 충돌 피해·분리·충전 소비를 유지한다.

- Lv.3 이상 만충 직접 충돌은 일반 Crash 실제 피해를 한 번 먼저 유지하고, 즉시 분리만 `0.60초` 뒤로 미룬다. 실제 표면 앵커를 `0.05초`마다 12틱 ×0.15, 합계 ×1.80으로 절단한 뒤, 충돌 순간 공통 물리가 계산한 양쪽 분리 에너지를 한 번 적용한다.
- Lv.6은 절단 피해를 첫 ×0.10에서 마지막 ×0.30까지 선형 증가시켜 합계 ×2.40으로 바꾼다. 절단 중 새 대상이나 같은 대상에 중첩하지 않는다.
- Lv.9 만충 전 유체장은 반경 `340px` 모든 적대 대상에 거리 smoothstep 감쇠와 접선 3:안쪽 1 방향의 최대 `420px/s²` 가속도를 적용한다. 첫 직접 충돌에서 종료되며, 이어지는 절단 12틱만 방어력을 무시한다.
- 절단 중에는 같은 두 전투원의 일반 충돌 피해·분리·충돌 후 능력 콜백을 건너뛴다. 종료 impulse 전까지 위치 스냅이나 `forceHeading`은 사용하지 않는다.
- 전투 카드에는 충전 중 `회전력`, 만충이면 `회전 절단`, 절단 중에는 누적 틱 수를 표시한다.

## Hero Ball / Hero Orb

### 능력: 성장 코어

Hero Ball은 최대 5개의 성장 스택을 1초마다 충전합니다. 적대 전투원과 실제로 충돌하면 현재 스택을 모두 소비하고 접점에서 같은 수의 성장 코어를 무작위 방향으로 방출합니다. 자동 시간 발사와 특수 오브는 사용하지 않습니다.

| 타입       | 색상             | 직접 회수 효과                         |
| ---------- | ---------------- | -------------------------------------- |
| `hp`       | 초록 (`#44dd44`) | `maxHp +5×amount`, 현재 HP도 함께 증가 |
| `damage`   | 빨강 (`#ff4444`) | `baseDamage × 1.02^amount`             |
| `speed`    | 파랑 (`#4488ff`) | `baseSpeed +4×amount`                  |
| `defense`  | 노랑 (`#dddd44`) | `baseDefense +0.33×amount`             |
| `skill`    | 보라 (`#bb66ff`) | 능력 쿨다운 스탯 `+amount`             |
| `critical` | 분홍 (`#ff7bd5`) | 치명타 확률 `+amount%p`, 최대 100%     |

`amount`는 2~6 중 균등 무작위다. 코어는 8초 동안 유지되며 Hero 소유자별 최대 5개만 존재한다. 여섯 번째 코어를 만들 때는 가장 오래된 활성 코어를 제거한다. 생성 직후 0.16초 동안은 소유자가 직접 회수할 수 없고, 적대 전투원과 부딪히면 피해 없이 물리적으로 튕긴다. 코어는 금색 육각 외곽선, 속성색 중심, 속성 아이콘과 소유자 연결선으로 구분한다.

- Lv.3: 성장 코어가 Hero 가까이에 들어오면 기존 물리 자석으로 Hero 방향에 가속한다. 순간 이동이나 전역 자동 회수는 없다.
- Lv.6: 실제 적용된 성장 코어마다 속성색 공명 조각을 최대 5개 저장한다. 다음 적대 전투원 충돌에서 조각을 0.1초 간격 곡선 궤도로 같은 대상에 보내고, 조각마다 총공격력 `×0.20` 피해를 준다.
- Lv.9: 정확히 5개 조각의 다섯 번째 적중에서만 Heroic Burst가 한 번 발동한다. 대상 포함 반경 90px의 모든 적에게 총공격력 `×0.75` 피해와 기본 충돌 넉백의 60%를 준다.

성장 스택은 Hero 둘레의 5분할 금색 링, 저장 조각은 공전하는 속성색 육각형으로 보인다. 소비 순간에는 방출 링, 조각 적중은 별 꼭짓점, Heroic Burst는 완성된 별과 금빛 충격 고리로 구분한다.

### Hero Orb UI 표시

- Hero Ball의 fighter card 스탯 줄은 시작 전 배분과 Hero Orb 보너스를 같은 항목에 합쳐 표시합니다.
- 예: `체력 +30%(+3) · 힘 +20%(+1) · 속도 +10%`
- Hero Orb 보너스 표기는 기본 배분 뒤에 공백 없이 붙입니다. 예: `+10%(+3)`
- 앞의 `%` 값은 토너먼트 시작 전 배분한 기본 스탯이고, 뒤의 `+n`은 전투 중 Hero Orb로 먹은 누적 보너스입니다.
- Hero Orb 보너스가 0인 스탯은 기본 배분만 표시합니다.
- Hero Ball은 표시 문자열이 길어질 수 있으므로 fighter card 스탯 줄의 폰트 크기를 다른 캐릭터보다 작게 사용합니다.

### Hero Ball 승리 시 스탯 계승 (Carryover)

Hero Ball(플레이어 여부와 관계없이 모든 Hero Ball)이 경기에서 승리하면, 이번 경기에서 성장 코어로 얻은 일반 스탯 6종의 절반을 **다음 스테이지에 계승**합니다.

- 계승량: 각 스탯별 `floor(이번 경기 획득량 × HERO_ORB_CARRYOVER_RATE)` (기본 `0.5`)
- 예: hp +5 → carry +2, damage +1 → carry +0, speed +4 → carry +2
- 패배 시 계승 없음
- 기존 carryover는 다시 절반 계산하지 않음 (이번 경기 신규 획득량만 기준)
- carryover 효과는 기존 Hero Orb 1점당 효과와 동일합니다.
    - hp carry 1 = `maxHp +5`, 현재 HP도 +5
    - damage carry 1 = `baseDamage × 1.02`
    - speed carry 1 = `baseSpeed +4`
    - defense carry 1 = `baseDefense +0.33`
    - skill carry 1 = `statAllocation.skill +1` (Ability.cooldown 반영)
    - critical carry 1 = `criticalChance +1%p` (최대 100%)
- UI 괄호 `(+n)`에는 carryover + 이번 경기 신규 획득량이 합산 표시됩니다.

### effect registry 확장

성장 코어의 effect type은 `HERO_ORB_EFFECTS` 레지스트리로 관리합니다. 각 entry의 실제 적용 결과만 공명 조각과 계승에 반영한다.

## 등수 규칙

현재 8명 로스터는 8강 슬롯에 배치되며 일부 캐릭터는 부전승을 받을 수 있습니다.

## 토너먼트 참가자 선발 규칙 (v0.10.0+)

- **전체 roster가 8명 이상** → 유저 캐릭터 1명 + 유저 제외 랜덤 7명 = 총 **8명**이 토너먼트에 참가합니다.
- **전체 roster가 정확히 8명** → 기존과 동일하게 유저 캐릭터 + 나머지 7명이 모두 참가합니다.
- **전체 roster가 8명 미만** → 가능한 모든 캐릭터가 참가하며, 기존 부전승(bye) 로직을 유지합니다. 강제로 8명을 채우기 위한 중복 캐릭터를 추가하지 않습니다.
- 유저 캐릭터가 상대 풀에 중복으로 들어가지 않습니다.
- 플레이어는 저장된 캐릭터 XP 레벨과 해금한 행동 강화 tier를 그대로 사용합니다.
- 다음 토너먼트에서 선택된 AI 전원은 현재 플레이어의 토너먼트 도전 단계에 따라 같은 시작 경험치 진행도를 사용합니다: 도전 0은 기본 Lv.1, 도전 1은 Lv.3, 도전 2는 Lv.6, 도전 3은 Lv.9입니다. 이 진행도는 AI 기본 스탯과 행동 강화 tier에 함께 적용되며 사냥터나 프로필 저장 XP는 바꾸지 않습니다. 숙련도는 도전 단계와 별도로 영구 보존됩니다.
- 시작 전 플레이어 패널은 현재 숙련도 등급 또는 `첫 도전`을, 바로 옆에 현재 도전 단계에서 계산한 AI의 실제 시작 레벨을 함께 표시합니다. 환생하면 도전 단계는 0으로 초기화되어 Lv.1 AI부터 다시 시작하지만 숙련도는 유지합니다.
- 첫 토너먼트 매치에서 클릭 액션 카드를 선택하면, VS 대진 직전에 같은 단계와 AI 시작 레벨을 오버레이로 한 번 더 보여 줍니다. 이미 선택한 액션을 이어 쓰는 다음 매치와 사냥터에는 이 안내를 표시하지 않습니다.

### 토너먼트 축 반사 완화 삼각형

토너먼트 전투에서만 현재 속도 직선으로 다음 벽 충돌을 짧게 예측한다. 벽의 안쪽 법선을 기준으로 한 입사각이 `25°` 이하이고 도달까지 `0.25초` 이하일 때, 예상 충돌점의 벽선에 길이 `120px`인 변을 붙인 임시 직각삼각형 하나를 만든다. 빗변의 법선은 벽 법선에서 `10°`만 기울고, 삼각형의 경기장 안쪽 깊이는 벽변 길이에서 `120 × tan(10°)`, 약 `21px`로 파생한다. 시드 난수로 고른 위·아래 방향 중 한쪽에 직각 꼭짓점을 둔다.

- 경사면은 기존 사냥터 다각형 지형과 같은 충돌·나무 재질 계약을 사용한다. 충돌 반사와 회전 충격은 공통 물리 응답이 계산하며, 경사면 시스템은 `velocity`나 `angularVelocity`를 직접 바꾸지 않는다.
- 벽 그 자체의 형상·재질·충돌 규칙은 바꾸지 않는다. 삼각형은 실제 공 충돌 직후 제거되며, 충돌하지 않으면 `0.7초` 뒤 제거한다. 생성과 제거 시에는 삼각형 중심에서만 짧은 파티클을 방출한다.
- 같은 공은 경사면을 만든 뒤 `1.2초` 동안 새 경사면을 요청할 수 없고, 경기장에는 동시에 하나만 존재한다. 코너, 기존 지형 또는 전투원과 겹칠 위치는 생성하지 않는다.
- 사냥터, 캐릭터 프리뷰, 일반 매치, 디버그 시뮬레이션에는 기본적으로 꺼져 있다. 충돌·가속·능력으로 속도가 바뀌면 긴 예측은 신뢰할 수 없으므로, 짧은 선행 시간에서만 사용한다.

- 우승: 1위
- 결승 탈락: 2위
- 준결승 탈락: 공동 3위
- 1라운드 탈락: 공동 5위

동률 표기는 현재 토너먼트 구조를 단순하게 설명하기 위한 규칙입니다.

## 밸런스 메모
