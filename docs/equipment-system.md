# 장비 시스템

> 상태: 개념 설계, 구현 전 (MVP 제외)
> 기준 문서: 2026-07-03 인터뷰 결정
> 관련 문서: [`hunting-grounds-system.md`](hunting-grounds-system.md), [`experience-system.md`](experience-system.md)

## 1. 시스템 정의

장비 시스템은 사냥터의 핵심 보상입니다. 로그라이크 게임처럼 각 장비는 **랜덤 수치**를 가지며, 사냥터에서 획득하여 캐릭터에 장착할 수 있습니다.

목적:
- 사냥터에 지속적인 파밍 동기 부여
- 토너먼트가 어려운 플레이어가 장비 파밍으로 캐릭터를 간접 강화
- 매 런 다른 장비 구성으로 다양한 플레이 경험

## 2. 설계 원칙

### 2.1 로그라이크 스타일 랜덤 수치

각 장비는 기본 스탯(HP, 공격력, 방어력, 속도 등)에 **랜덤 보너스**를 부여합니다. 같은 이름의 장비라도 드롭될 때마다 수치가 다릅니다.

### 2.2 사냥터 전용 획득

장비는 사냥터 전투/이벤트 보상으로만 획득할 수 있습니다. 토너먼트 보상으로는 장비를 주지 않습니다.

### 2.3 토너먼트에도 적용

획득한 장비는 **사냥터와 토너먼트 모두**에 장착할 수 있습니다. 이것이 "사냥터에서 파밍해서 토너먼트에 도전" 선순환의 핵심입니다.

### 2.4 슬롯 제한

모든 장비를 무제한 장착할 수 없습니다. 슬롯(무기/방어구/장신구 등) 제한으로 빌드 선택지를 만듭니다.

## 3. 용어

| 용어 | 의미 |
| --- | --- |
| 장비 | 랜덤 스탯 보너스를 가진 아이템 |
| 슬롯 | 장비를 장착할 수 있는 칸 |
| 등급 | 장비의 품질 (common/uncommon/rare/epic/legendary) |
| 기본 스탯 | HP, damage, defense, speed 중 하나에 보너스 |
| 특수 옵션 | 스탯 외 특수 효과 (ex: 충돌 데미지 +10%, 쿨다운 -5%) |

## 4. (초안) 장비 설계 방향

### 4.1 슬롯 구성 (미정)

```
무기 슬롯 1칸 — 공격형 스탯 위주
방어구 슬롯 1칸 — 방어형 스탯 위주
장신구 슬롯 1~2칸 — 혼합/특수 옵션
```

### 4.2 스탯 범위 (미정, 밸런스 필요)

| 등급 | 스탯 보너스 범위 | 특수 옵션 확률 |
| --- | --- | ---: |
| Common | +1~3 | 0% |
| Uncommon | +2~5 | 10% |
| Rare | +3~8 | 25% |
| Epic | +5~12 | 50% |
| Legendary | +8~15 | 80% |

### 4.3 중복 장착

동일한 장비를 2개 장착할 수 없습니다. 같은 슬롯에는 하나만 장착 가능합니다.

### 4.4 장비 파밍처

| 출처 | 등급 범위 |
| --- | --- |
| 일반 적 처치 | Common ~ Uncommon |
| 정예 적 처치 | Uncommon ~ Rare |
| 층 클리어 보너스 | Common ~ Rare |
| 상자 개봉 | Common ~ Legendary |
| 이벤트 보상 | Uncommon ~ Epic |

## 5. MVP 이후 구현 항목

- [ ] 장비 데이터 구조 정의 (id, name, slot, stats, specialOptions, rarity)
- [ ] 장비 드롭 로직 (등급별 확률, 스탯 랜덤 범위)
- [ ] 장비 인벤토리 UI (보유 장비 목록, 장착/해제)
- [ ] 장착 슬롯 UI (캐릭터 설정 화면)
- [ ] 장비 스탯 → 전투 스탯 반영 로직
- [ ] 사냥터 보상에 장비 추가
- [ ] 장비 중복 처리 (분해/합성/판매)

## 6. 후속 결정

- 슬롯 개수와 종류를 확정해야 합니다.
- 장비가 특수 옵션을 가질지, 순수 스탯 위주로 갈지 결정해야 합니다.
- 장비 등급별 드롭 확률을 balanceSim으로 검증해야 합니다.
- 장비 분해/재활용 시스템이 필요한지 결정해야 합니다.
- 장비 랜덤 수치의 밸런스 범위를 결정해야 합니다.
