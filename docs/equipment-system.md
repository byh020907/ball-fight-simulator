# 장비 시스템

> 상태: MVP 구현 완료
> 기준 문서: 2026-07-03 인터뷰 결정
> 관련 문서: [`hunting-grounds-system.md`](hunting-grounds-system.md), [`experience-system.md`](experience-system.md)

## 1. 시스템 정의

장비 시스템은 사냥터의 핵심 보상입니다. 각 장비는 **랜덤 수치**를 가지며 사냥터에서 획득하여 캐릭터에 **영구 장착**할 수 있습니다. 장착한 장비는 사냥터와 토너먼트 모두에 적용됩니다.

목적:
- 사냥터에 지속적인 파밍 동기 부여
- 토너먼트가 어려운 플레이어가 장비 파밍으로 캐릭터를 간접 강화
- 매 런 다른 장비 구성으로 다양한 플레이 경험

## 2. 설계 원칙

### 2.1 영구 소장

장비는 사냥터에서 획득하면 **영구적으로 보유**합니다. 귀환/패배로 사라지지 않습니다. 언제든 장착/해제 가능합니다.

### 2.2 사냥터 전용 획득

장비는 사냥터 전투/이벤트 보상으로만 획득합니다. 토너먼트 보상으로는 장비를 주지 않습니다.

### 2.3 토너먼트에도 적용

획득한 장비는 **사냥터와 토너먼트 모두**에 장착 가능합니다. "사냥터 파밍 → 토너먼트 도전" 선순환의 핵심입니다.

### 2.4 슬롯 제한

슬롯 시스템으로 장비 착용을 제한합니다. 기본 슬롯 구성은 아래와 같으며, 변수화하여 변경이 쉽게 설계합니다.

### 2.5 확장성

모든 스탯 종류(HP, Damage, Defense, Speed, 특수 옵션)를 지원하며, 데이터 구조는 새 스탯/효과 추가가 용이해야 합니다. 외형 draw()도 장비별 메서드 호출 방식으로 확장 가능하게 설계합니다.

## 3. 용어

| 용어 | 의미 |
| --- | --- |
| 장비 | 랜덤 스탯 보너스를 가진 영구 아이템 |
| 슬롯 | 장비를 장착할 수 있는 칸 (개수/종류 변수화) |
| 등급 | 장비의 품질 (common/uncommon/rare/epic/legendary) |
| 스탯 | HP, damage, defense, speed 중 하나에 보너스 |
| 특수 옵션 | 파쇄·순환·갈망처럼 기본 스탯과 별도로 전투 규칙을 바꾸는 효과 |
| 강화석 | 장비 분해 시 획득하는 강화 전용 재화 |
| 강화 레벨 | 장비의 강화 단계 (0~+5) |

## 4. 슬롯 구성

기본값 (변수화, 추후 조정 가능):

```
무기 슬롯 x1  — 공격형 스탯 위주
방어구 슬롯 x1 — 방어형 스탯 위주
장신구 슬롯 x2 — 혼합/특수 옵션
```

슬롯 수와 종류는 `EQUIPMENT_SLOTS` 같은 설정 배열로 관리하여 코드 변경 없이 조정 가능합니다.

## 5. 장비 등급

### 5.1 등급별 스탯 범위

각 장비는 드롭 시 랜덤으로 **1~2개의 스탯**을 부여받습니다. Rare 이상은 특수 옵션이 추가로 등장할 수 있습니다.

| 등급 | 스탯 보너스 범위 | 특수 옵션 등장 |
| --- | --- | ---: |
| Common | +1~3 | 없음 |
| Uncommon | +2~5 | 없음 |
| Rare | +4~8 | 있음 (25%) |
| Epic | +6~12 | 있음 (50%) |
| Legendary | +10~18 | 있음 (80%) |

### 5.2 스탯 종류

- `hp` — 최대 HP 증가
- `damage` — 충돌/기본 공격력 증가
- `defense` — 받는 피해 감소
- `speed` — 이동 속도 증가
- 특수 옵션 (Rare+): 파쇄(충돌 피해 +X%), 순환(스킬 쿨다운 -X%), 갈망(충돌 피해 기반 HP 회복 X%), 중량·반향·소용돌이(물리 파라미터 보정)

특수 옵션 목록은 확장 가능하며, `src/hunting/equipmentData.js`의 `SPECIALS.POOL`로 정의합니다.

### 5.3 특수 옵션 적용 규칙

- 특수 옵션은 장비명 접미사로 표시한다. 예: `질풍의 철검 갈망`.
- 같은 특수 옵션을 여러 장착 장비에서 얻으면 가장 높은 수치 하나만 적용한다. 서로 다른 특수 옵션은 함께 적용한다.
- `파쇄`는 적대 전투원 간 한 번의 충돌 거래에서 자신이 주는 기본 충돌·접촉형 스킬 피해를 증가시킨다.
- `순환`은 스킬 스탯과 숙련도 보정에 이어 적용되는 쿨다운 감소 배율이다.
- `갈망`은 해당 충돌 거래에서 준 실제 총 피해를 기준으로 회복한다. HP가 부족하고 `2.5초` 장비 효과 쿨다운이 준비됐을 때만 발동한다. 투사체 피해는 갈망 거래에 포함하지 않는다.
- `중량`은 장착 볼의 실제 질량을 증가시킨다. 충돌 solver가 반동, 상대 전달량, 회전 관성을 함께 계산한다.
- `반향`은 벽 반사 직후 바깥쪽 법선 속도를 증가시킨다. 반사 결과에 물리 속도 변화로만 더해진다.
- `소용돌이`는 fighter collision solver가 소유자에게 전달하는 각충격을 증가시킨다. 각속도를 직접 대입하지 않는다.

### 5.4 드롭 출처

| 출처 | 등급 범위 |
| --- | --- |
| 일반 적 처치 | Common ~ Uncommon |
| 정예 적 처치 | Uncommon ~ Rare |
| 층 클리어 보너스 | Common ~ Rare |
| 상자 개봉 | Common ~ Legendary |
| 이벤트 보상 | Uncommon ~ Epic |

## 6. 캐릭터 레벨 제한

더 높은 등급의 장비를 장착하려면 캐릭터 레벨이 필요합니다.

| 등급 | 필요 레벨 |
| --- | ---: |
| Common | 제한 없음 |
| Uncommon | Lv.3 |
| Rare | Lv.5 |
| Epic | Lv.8 |
| Legendary | Lv.10 |

레벨 제한은 캐릭터별로 적용됩니다. 캐릭터 A가 Lv.8이면 A에게 Epic 장착 가능, B가 Lv.2면 B에게 Uncommon만 가능.

현재 구현에서는 장착 버튼과 전투 스탯 적용이 같은 요구 레벨 판정을 사용합니다. 요구 레벨보다 낮은 캐릭터는 장비를 보유할 수는 있지만 장착할 수 없고, 기존에 장착 ID가 남아 있어도 전투 스탯과 외형에는 반영되지 않습니다.

## 7. 강화 시스템

### 7.1 기본 규칙

- 최대 강화 레벨: **+5**
- 강화 성공 시: 장비 스탯이 **+20% 증가** (최종: +100% = 2배)
- 강화 실패 시: 강화 레벨이 **1단계 하락** (장비 파괴 없음, 0 이하로는 안 떨어짐)

### 7.2 실패율 공식

```
실패율 = 최대실패율 × (현재강화레벨 / 최대강화레벨)

최대실패율 = 80%
최대강화레벨 = 5
```

| 시도 | 실패율 | 성공률 |
| ---: | ---: | ---: |
| +0 → +1 | 16% | 84% |
| +1 → +2 | 32% | 68% |
| +2 → +3 | 48% | 52% |
| +3 → +4 | 64% | 36% |
| +4 → +5 | 80% | 20% |

### 7.3 강화 재료 및 비용

**강화석** — 장비 분해 시 등급별로 획득하는 전용 재화.

| 분해 장비 등급 | 획득 강화석 |
| --- | ---: |
| Common | 1개 |
| Uncommon | 3개 |
| Rare | 8개 |
| Epic | 20개 |
| Legendary | 50개 |

**강화 시도 비용** (강화석 + 열쇠 조각):

| 시도 | 강화석 | 열쇠 조각 |
| ---: | ---: | ---: |
| +0 → +1 | 2 | 10 |
| +1 → +2 | 4 | 15 |
| +2 → +3 | 8 | 25 |
| +3 → +4 | 15 | 40 |
| +4 → +5 | 25 | 60 |

## 8. 중복 장비 처리

이미 보유한 장비를 중복 획득 시 3가지 처리 방식 모두 지원합니다.

| 방식 | 효과 |
| --- | --- |
| **분해** | 장비 제거 → 등급별 강화석 획득 |
| **합성 승급** | 같은 등급 장비 2개 + 추가 재료 소모 → 한 단계 높은 등급 장비 1개 (랜덤) |
| **판매** | 장비 제거 → 열쇠 조각 획득 (등급별 차등) |

현재 MVP 수치는 아래 기준으로 시작하며, 밸런스 단계에서 조정합니다.

| 방식 | 수치 |
| --- | --- |
| 합성 승급 | 같은 등급 장비 2개 + 강화석/파편 비용 → 한 단계 높은 랜덤 장비 1개 |
| 판매 | Common 5 / Uncommon 12 / Rare 30 / Epic 80 / Legendary 200 파편 |

| 합성 기준 등급 | 강화석 | 파편 |
| --- | ---: | ---: |
| Common | 2 | 20 |
| Uncommon | 5 | 40 |
| Rare | 12 | 80 |
| Epic | 25 | 150 |

## 9. 인벤토리

- **기본 용량**: 5칸
- **확장**: 열쇠 조각 100개 소모 → +3칸 확장
- 확장은 횟수 제한 없음 (단, 비용 고정)
- 용량 초과 시 새 장비를 획득할 수 없으며, 인벤토리 정리 후 획득 가능

## 10. 장비 외형

장비는 장착 시 전투 캔버스에 시각 효과를 표시할 수 있습니다. **MVP에서는 제외**하지만, 데이터 구조는 아래 규칙으로 설계합니다.

- 각 장비는 저장 데이터에 `draw` 키를 가짐 (`weapon`/`armor`/`accessory`)
- 전투 스펙 생성 시 장착 장비 목록을 `equipment.equippedItems`로 전달
- 전투 렌더러는 `BattleBall.draw()`에서 슬롯별 draw 함수를 순차 호출
- draw 함수는 볼 position/radius 정보를 받아 캔버스에 오버레이 그림
- 무기: 몸 오른쪽 외곽에 장착된 창/칼날 실루엣
- 방어구: 얼굴을 가리지 않는 상단/하단 장갑 띠와 왼쪽 방패
- 장신구: 장갑 띠 위의 작은 원형 보석/리벳
- 장비는 얼굴 중앙 영역을 피하고, 장착물로 읽히는 외곽 실루엣을 우선합니다.

## 11. UI 위치

장비 관리는 다음 두 곳에서 모두 접근 가능합니다.

| 위치 | 기능 |

| **토너먼트/사냥터 시작 전** | 현재 장착 슬롯/적용 스탯 요약 + 장비 탭 바로가기 |

## 12. 데이터 구조 (권장)

```js
// 장비 정의
{
    id: "iron_sword",
    name: "철검",
    description: "기본적인 철제 검",
    rarity: "common",          // common | uncommon | rare | epic | legendary
    slot: "weapon",            // weapon | armor | accessory
    stats: [
        { type: "damage", value: 3, min: 1, max: 3 }  // 랜덤 range 기록 (재계산용)
    ],
    specialOptions: null,      // [{ type: "crashDamage", value: 10 }]
    enhanceLevel: 0,           // 0~5
    draw: "weapon"             // 런타임 draw 키
}

// 플레이어 프로필 내 인벤토리
{
    equipment: {
        inventory: [
            { /* 장비 데이터 */, instanceId: "uuid" }  // 중복 방지용 instanceId
        ],
        equipped: {
            weapon: "uuid" | null,
            armor: "uuid" | null,
            accessory1: "uuid" | null,
            accessory2: "uuid" | null
        },
        enhancementStones: 0,
        maxInventorySlots: 5
    }
}
```
