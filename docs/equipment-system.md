# 장비 시스템

> 상태: MVP 구현 완료
> 기준 문서: 2026-07-03 인터뷰 결정
> 관련 문서: [`hunting-grounds-system.md`](hunting-grounds-system.md), [`experience-system.md`](experience-system.md)

## 1. 시스템 정의

장비 시스템은 사냥터의 핵심 보상입니다. 각 장비는 **랜덤 수치**를 가지며 사냥터에서 획득하여 캐릭터에 **영구 장착**할 수 있습니다. 장착한 장비는 사냥터와 토너먼트 모두에 적용됩니다.

목적:
- 사냥터에 지속적인 파밍 동기 부여
- 토너먼트가 어려운 플레이어가 장비 파밍으로 캐릭터를 간접 강화
- 매 런 다른 장비 구성으로 다양한 플레이 경험

## 2. 설계 원칙

### 2.1 영구 소장

장비는 사냥터에서 획득하면 **영구적으로 보유**합니다. 귀환/패배로 사라지지 않습니다. 언제든 장착/해제 가능합니다.

### 2.2 사냥터 전용 획득

장비는 사냥터 전투/이벤트 보상으로만 획득합니다. 토너먼트 보상으로는 장비를 주지 않습니다.

### 2.3 토너먼트에도 적용

획득한 장비는 **사냥터와 토너먼트 모두**에 장착 가능합니다. "사냥터 파밍 → 토너먼트 도전" 선순환의 핵심입니다.

### 2.4 슬롯 제한

슬롯 시스템으로 장비 착용을 제한합니다. 기본 슬롯 구성은 아래와 같으며, 변수화하여 변경이 쉽게 설계합니다.

### 2.5 확장성

모든 스탯 종류(HP, Damage, Defense, Speed, 특수 옵션)를 지원하며, 데이터 구조는 새 스탯/효과 추가가 용이해야 합니다. 장비 외형은 개별 아이템 그림 대신 활성 장착 목록의 공통 문양으로 표현해 모든 전투·프리뷰가 같은 규칙을 재사용합니다.

## 3. 용어

| 용어 | 의미 |
| --- | --- |
| 장비 | 랜덤 스탯 보너스를 가진 영구 아이템 |
| 슬롯 | 장비를 장착할 수 있는 칸 (개수/종류 변수화) |
| 등급 | 장비의 품질 (common/uncommon/rare/epic/legendary) |
| 스탯 | HP, damage, defense, speed 중 하나에 보너스 |
| 특수 옵션 | 파쇄·순환·갈망처럼 기본 스탯과 별도로 전투 규칙을 바꾸는 효과 |
| 강화석 | 장비 판매 시 확률로 획득하며 강화 실패 단계를 복구하는 재화 |
| 강화 레벨 | 등급별 강화 단계 (common +1 ~ legendary +5) |

게임 UI의 등급 표기는 내부 키와 같은 소문자 영문 `common`, `uncommon`, `rare`, `epic`, `legendary`를 사용한다. 일반·고급·희귀처럼 번역한 등급명이나 대문자 표기를 섞지 않는다.

사냥터 일반 몬스터도 이 희귀도 체계의 앞 네 단계(`common`·`uncommon`·`rare`·`epic`)를 공유합니다. 일반 몬스터에는 `legendary`가 없고, 별도 `unique` 키도 사용하지 않습니다. 몬스터 희귀도는 XP·전장 루팅·도감 업적의 경제 단계를 선택하지만 장비 자체의 생성 규칙은 바꾸지 않습니다.

## 4. 슬롯 구성

기본값 (변수화, 추후 조정 가능):

```
무기 슬롯 x1  — 공격형 스탯 위주
방어구 슬롯 x1 — 방어형 스탯 위주
장신구 슬롯 x2 — 혼합/특수 옵션
```

슬롯 수와 종류는 `EQUIPMENT_SLOTS` 같은 설정 배열로 관리하여 코드 변경 없이 조정 가능합니다.

## 5. 장비 등급

### 5.1 등급별 스탯 범위

각 장비는 드롭 시 랜덤으로 **1~2개의 스탯**을 부여받습니다. rare 이상은 특수 옵션이 추가로 등장할 수 있습니다.

| 등급 | 스탯 보너스 범위 | 특수 옵션 등장 |
| --- | --- | ---: |
| common | +1~3 | 없음 |
| uncommon | +2~5 | 없음 |
| rare | +4~8 | 있음 (25%) |
| epic | +6~12 | 있음 (50%) |
| legendary | +10~18 | 있음 (80%) |

### 5.2 스탯 종류

- `hp` — 최대 HP 증가
- `damage` — 충돌/기본 공격력 증가
- `defense` — 비율 피해 감소에 쓰는 방어 점수. 실제 피해는 `피해 × 50 / (50 + 방어)`로 계산하며 최소 1입니다.
- `speed` — 장비 속도 점수. 장비 적용 전 속도에 더해지되, 최종 속도는 그 기준의 2배를 향해 완만하게 수렴합니다. 능력과 사냥터 터치 가속은 그 뒤에 별도로 적용됩니다.
- 특수 옵션 (rare+): 파쇄(충돌 피해 +X%), 순환(스킬 쿨다운 -X%), 갈망(충돌 피해 기반 HP 회복 X%), 중량·반향·소용돌이(물리 파라미터 보정)

특수 옵션 목록은 확장 가능하며, `src/hunting/equipmentData.js`의 `SPECIALS.POOL`로 정의합니다.

### 5.3 특수 옵션 적용 규칙

- 특수 옵션은 장비명과 구분되는 점 접미사로 표시한다. 예: `질풍의 철검 • 갈망`.
- 같은 특수 옵션을 여러 장착 장비에서 얻으면 가장 높은 수치 하나만 적용한다. 서로 다른 특수 옵션은 함께 적용한다.
- `파쇄`는 적대 전투원 간 한 번의 충돌 거래에서 자신이 주는 기본 충돌·접촉형 스킬 피해를 증가시킨다.
- `순환`은 스킬 스탯과 숙련도 보정에 이어 적용되는 쿨다운 감소 배율이다.
- `갈망`은 해당 충돌 거래에서 준 실제 총 피해를 기준으로 회복한다. HP가 부족하고 `2.5초` 장비 효과 쿨다운이 준비됐을 때만 발동한다. 투사체 피해는 갈망 거래에 포함하지 않는다.
- `중량`은 장착 볼의 실제 질량을 증가시킨다. 충돌 solver가 반동, 상대 전달량, 회전 관성을 함께 계산한다.
- `반향`은 벽 반사 직후 바깥쪽 법선 속도를 증가시킨다. 반사 결과에 물리 속도 변화로만 더해진다.
- `소용돌이`는 fighter collision solver가 소유자에게 전달하는 각충격을 증가시킨다. 각속도를 직접 대입하지 않는다.

### 5.4 드롭 출처

| 출처 | 등급 범위 |
| --- | --- |
| 일반 적 처치 | common ~ uncommon |
| 정예 적 처치 | uncommon ~ rare |
| 층 클리어 보너스 | common ~ rare |
| 상자 개봉 | common ~ legendary |
| 이벤트 보상 | uncommon ~ epic |

## 6. 캐릭터 레벨 제한

더 높은 등급의 장비를 장착하려면 캐릭터 레벨이 필요합니다.

| 등급 | 필요 레벨 |
| --- | ---: |
| common | 제한 없음 |
| uncommon | Lv.3 |
| rare | Lv.5 |
| epic | Lv.8 |
| legendary | Lv.10 |

레벨 제한은 캐릭터별로 적용됩니다. 캐릭터 A가 Lv.8이면 A에게 epic 장착 가능, B가 Lv.2면 B에게 uncommon만 가능.

현재 구현에서는 장착 버튼, 전투 스탯·외형, 시작 전 메인 화면의 `적용 중인 장비 스탯` 요약이 같은 요구 레벨 판정을 사용합니다. 요구 레벨보다 낮은 캐릭터는 장비를 보유할 수는 있지만 장착할 수 없고, 기존에 장착 ID가 남아 있어도 전투 스탯·외형·메인 화면의 스탯 합계와 활성 장비 수에는 반영되지 않습니다. 메인 화면 슬롯의 잠금 표시와 필요 레벨 안내는 유지됩니다.

## 7. 강화 시스템

### 7.1 기본 규칙

- 등급별 최대 강화 레벨: **common +1 / uncommon +2 / rare +3 / epic +4 / legendary +5**
- 강화 성공 시: 장비의 기본 스탯 배율이 단계마다 **2배**가 됩니다. (`×2 / ×4 / ×8 / ×16 / ×32`)
- 특수 옵션의 수치에는 강화 배율이 적용되지 않습니다.
- 강화 실패 시: 강화 레벨이 **1단계 하락** (장비 파괴 없음, 0 이하로는 안 떨어짐)

### 7.2 실패율 공식

```
실패율 = 최대실패율 × ((현재강화레벨 + 1) / 전체강화단계수)

최대실패율 = 80%
전체강화단계수 = 5
```

| 시도 | 실패율 | 성공률 |
| ---: | ---: | ---: |
| +0 → +1 | 16% | 84% |
| +1 → +2 | 32% | 68% |
| +2 → +3 | 48% | 52% |
| +3 → +4 | 64% | 36% |
| +4 → +5 | 80% | 20% |

### 7.3 강화 재료 및 비용

강화 시도에는 파편만 사용합니다. 실패해 단계가 내려가면 결과 팝업에서 강화석 1개로 방금 하락한 단계를 복구할 수 있습니다. 복구는 자동으로 실행되지 않으며 팝업을 닫거나 다음 조작을 시작하면 지난 실패를 다시 복구할 수 없습니다.

| 시도 | 파편 |
| ---: | ---: |
| +0 → +1 | 200 |
| +1 → +2 | 300 |
| +2 → +3 | 500 |
| +3 → +4 | 800 |
| +4 → +5 | 1200 |

## 8. 중복 장비 처리

이미 보유한 장비는 합성하거나 판매할 수 있습니다.

| 방식 | 효과 |
| --- | --- |
| **합성 승급** | 같은 등급 비장착 장비 3개 → 한 단계 높은 등급 장비 1개 (랜덤) |
| **판매** | 장비 제거 → 파편 확정, 등급별 확률로 강화석 추가 획득 |

현재 MVP 수치는 아래 기준으로 시작하며, 밸런스 단계에서 조정합니다.

| 방식 | 수치 |
| --- | --- |
| 합성 승급 | 같은 등급 비장착 장비 3개 → 한 단계 높은 랜덤 장비 1개 |
| 판매 | common 5 / uncommon 12 / rare 30 / epic 80 / legendary 200 파편 + 확률 강화석 |

| 합성 기준 등급 | 파편 |
| --- | ---: |
| common | 0 |
| uncommon | 0 |
| rare | 0 |
| epic | 0 |

합성은 장비 3개를 소모하는 것 자체를 비용으로 보고 추가 파편을 요구하지 않습니다. 파편은 확정 성장에 가까운 강화의 핵심 비용으로 집중합니다. 1층 일반 몬스터의 평균 25파편 수급을 기준으로 +0 강화는 약 8마리분부터 시작하며, +0부터 +5까지 모두 성공해도 총 3000파편을 사용합니다.

합성 UI는 같은 등급의 비장착 장비 중 능력치 가치가 가장 낮은 3개를 자동 추천합니다. 능력치 가치는 기존 능력치 가치 비율로 정규화한 합계에 강화 배율을 반영해 계산합니다. 사용자는 실제로 사라질 장비 3개와 결과 등급, 파편 비용을 확인한 뒤 한 번씩 합성합니다. 합성 후 같은 화면에서 다음 추천이 즉시 갱신됩니다. 결과 장비는 새로 생성되므로 기존 슬롯, 스탯, 특수 옵션, 강화 단계는 이어지지 않습니다.

| 판매 등급 | 강화석 확률 | 성공 시 개수 |
| --- | ---: | ---: |
| common | 5% | 1 |
| uncommon | 10% | 1 |
| rare | 18% | 2 |
| epic | 30% | 4 |
| legendary | 50% | 8 |

강화석 확률에는 누적 천장이 없습니다. 등급이 높을수록 획득 확률과 성공 시 개수가 함께 증가합니다.

## 9. 인벤토리

- **기본 용량**: 5칸
- **확장**: 열쇠 조각 100개 소모 → +3칸 확장
- 확장은 횟수 제한 없음 (단, 비용 고정)
- 용량 초과 시 새 장비를 획득할 수 없으며, 인벤토리 정리 후 획득 가능

## 10. 장비 외형

장비 외형은 개별 무기·방어구·장신구 그림이 아니라, **공 내부 얼굴 바깥쪽을 따라 그리는 단일 점선 원**으로 통일합니다. 그래서 장착 수가 늘어도 캐릭터의 얼굴, 능력 이펙트, 궤적, 외곽 실루엣이 복잡해지지 않습니다.

### 10.1 활성 장비 목록

- 전투 스펙과 두 종류의 프리뷰는 모두 `getEquippedItems(profile, characterId)`가 반환한 `equipment.equippedItems`만 사용합니다.
- 이 목록은 무기, 방어구, 장신구1, 장신구2 슬롯 순서를 유지합니다.
- 캐릭터 레벨 제한을 만족하지 못한 장비는 장착 ID가 남아 있어도 목록에서 제외합니다. 따라서 스탯과 외형 모두에 반영되지 않습니다.
- 장비 인스턴스의 기존 `draw` 값은 저장 데이터 호환용 메타데이터로 남길 수 있지만, 외형 렌더러는 더 이상 슬롯별 그림을 분기하지 않습니다.

### 10.2 점선 문양 규칙

- 점선 원의 중심 반지름은 공 반지름에서 캐릭터 외곽선 두께의 절반, 점선 두께의 절반, 0.6px 안쪽 여백을 차감해 계산합니다. 점선의 바깥 끝은 항상 공 외곽 검은선의 안쪽 끝 바로 아래에 두되, 얼굴의 눈·입 영역과 능력 이펙트는 침범하지 않습니다.
- 활성 장비 수가 1/2/3/4개일 때 점선 수는 각각 6/12/18/24개입니다. 한 개만 장착하면 성기게, 네 개를 모두 장착하면 거의 완성된 원처럼 보입니다.
- 점선은 원 주위에 균등 간격으로 배치하며, 작은 프리뷰(지름 약 46px)에서도 끊김을 알아볼 수 있는 선 굵기와 길이를 유지합니다.
- 점선 색은 활성 장비의 등급색을 장착 슬롯 순서대로 반복합니다. 같은 등급 장비가 연속돼도 각 점선은 해당 순서의 장비 색을 사용합니다.
- 강화는 해당 장비가 맡은 점선의 불투명도만 올립니다. 선 굵기, 길이, 위치, 회전, 광원 효과는 강화로 바뀌지 않습니다.
- 활성 장비가 없으면 점선 문양을 그리지 않습니다.

### 10.3 공통 렌더링 위치

- 일반 전투와 사냥터 전투의 `BattleBall.draw()`는 얼굴을 그리기 전에 공통 점선 문양을 그립니다.
- 설정 화면의 작은 플레이어 얼굴 프리뷰도 같은 렌더러와 같은 활성 장비 목록을 사용합니다.
- 캐릭터 재선택 애니메이션의 나가는 공과 들어오는 공도 각각 해당 캐릭터의 활성 장비 목록을 전달받아 같은 문양을 그립니다.
- 장비 문양은 별도 애니메이션이 없고, 공이 물리적으로 회전할 때만 공과 함께 회전합니다.

## 11. UI 위치

장비 관리는 다음 두 곳에서 모두 접근 가능합니다.

| 위치 | 기능 |
| --- | --- |
| **컬렉션 허브 — 장비 탭** | 인벤토리, 장착/해제, 판매, 강화, 전용 합성 UI |
| **토너먼트/사냥터 시작 전** | 현재 장착 슬롯/적용 스탯 요약 + 장비 탭 바로가기 |

컬렉션 허브의 장비 카드에서는 특수 옵션을 각각 독립된 배지로 표시합니다. 배지에 마우스를 올리거나 키보드·터치로 포커스하면 충돌 피해, 스킬 쿨다운, 피해 흡혈, 질량, 벽 반사 속도, 충돌 각충격 중 실제로 강화되는 효과를 작은 툴팁으로 설명합니다.

### 11.1 관리 액션 결과

강화, 합성, 판매, 보관함 상자 개봉은 장비 또는 재화 보유량을 확정적으로 바꾸는 관심 높은 액션입니다. 각 액션이 끝나면 공통 확인형 팝업으로 결과를 보여 주며, 컬렉션 허브는 닫지 않습니다.

- 강화: 성공/실패와 강화 단계 변화, 소모한 파편을 표시하고 실패 직후 강화석 복구를 선택할 수 있습니다.
- 합성: 자동 추천으로 소모한 장비 3개, 소모한 파편, 새로 생성된 장비를 표시합니다.
- 판매: 처리한 장비, 확정 파편, 확률로 획득한 강화석을 표시합니다.
- 상자 개봉: 획득한 파편 또는 장비의 등급·설명·스탯을 표시합니다.
- 정보량이 작고 즉시 지나가도 되는 상태 알림만 토스트를 사용합니다.

## 15. 관리 자동화

- 새 장비를 얻으면 현재 선택 캐릭터의 장착 가능 레벨을 먼저 확인합니다.
- 빈 슬롯은 즉시 자동으로 채웁니다. 장신구는 `accessory1`, `accessory2` 순서로 빈 슬롯을 찾습니다.
- 슬롯이 찼다면 새 장비가 현재 장비의 모든 기본 스탯과 특수 효과를 같거나 높게 유지하면서 하나 이상 높을 때만 자동 교체합니다.
- 서로 다른 스탯이나 특수 효과 사이에 장단점이 생기면 자동 교체하지 않습니다. 이 경우 두 장비를 모두 보관해 사용자가 빌드를 선택할 수 있게 합니다.
- 강화 단계의 스탯 배율도 비교에 포함합니다. 따라서 강화된 기존 장비가 실제로 더 강하면 새 장비의 등급만 높다는 이유로 교체하지 않습니다.
