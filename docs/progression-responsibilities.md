# 성장 책임 기준

> 상태: 업적 전투 보정, 숫자 도전 단계, 가변 스탯 배분 보정은 제거 완료. 구간별 상대 레벨 생성은 다음 단계에서 정렬한다.
> 관련 문서: [experience-system.md](experience-system.md), [character-mastery-system.md](character-mastery-system.md), [collection-achievements-system.md](collection-achievements-system.md), [reward-balance.md](reward-balance.md)

## 1. 성장 축

| 축 | 소유자 | 역할 | 직접 전투 수치 |
| --- | --- | --- | --- |
| 캐릭터 레벨 | 해당 캐릭터 | 기본 수치와 대표 행동의 고유 강화 | 가능 |
| 환생 보상 | 해당 캐릭터 | Lv.10 반복 보상, 영구 기초 수치와 최대 3장 액션·패시브 빌드 선택 | 가능 |
| 스탯 배분 | 플레이어 | 모든 캐릭터 공통 100포인트 빌드 선택 | 가능 |
| 장비 | 플레이어 | 고정 추가 수치와 특수 옵션으로 빌드 보완 | 가능 |
| 마스터리 | 다른 캐릭터 | 캐릭터 수집을 유도하는 교차 지원 효과 | 가능, 단 자기 자신에게는 미적용 |
| 업적 | 계정 | 수집과 경제 보상 | 불가 |

전투 수치의 기본 계산 순서는 다음과 같다.

```text
캐릭터 기본 수치 + 레벨 보상 + 영구 환생 기초 수치
  -> 스탯 배분 퍼센트 배율
  -> 장비 고정 추가 수치
  -> 마스터리의 교차 지원 퍼센트
```

## 2. 레벨

캐릭터 레벨은 Lv.1부터 Lv.10까지다. Lv.2~10의 모든 보상 행은 해당 캐릭터의 기본 수치를 강화하고, Lv.3·6·9 행은 대표 행동 tier도 함께 올린다. 진행 스냅샷의 `abilityTier`를 해당 Ability가 소비하는 방식으로 확장한다.

레벨은 그 캐릭터 자신에게만 적용된다. 다른 캐릭터를 수집해서 얻는 효과는 레벨이 아니라 마스터리의 책임이다.

## 2.1 환생 보상

환생은 새 Lv.10 도달 뒤 결과 확인으로 준비 화면에 복귀했을 때 한 번 안내하고, 해당 캐릭터의 도감 상세 환생 탭에서 후보 3장 중 하나를 고른 뒤 자신의 XP만 Lv.1로 되돌리는 반복 성장 축이다. 후보는 탭 안에서 표시하기 전에 캐릭터 프로필에 저장하고, 실제 선택 전에는 절대 소진하지 않는다. 영구 기초 수치 보상은 해당 캐릭터에 허용된 HP·공격·속도·방어력 두 항목만 즉시 누적하고 장착하지 않는다. 액션 카드는 다른 볼의 Ability 인스턴스를 primary와 분리해 추가하며, 패시브 카드는 장착한 능력의 공통 modifier를 제공한다. 보유 액션·패시브 카드 중 최대 3장만 장착하며 중복은 rank로 합친다. rank는 기존 카드 효과만 강화한다. 누적 환생 횟수의 불꽃은 이 성장 축의 전투 수치 소유자가 아니라 Canvas 렌더러의 시각 전용 출력이며, 공 외곽 발사 후 부력 상승이라는 세부 계약은 게임 규칙과 밸런스 문서가 소유한다.

## 3. 스탯과 장비

스탯 배분은 공통 100포인트를 사용한다. 업적이나 도전 보상으로 총 포인트, 배분 유연성, 스탯별 한도를 추가하지 않는다.

장비는 스탯 배율 뒤에 더하는 고정 추가 수치다. 등급 범위는 공통 가치 포인트이고, 실제 HP·공격·방어·속도 수치는 `equipment.statValueRatios`가 환산한다.

## 4. 마스터리와 도전

마스터리는 캐릭터 A를 성장시키면 A를 제외한 다른 캐릭터가 받는 지원 효과다. 이것이 캐릭터별 수집의 동기다. 스탯형 마스터리는 장비 고정 수치까지 확정된 뒤 한 번만 퍼센트로 적용한다. 충돌·물리·쿨다운·주기형 효과는 각 최종 런타임 계산기에 별도 modifier로 전달한다. 동일한 키에 대한 숙련도 효과는 상한 없이 단순 가산된다.

도전은 마스터리 승급을 위한 세 관문이다. 캐릭터 최대 레벨 10을 세 구간으로 나눈다.

| 관문 | 기준 레벨 | 상대 규칙 |
| --- | ---: | --- |
| 브론즈 | Lv.3 | 상대는 Lv.3까지의 레벨 보상과 공통 100포인트 배분을 사용 |
| 실버 | Lv.6 | 상대는 Lv.6까지의 레벨 보상과 공통 100포인트 배분을 사용 |
| 골드 | Lv.10 | 상대는 Lv.10까지의 레벨 보상과 공통 100포인트 배분을 사용 |

무한 도전 단계, 단계별 AI 추가 포인트, HP·공격·방어 별도 배율, AI 균형 배분 가중치는 사용하지 않는다. AI 액션은 도전 관문 전투에서 활성화한다.

장비는 플레이어의 수집과 준비에 따른 우위로 남긴다. AI에 랜덤 장비를 부여하거나 플레이어 장비를 복사하지 않는다. 골드 관문의 난이도가 부족하거나 과하면 레벨 보상과 장비 가치 기준을 시뮬레이션으로 함께 조정한다.

## 5. 업적

업적은 모든 캐릭터가 달성할 수 있는 계정 목표이므로 직접 전투 스탯을 제공하지 않는다. 보상은 아래 범주만 사용한다.

- 파편
- 등급 상자
- 확정 장비
- 기능 또는 콘텐츠 해금
- 칭호와 수집 표시

확정 장비는 등급만 보장하는 무작위 장비가 아니다. 업적 정의가 이름, 슬롯, 고정 스탯, 특수 옵션을 함께 명시하며, 고정 스탯은 보조값으로 제한하고 보장 특수 옵션이 빌드의 중심이 된다. 인벤토리가 가득 차면 같은 장비 인스턴트를 보존하는 무료 보관 상자로 지급한다.

기존 `extraStatPoints`, `balanceTolerance`, `perStatCapBonus` 기반 업적 보상은 제거 대상이다.

## 6. 리팩터링 순서

1. 도전 상태를 브론즈·실버·골드 관문으로 교체한다.
2. AI 전투 스펙을 해당 관문의 레벨 보상과 공통 100포인트로 생성한다.
3. 업적 정의와 보상 처리에서 직접 전투 스탯 보상을 제거한다.
4. 마스터리 효과는 교차 지원 효과만 유지한다.
5. 각 관문의 표준 전투 시뮬레이션으로 장비 보유 여부별 승률을 검증한다.
