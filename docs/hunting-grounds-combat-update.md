# 사냥터 전투 컨셉 업데이트

작성일: 2026-07-04

## 핵심 결정

사냥터는 로스터 캐릭터 1대1 반복 모드가 아니라, 플레이어 캐릭터 1명과 다수 몹이 싸우는 로그라이크 전투 모드로 정의한다. 일반 전투의 중간 보스는 상위 등급 몬스터를 강화한 형태이고, 로스터 캐릭터는 챔피언 난입과 최종 보스에서만 보스 역할로 등장한다.

## 전투 구성

- 플레이어: 사냥터에 입장한 캐릭터 1명, `teamId: "hunting-player"`.
- 일반 적: 사냥터 전용 몹 여러 명, `teamId: "hunting-enemy"`.
- 중간 보스: 일반 전투에서만 누적 확률로 등장하는 `rare`·`unique`·`epic` 몬스터 강화형 적. 최초 5%에서 중간 보스를 만나지 못한 일반 전투마다 다음 일반 전투 확률이 5%p 올라가며, 최대 30%다. 조우하면 5%로 초기화한다. 행동·속도·몬스터 태그·기본 전장 루팅은 원본 몬스터를 유지하고, 반경 1.5배·질량 2.25배·HP 2배·공격력/방어력 1.5배를 적용한다.
- 정예 몹 습격: `elite_mob` 이벤트에서만 등장하는 검증된 3·4·5마리 몬스터 조합. 커밋된 생성 결과를 같은 확률로 고르고, 실제 층의 `elite` HP·공격력·방어력 보정을 각 몬스터에 적용한다. 일반 몬스터 태그와 전장 루팅은 유지하며, 챔피언 침입과 별개다.
- 챔피언/최종 보스: 기존 로스터 캐릭터를 사냥터 보스로 변환한 스펙. 챔피언은 `champion_intrusion` 이벤트, 최종 보스는 100층에서만 등장하며 일반 몬스터 태그와 일반 몬스터 전장 루팅은 받지 않는다.
- 같은 팀끼리는 `Simulation.isHostile()` 기준으로 피해 대상에서 제외한다.

## 몹 타입 MVP

MVP에서는 몹 타입을 2종으로 둔다.

| 타입 | 코드 | 전투 성격 | 임시 능력 매핑 |
| --- | --- | --- | --- |
| 근접 몹 | `melee` | 앞으로 파고들어 충돌/돌진 압박 | `dash` |
| 원거리 몹 | `ranged` | 후방에서 투사체 견제 | `archer` |

현재 MVP는 기존 능력 클래스를 재사용해 전투 감각을 빠르게 검증한다. 이후 사냥터 전용 능력/외형이 필요하면 `huntingMonsters.js`의 몹 스펙을 유지한 채 ability만 교체한다.

## 층별 몹 수 확률

```text
모든 층: 플레이어 1명 vs 몹 2~10명
저층: 2~3명 편성의 비중이 높음
고층: 7~10명 편성의 비중이 높음
```

일반 전투가 중간 보스 조우로 판정되면 몬스터 기반 중간 보스 1명과 몹 일부를 함께 배치한다. 챔피언 난입은 로스터 기반 적의 `enemyType`을 `champion`으로 올려 HP·공격력·방어력 보정을 더 크게 적용한다. 챔피언 본체의 일반 몬스터 전장 보상 배율은 없지만, 모든 보스 역할은 전장에서 회수 가능한 강화석을 남긴다.

## 보스 강화석

일반 전투 중간 보스, 챔피언 난입, 최종 보스는 처치 시 강화석을 각각 물리 아이템으로 떨어뜨린다. 개별 강화석의 값은 항상 1이며, 보스 역할에 따른 차이는 없다. 드롭 수는 1층에서 1~3개, 층수에 따라 선형으로 늘어나 100층에서 4~12개다. 강화석은 파편과 같은 수집 유예·자석·HUD·귀환/클리어 정산·패배 보존/손실 흐름을 사용한다. 몬스터 기반 중간 보스만 기존 일반 몬스터 파편과 추가 전리품도 함께 남긴다.

## 맵 크기와 카메라

사냥터는 실제 물리 경기장 크기를 적 수에 맞춰 키운다. 기본 토너먼트 맵은 960×960을 유지하고, 동굴의 2마리 전투는 1000×1000에서 시작한다. 전장 면적 배율은 `1 + (적 수 - 2) / 8`로 계산해 적 수에 비례하되 최대 2배까지만 커진다. `ArenaCamera`는 현재 시뮬레이션의 전체 맵이 캔버스 안에 들어오도록 fit-to-map 배율을 계산한다.

- 일반 1대1: 물리 맵 960×960, 960 캔버스에서 scale 1.0
- 사냥터 1대2: 물리 맵 1000×1000
- 사냥터 1대6: 물리 맵 약 1225×1225, 면적 1.5배
- 사냥터 1대10: 물리 맵 약 1414×1414, 면적 2배
- `cameraZoom`은 전체 맵 fit 배율 위에 곱해지는 선택적 배율이다.
- 충돌, 이동, 벽 반사는 현재 `Simulation.width/height`를 기준으로 처리한다.

이 방식은 다수 전투의 실제 이동 공간을 넓히면서도, 화면에는 항상 전체 맵 경계를 보여주기 위한 MVP 선택이다.
