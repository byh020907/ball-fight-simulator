# 사냥터 전투 컨셉 업데이트

작성일: 2026-07-04

## 핵심 결정

사냥터는 로스터 캐릭터 1대1 반복 모드가 아니라, 플레이어 캐릭터 1명과 다수 몹이 싸우는 로그라이크 전투 모드로 정의한다. 기본 로스터 캐릭터는 일반 적으로 계속 등장하지 않고, 특정 층이나 챔피언 난입 이벤트에서 중간 보스 역할로 등장한다.

## 전투 구성

- 플레이어: 사냥터에 입장한 캐릭터 1명, `teamId: "hunting-player"`.
- 일반 적: 사냥터 전용 몹 여러 명, `teamId: "hunting-enemy"`.
- 중간 보스: 기존 로스터 캐릭터를 사냥터 적으로 변환한 스펙. 3층마다 또는 `champion_intrusion` 이벤트에서 등장.
- 같은 팀끼리는 `Simulation.isHostile()` 기준으로 피해 대상에서 제외한다.

## 몹 타입 MVP

MVP에서는 몹 타입을 2종으로 둔다.

| 타입 | 코드 | 전투 성격 | 임시 능력 매핑 |
| --- | --- | --- | --- |
| 근접 몹 | `melee` | 앞으로 파고들어 충돌/돌진 압박 | `dash` |
| 원거리 몹 | `ranged` | 후방에서 투사체 견제 | `archer` |

현재 MVP는 기존 능력 클래스를 재사용해 전투 감각을 빠르게 검증한다. 이후 사냥터 전용 능력/외형이 필요하면 `huntingMonsters.js`의 몹 스펙을 유지한 채 ability만 교체한다.

## 층별 기본 몹 수

```text
1~2층: 플레이어 1명 vs 몹 2명
3~4층: 플레이어 1명 vs 몹 3명
5층:   플레이어 1명 vs 몹 4명
```

3층 단위 중간 보스 조우에서는 중간 보스 1명과 몹 일부를 함께 배치한다. 챔피언 난입은 중간 보스의 `enemyType`을 `champion`으로 올리고 보상 배율을 높인다.

## 카메라/시야

사냥터는 실제 물리 경기장 크기를 키우지 않는다. 대신 `ArenaCamera`가 렌더링 배율을 낮춰 같은 물리 공간을 더 넓게 보는 느낌을 준다.

- 일반 1대1: `zoom = 1.0`
- 사냥터 다수전: `cameraZoom = 0.78`
- 충돌, 이동, 벽 반사는 기존 `Simulation.width/height`를 그대로 사용

이 방식은 기존 밸런스 계산과 물리 규칙을 유지하면서 다수 전투의 시야 혼잡만 줄이기 위한 MVP 선택이다.
