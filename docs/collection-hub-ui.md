# 컬렉션 허브 UI

> 상태: 도감/숙련도/업적 구현, 보관함 최소 표시 구현
> 관련 문서: [`hunting-grounds-system.md`](hunting-grounds-system.md), [`experience-system.md`](experience-system.md), [`collection-achievements-system.md`](collection-achievements-system.md), [`character-mastery-system.md`](character-mastery-system.md), [`meta-progression-system.md`](meta-progression-system.md), [`design.md`](design.md)

## 1. 목적

도감, 캐릭터 연계, 업적과 이후 추가될 보관함·장비·기록·도전 콘텐츠를 하나의 팝업에서 확인하는 **컬렉션 허브**를 제공합니다.

- 수집 요소마다 별도 팝업을 만들지 않습니다.
- 탭을 이동해도 팝업은 닫히지 않습니다.
- 현재 탭과 필터 상태는 팝업을 닫기 전까지 유지합니다.
- 수집 정보 확인만 담당하며 전투 중 능력이나 장비를 변경하지 않습니다.
- 향후 장비 탭이 추가되면 장착 기능은 장비 탭만 소유합니다.

## 2. 진입점

상태 UI 또는 도움말 버튼 옆에 컬렉션 버튼을 추가합니다.

- 아이콘: 기존 아이콘 라이브러리가 없다면 단순한 책 모양 문자 또는 텍스트 `컬렉션`
- 툴팁: `컬렉션`
- 준비, 전투, 결과 상태에서 모두 열 수 있습니다.
- 전투 중 열면 시뮬레이션은 계속 진행합니다.
- 전투 화면을 가리는 것이 불편할 수 있으므로 전투 중 첫 진입 시 별도 경고를 띄우지 않습니다.
- 컬렉션을 열었다고 게임 시간을 멈추지 않습니다.

모바일에서는 상단 상태 UI의 고정 폭을 늘리지 않도록 아이콘 버튼 하나만 표시합니다.

## 3. 팝업 구조

```text
┌────────────────────────────────────────────┐
│ 컬렉션                             [닫기] │
│ 수집 4/9 · 업적 3/9 · 숙련도 합계 7       │
├────────────────────────────────────────────┤
│ [도감] [연계] [업적]                       │
├────────────────────────────────────────────┤
│ 검색 / 필터 / 정렬                         │
├────────────────────────────────────────────┤
│                                            │
│              현재 탭 콘텐츠                │
│                                            │
└────────────────────────────────────────────┘
```

레이아웃은 네 영역으로 나눕니다.

1. **고정 헤더**: 제목, 전체 진행 요약, 닫기 버튼
2. **고정 탭 바**: 현재 탭과 탭별 진행 숫자
3. **고정 탭별 도구 바**: 검색, 필터, 정렬
4. **스크롤 콘텐츠**: 카드 그리드 또는 목록
5. **도감 고정 상세 영역**: 선택 캐릭터 정보. 도감 목록 스크롤에 포함하지 않음

팝업 전체가 페이지 스크롤을 만들지 않습니다. 각 탭의 카드 또는 목록 영역만 스크롤합니다.

프레임은 오버레이의 반응형 안전 여백 안에서 가용 너비·높이를 모두 사용한다. 헤더와 탭은 `flex-shrink: 0`으로 보존하고, 콘텐츠·목록·상세는 `min-height: 0`인 grid/flex 자식으로 가용 공간을 나눈다. 따라서 화면 종류별 고정 폭·높이 보정이나 임의의 `calc()` 컨테이너를 만들지 않는다.

## 4. 탭 구성

초기 탭 순서:

```js
export const COLLECTION_HUB_TABS = Object.freeze([
    { id: "roster", label: "도감" },
    { id: "mastery", label: "숙련도" },
    { id: "achievements", label: "업적" },
    { id: "storage", label: "보관함" },
    { id: "equipment", label: "장비" }
]);
```

향후 추가 순서:

```text
도감 · 숙련도 · 업적 · 보관함 · 장비 · 기록 · 도전
```

- 존재하지 않는 기능의 빈 탭을 미리 표시하지 않습니다. 단, 보관함은 사냥터 MVP 기반 저장 구조가 들어오면서 해조각/상자 확인용 최소 탭을 먼저 표시합니다.
- 탭 추가는 템플릿 분기와 탭 정의 등록으로 처리합니다.
- 특정 탭이 없어도 나머지 탭의 인덱스나 동작이 바뀌지 않도록 ID를 사용합니다.
- URL 라우팅은 현재 필요하지 않습니다.

## 5. 공통 상태

`appStore()`에 전용 상태를 추가합니다.

```js
collectionHub: {
    visible: false,
    activeTab: "roster",
    tabs: [],
    summary: {
        playedCharacters: 0,
        rosterSize: 0,
        unlockedLinks: 0,
        totalLinks: 0,
        unlockedAchievements: 0,
        totalAchievements: 0,
        masteryTotal: 0
    },
    filters: {
        roster: "all",
        monsters: "all",
        links: "all",
        achievements: "all"
    },
    searchQueries: {
        roster: "",
        monsters: "",
        links: "",
        achievements: ""
    },
    codexSection: "characters",
    selectedCharacterId: null,
    selectedMonsterId: null,
    selectedMonsterRegionId: null
}
```

필수 액션:

```js
openCollectionHub(tabId = "roster");
closeCollectionHub();
setCollectionTab(tabId);
setCollectionFilter(filterId);
selectCollectionCharacter(characterId);
clearCollectionSearch(tabId);
```

- 잘못된 `tabId`는 첫 번째 등록 탭으로 보정합니다.
- 팝업을 닫을 때 선택 캐릭터 상세 화면은 닫습니다.
- 마지막 탭은 같은 페이지 세션 동안 유지합니다.
- localStorage에는 UI 탭과 검색 상태를 저장하지 않습니다.

## 6. 도감 탭

도감 탭 안에서는 `캐릭터 | 몬스터` 분할 탭으로 두 수집 대상을 전환한다. 두 분할 탭은 검색어, 필터, 정렬, 선택 항목, 목록 스크롤 위치를 각각 유지하고 컬렉션을 닫을 때만 초기화한다. 상세는 별도 전체 화면이나 목록 복귀 버튼 없이 현재 목록 아래의 고정 영역에서 연다. 상세 우측 상단 X는 팝업을 닫지 않고 선택만 해제해, 특히 모바일에서 목록을 다시 넓게 사용할 수 있게 한다.

### 6.1 캐릭터 도감 상단 도구

- 검색: 캐릭터 이름
- 필터: `전체`, `미기록`, `플레이`, `우승`, `마스터`
- 정렬: `로스터 순`, `우승 많은 순`, `최근 플레이`

#### 개발자 세션 진입

캐릭터 검색에 정확히 `디버그`를 입력하면 영구 프로필을 복제한 메모리 전용 개발 세션이 시작되고, 그 세션에만 `개발자` 탭을 추가한다.

- 개발자 탭은 캐릭터 설정과 사냥터 테스트를 목적별 영역으로 나누고, 환생 횟수·적용 및 이벤트 선택·미리보기처럼 서로 의존하는 입력과 실행 버튼을 같은 줄에 배치한다. 캐릭터 Lv.10, 토너먼트 1위 우승, 환생 횟수, 지역·시작 층을 재현하며, 우승 처리는 일반 토너먼트 통계 리포트 경로를 사용해 도감 우승 기록과 사냥터 진입 조건을 함께 갱신한다.
- 고층 사냥터는 입력한 층을 다음 실제 조우 층으로 사용한다. 정상 사냥터의 첫 안내와 이동 규칙은 바꾸지 않는다.
- 개발자 탭을 닫는 동작은 세션을 끝내고 검색어를 비운 뒤 도감 탭으로 돌아간다.
- 모바일 폭 또는 기기 방향이 바뀌면 현재 선택 탭이 가로 탭 영역 안에 다시 보이도록 스크롤 위치를 동기화한다.
- 이벤트 미리보기는 선택한 지역·층에서 포탈, 상인, 축복, 함정, 상자방, 휴식지, 제단, 챔피언, 정예 몹의 실제 이벤트 해석과 화면을 즉시 연다. 정예 몹을 선택하면 실제 수동 조합 목록에서 특정 조합을 고를 수 있으며, 시작 층은 해당 조합의 최소 등장 층보다 낮지 않게 맞춘다. 선택한 조합은 일반 이벤트 payload와 전투 준비 경로를 그대로 사용한다. 랜덤 조우·전투 보상·영구 프로필 저장은 별도 목업으로 우회하지 않으며, 메모리 전용 개발 세션에서만 사용한다.
- 전투 조우 테스트는 일반 몬스터, 중간 보스, 챔피언, 선택한 정예 조합, 최종 보스를 무작위 층 판정 없이 실제 전투 준비 화면으로 연다. 정예 조합은 최소 등장 층을, 최종 보스는 실제 최종 층을 자동 적용한다.
- `Elementalist VFX` 영역은 젖음 상태, 단일 5원소, 복합 주문 10종을 선택해 실제 전투 이펙트와 오브 렌더러로 재생한다. 젖음 항목은 마른 상태로 시작하며 `젖음 발동`을 누를 때 실제 전투와 같은 공용 상태 적용 함수를 호출한다. 반복 발동은 효과 엔티티를 늘리지 않고 최대 3개의 독립 2.5초 중첩을 관리하며, 활성 중첩 수에 따라 물막과 물방울 밀도를 높인다. 주문 항목은 `마른 대상 재생`과 `젖은 대상 재생`을 나란히 제공한다. 젖은 대상은 공용 젖음 VFX를 유지한 채 선택한 채널과 반응을 함께 보여 줘 마른 재생과 시각 흐름을 직접 비교할 수 있다. 같은 영역에서 실제 주문·젖음 반응 설정으로 계산한 방어 적용 전 총공격력 배율, 추가 피해 증가율과 속박 시간도 함께 표시한다. 별도 목업 VFX나 독자적인 피해표를 만들지 않으며 탭을 벗어나거나 팝업을 닫으면 애니메이션을 중지한다. 캔버스·버튼·피해 비교 카드는 고정 컨테이너 폭 없이 개발자 카드의 가용 폭과 유동 그리드를 따른다.
- 컬렉션 샘플은 선택 캐릭터를 Lv.10으로 맞춘 뒤 실제 장비·상자 생성·장착 API로 장비 6개, 합성 가능한 일반 장비 3개, 장착 슬롯, 강화석, 파편, 등급별 상자를 메모리 전용 개발 세션에 채운다. 장비/보관함/상점/합성의 비어 있지 않은 상태를 검증할 때만 사용한다.
- 이 진입 방법은 개발용 내부 동작이며 게임 패치노트에는 안내하지 않는다.

기본값:

```text
필터 전체 · 정렬 로스터 순
```

### 6.2 캐릭터 도감 카드

카드에 표시:

- 캐릭터 얼굴
- 이름
- 숙련도 `Lv.0~3`
- 경험치 레벨과 누적 XP (`Lv.N · NXP`)
- 토너먼트 우승 횟수
- 최고 등수
- 연계 효과 해금 여부

```text
┌────────────────────┐
│  (캐릭터 얼굴)      │
│ Archer Ball        │
│ 숙련도 Lv.2        │
│ XP Lv.3 · 235XP    │
│ 우승 6 · 최고 1위  │
│ 연계 효과 해금     │
└────────────────────┘
```

- 미기록 캐릭터도 얼굴과 이름을 공개합니다.
- 미기록 카드는 숨기거나 실루엣 처리하지 않고 채도를 낮춥니다.
- 색상만으로 숙련도를 구분하지 않고 `Lv.N` 텍스트를 함께 표시합니다.
- 카드 전체를 버튼으로 구현해 키보드 포커스가 가능해야 합니다.

### 6.3 캐릭터 도감 상세

카드를 선택하면 도감 탭 하단의 고정 상세 영역에 정보를 표시합니다.

- 별도 중첩 팝업을 만들지 않습니다.
- 상세 영역은 카드 목록의 스크롤에 포함하지 않습니다.
- 목록을 스크롤해도 상세 영역과 상단 검색·필터·정렬 도구는 현재 위치를 유지합니다.
- 도감 목록이 같은 화면에 계속 표시되므로 별도의 목록 복귀 버튼은 두지 않습니다.
- 상세 우측 상단 X는 선택한 캐릭터만 해제하고, 현재 탭·검색·필터·목록 스크롤은 유지합니다.
- 상세 내부는 `상세 정보`, `레벨 보상`, 조건부 `환생` 탭으로 전환합니다.
- `상세 정보` 탭은 토너먼트 완료·우승, 매치 승리, 최고 등수, 누적 피해, 역전승, 경험치 진행도와 숙련도 단계를 표시합니다.
- `레벨 보상` 탭은 Lv.2~Lv.10 전체 보상을 레벨 순서로 표시하고, 현재 레벨까지는 `획득`, 이후는 `예정`으로 구분합니다.
- 다른 캐릭터를 선택하면 상세 탭은 `상세 정보`로 초기화합니다.
- 캐릭터를 선택한 상태에서는 목록과 상세를 같은 가용 높이의 grid 행으로 나누고, 둘 중 내용이 긴 쪽만 내부 스크롤한다. 환생 후보·보유 카드가 많아도 상세가 목록을 밀어내거나 프레임 밖으로 넘치지 않는다.

#### 환생 탭

- Lv.10에 도달했거나 환생 이력이 있는 캐릭터에만 표시한다.
- 누적 환생 횟수, 해당 횟수에서 계산한 불꽃 단계, 누적 기초 수치(HP·공격·속도·방어력), 활성 장착 카드 수(`0~3`)와 보유 카드 단계를 표시한다. 카드에는 `단계 n/4`, 현재 효과, 다음 원 캐릭터 Lv. 성장 해금을 표시하고 4단계는 `MAX`와 최종 효과를 표시한다. 불꽃 단계는 별도 장비·스탯 설정이 아니라 전투와 미리보기에 공통 적용되는 시각 전용 환생 표현이다.
- 새로 Lv.10에 도달한 캐릭터는 전투 결과를 `확인`해 준비 화면으로 복귀한 뒤 한 번만 환생 가능 안내를 받는다. 안내의 주 동작은 해당 캐릭터의 이 탭을 바로 연다. 이미 Lv.10인 캐릭터로 준비 화면에 들어오는 일만으로는 안내를 반복하지 않는다.
- Lv.10일 때 `환생 보상 선택` 버튼은 후보 3장을 프로필에 저장한 뒤 현재 탭 안에 표시한다. 후보 카드는 이름·효과·분류와 각 `선택` 동작을 가지며, 선택 전 탭을 닫거나 새로고침한 뒤 다시 열어도 같은 후보를 표시한다. 대기 중 후보가 있으면 버튼 문구를 `진행 중인 보상 선택`으로 바꾼다.
- 환생 완료 시 해당 캐릭터 XP만 Lv.1로 초기화한다. 영구 기초 수치 보상은 즉시 누적되며 장착·단계 대상으로 만들지 않는다. 액션·패시브 카드 획득, 중복 단계 상승, 장착·해제는 준비 화면에서만 허용한다. 최대 단계 카드는 후보에서 빠지고 기초 수치 카드가 빈 자리를 채운다.
- 액션 카드는 전투 fighter card에 primary와 분리된 쿨다운 상태로 나타난다. 패시브는 별도 능력 행을 추가하지 않고 장착한 능력의 공통 modifier로만 적용한다.

다음 숙련도까지:

```text
숙련도 Lv.2
다음 단계까지 우승 9회
[██████░░░░] 6 / 15
```

최고 단계에서는 `마스터 완료`를 표시하고 다음 목표 숫자를 만들지 않습니다.

### 6.4 몬스터 도감

몬스터 도감은 일반 사냥터 몬스터의 행동 타입마다 카드 하나를 제공한다. 카드와 상세 데이터는 `huntingMonsters.js`의 실제 정의와 사냥터 영구 통계를 변환한 ViewModel에서만 만들며, 템플릿은 행동·스탯·지역 규칙을 직접 계산하지 않는다.

- 필터: `전체`, `조우`, `미조우`, `common`, `rare`, `unique`, `epic`
- 정렬: `도감 순`, `처치 많은 순`
- 미조우 카드: 회색 실루엣과 `미조우`, `첫 조우 후 상세 공개`만 표시한다.
- 조우 카드: 얼굴, 이름, 희귀도, 타입 전체 처치 수, 조우한 지역 수를 표시한다.
- 카드를 선택하면 같은 탭 하단의 고정 상세에서 행동 설명, 정확한 기본 `HP · 공격 · 방어 · 속도`, 타입 전체 처치 수, 선택 지역 처치 수, 첫·마지막 조우 층을 표시한다.
- 상세의 지역 드롭다운에는 한 번이라도 진입한 지역만 표시한다. 선택 지역에서 아직 해당 몬스터를 만나지 않았다면 `미조우` 상태로 외형·행동·기본 스탯을 가리고, 지역 이름과 잠금 안내만 표시한다.
- 타입 전체 처치 기록과 지역별 처치 기록은 함께 보여 주되, 몬스터 처치 업적은 타입 전체 기록만 사용한다.
- 모바일에서도 카드 목록과 고정 상세는 내부 스크롤로 분리하며, 별도의 목록 복귀 버튼을 만들지 않는다.
- 상세 우측 상단 X는 선택한 몬스터와 지역 선택만 해제하고, 현재 탭·검색·필터·목록 스크롤은 유지한다.

## 7. 연계 탭

### 7.1 상단 요약

```text
연계 수집 4 / 9
현재 캐릭터에 3개 자동 적용 중
```

현재 캐릭터 자신의 연계 효과는 해금돼 있어도 적용 개수에서 제외합니다.

### 7.2 필터

- `전체`
- `해금`
- `미해금`
- `현재 적용`

장착 또는 효과 끄기 필터는 만들지 않습니다.

### 7.3 연계 카드

카드에 표시:

- 원본 캐릭터 얼굴과 이름
- 연계 효과 이름
- 구체적인 효과 수치
- `해금`, `미해금`, `현재 캐릭터 효과`, `적용 중` 상태
- 해금 조건: `{캐릭터명}으로 토너먼트 우승`

상태 예시:

```text
해금 · 적용 중
해금 · 현재 캐릭터 효과라 제외
미해금 · Archer Ball로 우승
```

- 해금 효과는 자동 적용임을 탭 상단에서 한 번만 안내합니다.
- 카드마다 장황한 자동 적용 설명을 반복하지 않습니다.
- 버튼, 체크박스, 토글, 슬롯 UI를 만들지 않습니다.
- 장비 시스템이 추가되어도 연계 카드에는 장착 버튼을 추가하지 않습니다.

## 8. 업적 탭

### 8.1 상단 도구

- 필터: `전체`, `달성`, `미달성`
- 등급: `전체`, `Bronze`, `Silver`, `Gold`
- 정렬: `기본 순`, `달성 최근 순`, `진행도 높은 순`

### 8.2 업적 항목

업적은 카드 그리드보다 세로 목록을 사용합니다. 조건과 진행도를 한 줄 폭 안에서 읽기 쉽기 때문입니다.

항목에 표시:

- 업적 이름
- 등급 배지
- 조건 설명
- 현재 진행도
- 보상
- 달성 시각 또는 미달성 상태

수치형 업적은 정의의 `getProgress()`가 제공한 현재값과 목표값을 `진행: 현재 / 목표`로 조건·보상과 같은 정보 영역에 표시합니다. 희귀도 처치처럼 태그로 확장되는 조건도 별도 UI 분기 없이 이 형식을 재사용합니다.

```text
[SILVER] 반격 전문가
카운터 성공 10회
[███████░░░] 7 / 10
보상: 빌드 유연성 +5
```

- 진행도를 계산할 수 없는 일회성 조건은 `달성 전`으로 표시합니다.
- 달성 후에도 조건과 보상을 숨기지 않습니다.
- 수령 버튼은 만들지 않습니다. 보상은 업적 해금 시 자동 지급합니다.
- `rewardClaimed`는 중복 방지를 위한 내부 상태일 뿐 사용자 조작 UI가 아닙니다.

## 9. 미래 탭

### 장비

- 장비만 슬롯, 장착, 해제, 비교 UI를 가집니다.
- 연계 탭과 시각적으로 구분합니다.
- 장비 미구현 상태에서는 탭 자체를 등록하지 않습니다.

### 파편 상점

- 장비 탭 우측 상단의 `상점` 팝업은 컬렉션을 닫지 않고, 상자·리롤·소모품을 한 목록에서 제공한다.
- HP 물약은 별도 탭 없이 소모품 행 `보유 n/10 · 구매 100`으로 표시한다. 바로 아래에는 영구 성장 행 `원정 물약 사용 한도 n/5 · +1 확장`을 둔다.
- 구매와 확장은 성공 즉시 해당 행의 보유 수량·파편·가격을 갱신하고, 최대 보유/최대 한도에서는 버튼을 비활성화한다.
- 이후 경험치 물약 등은 같은 소모품 목록에 추가한다. 상점은 하위 팝업이므로 구매 결과가 닫혀도 컬렉션 허브와 장비 탭 상태는 유지한다.

### 보관함

- 사냥터에서 얻은 상자와 열쇠 조각을 확인합니다.
- 상자 개봉과 설계도 해금은 보관함 탭이 소유합니다.
- 보관함은 전투 중 능력 변경을 하지 않고, 다음 사냥터/토너먼트 시작 시 적용 가능한 해금 상태만 보여줍니다.

### 기록

- 최근 토너먼트 목록, 캐릭터별 승률, 최고 도전 단계
- 대량 기록이 필요하면 localStorage가 아니라 IndexedDB 전환을 검토합니다.

### 도전

- 최고 해금 단계
- 단계별 최초 클리어
- 캐릭터별 최고 클리어 단계
- 실제 난이도 선택은 스탯 설정 팝업에서 유지하고 컬렉션 허브에서는 기록만 봅니다.

## 10. 반응형 규칙

### 데스크톱

- 너비: `min(920px, calc(100vw - 40px))`
- 높이: `min(760px, calc(100vh - 40px))`
- 도감과 연계 카드: 3열, 충분한 폭이면 4열
- 상세 화면: 얼굴/요약과 기록을 2열 grid로 배치 가능

### 모바일

- 팝업은 화면 가장자리에서 최소 8px 안전 여백을 둡니다.
- 너비는 가용 영역 100%를 사용합니다.
- 높이는 `calc(100dvh - 16px)` 이내로 제한합니다.
- 헤더와 탭은 고정하고 콘텐츠만 스크롤합니다.
- 카드 grid는 기본 2열, 360px 미만에서는 1열을 허용합니다.
- 탭 수가 화면 너비를 넘으면 탭 바만 가로 스크롤합니다.
- 탭 글자 크기는 뷰포트 비례로 커지지 않게 `rem`과 `em`을 사용합니다.
- 최소 터치 높이는 40px입니다.
- 브라우저 주소창 변화에 대응하기 위해 `100vh`보다 `100dvh`를 우선합니다.
- 장비 탭은 독립 컴포넌트 호스트가 가용 높이를 전달하고, 장착 슬롯·보유 장비 목록은 각각 유동 grid와 내부 스크롤을 사용한다. 장비 개수나 모바일 폭을 기준으로 프레임 높이를 따로 고정하지 않는다.

## 11. 시각 규칙

- 기존 플랫 디자인을 유지합니다.
- 그라데이션, 그림자, 블러, 글로우를 사용하지 않습니다.
- 팝업 배경은 밝은 단색, 외곽선은 진한 단색을 사용합니다.
- 탭 활성 상태는 밑줄, 배경색, 굵기 중 두 가지 이상으로 구분합니다.
- Bronze, Silver, Gold는 채도가 다른 단색 배지로 표현하고 광택 금속 효과를 넣지 않습니다.
- 카드 모서리는 기존 시스템과 같은 8px 이하를 사용합니다.
- 카드 안에 다시 카드 형태의 프레임을 중첩하지 않습니다.
- 진행도 바는 단색 채움과 숫자를 함께 표시합니다.

## 12. 접근성 및 입력

- 팝업 컨테이너: `role="dialog"`, `aria-modal="true"`
- 탭 바: `role="tablist"`
- 탭: `role="tab"`, `aria-selected`, `aria-controls`
- 탭 콘텐츠: `role="tabpanel"`
- `Escape`: 팝업 닫기
- 보관함에서 연 상자 결과처럼 하위 모달이 보이면, 각 모달은 자기 backdrop 클릭과 포커스 안의 `Escape`만 처리합니다. 따라서 결과를 닫아도 컬렉션 허브는 유지됩니다.
- 좌우 방향키: 탭 이동
- 팝업이 열리면 닫기 버튼 또는 활성 탭으로 포커스 이동
- 닫으면 컬렉션 진입 버튼으로 포커스 복귀
- 배경 콘텐츠는 포커스할 수 없게 처리

전투 캔버스의 pointer 이벤트가 팝업 클릭을 받지 않도록 overlay가 이벤트를 차단해야 합니다.

## 13. Alpine 구조

컬렉션 허브는 `PopupService`를 사용하지 않습니다.

이유:

- 현재 `PopupService`는 `bodyHtml`과 `x-html`을 사용합니다.
- 탭, 검색, 필터, 카드 선택처럼 반응형 상태가 많은 UI와 맞지 않습니다.
- 개발 규칙은 문자열 HTML 조립 대신 Alpine 템플릿을 요구합니다.

`index.html`에 전용 템플릿을 둡니다.

```html
<div
    class="collection-hub-overlay"
    x-show="collectionHub.visible"
    role="dialog"
    aria-modal="true"
>
    <section class="collection-hub">
        <header>...</header>
        <nav role="tablist">...</nav>
        <div class="collection-hub-content">
            <section x-show="collectionHub.activeTab === 'roster'">...</section>
            <section x-show="collectionHub.activeTab === 'links'">...</section>
            <section x-show="collectionHub.activeTab === 'achievements'">...</section>
        </div>
    </section>
</div>
```

### 중첩 결과 모달

컬렉션 허브 자체는 독립 Alpine 오버레이이지만, 보관함 상자 개봉 결과처럼 짧은 하위 결과는 `PopupService`를 사용합니다. 두 오버레이 모두 자기 루트의 backdrop 클릭과 포커스 안의 `Escape`만 처리합니다. 결과 팝업은 열릴 때 포커스를 소유하므로 상위 컬렉션은 하위 팝업의 존재를 검사할 필요가 없고, 결과를 닫아도 보관함 탭과 현재 상태는 유지됩니다.

데이터 흐름:

```text
player profile + roster + definitions
  -> collection view-model formatter
  -> UIController.renderCollectionHub(viewModel)
  -> appStore.collectionHub
  -> Alpine template
```

- 템플릿이 도메인 규칙을 계산하지 않습니다.
- 숙련도, 진행도, 적용 중 연계 효과는 view-model 생성 단계에서 계산합니다.
- UIController는 전달과 상태 갱신만 담당합니다.

### 하위 화면 컴포넌트

컬렉션 허브의 공통 ViewModel, 현재 탭, 도감 선택 상태는 `collection-hub`이 계속 소유합니다. 같은 화면에서 그 상태를 읽어야 하는 하위 UI는 `uiManager.requireComponent("collectionHub")`의 공개 `state`만 읽고, 도메인 변경은 반드시 `uiManager.invokeGameAction()`으로 요청합니다.

- `collection-equipment-panel`은 장비 탭 렌더링과 장착·강화·분해·판매 액션을 소유한다.
- `collection-fusion-dialog`은 합성 모달의 표시 여부와 재료 선택을 소유한다. 허브를 닫으면 선택을 즉시 비운다.
- `collection-shop-panel`은 상점 모달의 표시 여부, 1초 갱신 시계, 상자 교체 애니메이션을 소유한다. 구매·리롤 성공 때만 애니메이션을 재생한다.
- 허브는 닫힐 때 두 하위 모달을 `uiManager`로 닫아, 이전 화면의 모달·선택 상태가 다음 진입에 남지 않게 한다.

## 15. 빈 상태와 오류 상태

- 프로필 로드 실패: `기록을 불러오지 못했습니다`와 재시도 버튼
- 검색 결과 없음: `검색 결과가 없습니다`
- 필터 결과 없음: `이 조건에 맞는 항목이 없습니다`
- 업적 정의 없음: `등록된 업적이 없습니다`
- 연계 효과 없음: `아직 해금한 연계 효과가 없습니다`
- 기록 초기 상태: 캐릭터 카드는 유지하고 수치만 `0` 또는 `미기록`

로딩은 localStorage 기반이라 일반적으로 즉시 끝나지만, 향후 서버 저장 전환을 위해 `loading`, `error` 상태를 둡니다.

## 17. 필수 회귀 조건

- 팝업을 열어도 토너먼트와 전투 상태가 초기화되지 않아야 합니다.
- 탭 이동으로 검색과 선택 상태가 다른 탭에 잘못 공유되지 않아야 합니다.
- 카드 수와 업적 수를 하드코딩하지 않아야 합니다.
- 미기록 캐릭터도 도감에 표시되어야 합니다.
- 미조우 몬스터는 실루엣만 표시하고, 조우 후에만 실제 정의를 공개해야 합니다.
- 지역 선택지는 지역 진입 후 바로 열리되, 해당 지역에서 미조우인 몬스터의 상세를 공개해서는 안 됩니다.
- 몬스터 도감의 타입 전체 처치 수와 지역별 처치 수가 서로 섞이지 않아야 하며, 업적은 타입 전체 기록만 사용해야 합니다.
- 연계 탭에 장착, 해제, 슬롯 UI가 없어야 합니다.
- 업적 보상 수령 버튼이 없어야 합니다.
- 모바일에서 페이지 전체 스크롤이 생기지 않아야 합니다.
- 팝업 콘텐츠가 작은 화면에서 잘리지 않고 내부 스크롤되어야 합니다.
- 팝업 뒤 캔버스가 클릭을 받지 않아야 합니다.
- 닫은 뒤 키보드 포커스가 진입 버튼으로 돌아와야 합니다.
- 새 탭 정의를 추가해도 기존 탭 ID와 상태가 깨지지 않아야 합니다.
## 잠긴 히든 캐릭터 표시

- 잠긴 카드는 이름 `???`, 회색, 아이콘 `?`, 상태 `미해금`만 표시한다.
- 상세에는 `사냥터의 정체불명 챔피언을 격파하면 해금됩니다` 힌트만 표시하고 레벨·숙련도·환생 정보는 만들지 않는다.
- 검색은 공개 이름만 사용하므로 실제 이름으로 잠긴 카드를 찾을 수 없다. 숙련도 목록도 원본 이름과 효과를 `???`로 가린다.
- 잠긴 히든 캐릭터가 제공하는 환생 액션 카드도 이름·설명·효과를 후보와 보유 목록에서 숨기고 전투·활성 장착 수에서 제외한다. 저장된 단계와 장착 의도는 휴면 상태로 보존하며 재해금하면 같은 단계의 보유 카드로 돌아온다.
- 개발자 탭은 현재 디버그 세션 안에서 임의 히든 캐릭터를 잠금/해금할 수 있으며 영구 프로필은 오염시키지 않는다.
