# 컬렉션 허브 UI

> 상태: UI 설계, 구현 전
> 관련 문서: [`hunting-grounds-system.md`](hunting-grounds-system.md), [`experience-system.md`](experience-system.md), [`collection-achievements-system.md`](collection-achievements-system.md), [`character-mastery-system.md`](character-mastery-system.md), [`meta-progression-system.md`](meta-progression-system.md), [`design.md`](design.md)

## 1. 목적

도감, 캐릭터 연계, 업적과 이후 추가될 보관함·장비·기록·도전 콘텐츠를 하나의 팝업에서 확인하는 **컬렉션 허브**를 제공합니다.

- 수집 요소마다 별도 팝업을 만들지 않습니다.
- 탭을 이동해도 팝업은 닫히지 않습니다.
- 현재 탭과 필터 상태는 팝업을 닫기 전까지 유지합니다.
- 수집 정보 확인만 담당하며 전투 중 능력이나 장비를 변경하지 않습니다.
- 향후 장비 탭이 추가되면 장착 기능은 장비 탭만 소유합니다.

## 2. 진입점

상태 UI 또는 도움말 버튼 옆에 컬렉션 버튼을 추가합니다.

- 아이콘: 기존 아이콘 라이브러리가 없다면 단순한 책 모양 문자 또는 텍스트 `컬렉션`
- 툴팁: `컬렉션`
- 준비, 전투, 결과 상태에서 모두 열 수 있습니다.
- 전투 중 열면 시뮬레이션은 계속 진행합니다.
- 전투 화면을 가리는 것이 불편할 수 있으므로 전투 중 첫 진입 시 별도 경고를 띄우지 않습니다.
- 컬렉션을 열었다고 게임 시간을 멈추지 않습니다.

모바일에서는 상단 상태 UI의 고정 폭을 늘리지 않도록 아이콘 버튼 하나만 표시합니다.

## 3. 팝업 구조

```text
┌────────────────────────────────────────────┐
│ 컬렉션                             [닫기] │
│ 수집 4/9 · 업적 3/9 · 숙련도 합계 7       │
├────────────────────────────────────────────┤
│ [도감] [연계] [업적]                       │
├────────────────────────────────────────────┤
│ 검색 / 필터 / 정렬                         │
├────────────────────────────────────────────┤
│                                            │
│              현재 탭 콘텐츠                │
│                                            │
└────────────────────────────────────────────┘
```

레이아웃은 네 영역으로 나눕니다.

1. **고정 헤더**: 제목, 전체 진행 요약, 닫기 버튼
2. **고정 탭 바**: 현재 탭과 탭별 진행 숫자
3. **고정 탭별 도구 바**: 검색, 필터, 정렬
4. **스크롤 콘텐츠**: 카드 그리드 또는 목록
5. **도감 고정 상세 영역**: 선택 캐릭터 정보. 도감 목록 스크롤에 포함하지 않음

팝업 전체가 페이지 스크롤을 만들지 않습니다. 각 탭의 카드 또는 목록 영역만 스크롤합니다.

## 4. 탭 구성

초기 탭 순서:

```js
export const COLLECTION_HUB_TABS = Object.freeze([
    { id: "roster", label: "도감" },
    { id: "links", label: "연계" },
    { id: "achievements", label: "업적" }
]);
```

향후 추가 순서:

```text
도감 · 연계 · 업적 · 보관함 · 장비 · 기록 · 도전
```

- 존재하지 않는 기능의 빈 탭을 미리 표시하지 않습니다.
- 탭 추가는 템플릿 분기와 탭 정의 등록으로 처리합니다.
- 특정 탭이 없어도 나머지 탭의 인덱스나 동작이 바뀌지 않도록 ID를 사용합니다.
- URL 라우팅은 현재 필요하지 않습니다.

## 5. 공통 상태

`appStore()`에 전용 상태를 추가합니다.

```js
collectionHub: {
    visible: false,
    activeTab: "roster",
    tabs: [],
    summary: {
        playedCharacters: 0,
        rosterSize: 0,
        unlockedLinks: 0,
        totalLinks: 0,
        unlockedAchievements: 0,
        totalAchievements: 0,
        masteryTotal: 0
    },
    filters: {
        roster: "all",
        links: "all",
        achievements: "all"
    },
    searchQueries: {
        roster: "",
        links: "",
        achievements: ""
    },
    selectedCharacterId: null
}
```

필수 액션:

```js
openCollectionHub(tabId = "roster");
closeCollectionHub();
setCollectionTab(tabId);
setCollectionFilter(filterId);
selectCollectionCharacter(characterId);
clearCollectionSearch(tabId);
```

- 잘못된 `tabId`는 첫 번째 등록 탭으로 보정합니다.
- 팝업을 닫을 때 선택 캐릭터 상세 화면은 닫습니다.
- 마지막 탭은 같은 페이지 세션 동안 유지합니다.
- localStorage에는 UI 탭과 검색 상태를 저장하지 않습니다.

## 6. 도감 탭

### 6.1 상단 도구

- 검색: 캐릭터 이름
- 필터: `전체`, `미기록`, `플레이`, `우승`, `마스터`
- 정렬: `로스터 순`, `우승 많은 순`, `최근 플레이`

기본값:

```text
필터 전체 · 정렬 로스터 순
```

### 6.2 캐릭터 카드

카드에 표시:

- 캐릭터 얼굴
- 이름
- 숙련도 `Lv.0~3`
- 경험치 레벨과 누적 XP (`Lv.N · NXP`)
- 토너먼트 우승 횟수
- 최고 등수
- 연계 효과 해금 여부

```text
┌────────────────────┐
│  (캐릭터 얼굴)      │
│ Archer Ball        │
│ 숙련도 Lv.2        │
│ XP Lv.3 · 235XP    │
│ 우승 6 · 최고 1위  │
│ 연계 효과 해금     │
└────────────────────┘
```

- 미기록 캐릭터도 얼굴과 이름을 공개합니다.
- 미기록 카드는 숨기거나 실루엣 처리하지 않고 채도를 낮춥니다.
- 색상만으로 숙련도를 구분하지 않고 `Lv.N` 텍스트를 함께 표시합니다.
- 카드 전체를 버튼으로 구현해 키보드 포커스가 가능해야 합니다.

### 6.3 캐릭터 상세

카드를 선택하면 도감 탭 하단의 고정 상세 영역에 정보를 표시합니다.

- 별도 중첩 팝업을 만들지 않습니다.
- 상세 영역은 카드 목록의 스크롤에 포함하지 않습니다.
- 목록을 스크롤해도 상세 영역과 상단 검색·필터·정렬 도구는 현재 위치를 유지합니다.
- 상단에 `← 도감` 뒤로가기 버튼을 표시합니다.
- 캐릭터 얼굴, 이름, 설명
- 토너먼트 완료, 토너먼트 우승, 매치 승리
- 최고 등수, 누적 피해, 역전승
- 경험치 레벨, 누적 XP, 다음 레벨 진행 바, 다음 보상
- 숙련도 현재 단계와 다음 단계 진행
- 캐릭터 연계 효과 이름, 효과, 해금 상태

다음 숙련도까지:

```text
숙련도 Lv.2
다음 단계까지 우승 9회
[██████░░░░] 6 / 15
```

최고 단계에서는 `마스터 완료`를 표시하고 다음 목표 숫자를 만들지 않습니다.

## 7. 연계 탭

### 7.1 상단 요약

```text
연계 수집 4 / 9
현재 캐릭터에 3개 자동 적용 중
```

현재 캐릭터 자신의 연계 효과는 해금돼 있어도 적용 개수에서 제외합니다.

### 7.2 필터

- `전체`
- `해금`
- `미해금`
- `현재 적용`

장착 또는 효과 끄기 필터는 만들지 않습니다.

### 7.3 연계 카드

카드에 표시:

- 원본 캐릭터 얼굴과 이름
- 연계 효과 이름
- 구체적인 효과 수치
- `해금`, `미해금`, `현재 캐릭터 효과`, `적용 중` 상태
- 해금 조건: `{캐릭터명}으로 토너먼트 우승`

상태 예시:

```text
해금 · 적용 중
해금 · 현재 캐릭터 효과라 제외
미해금 · Archer Ball로 우승
```

- 해금 효과는 자동 적용임을 탭 상단에서 한 번만 안내합니다.
- 카드마다 장황한 자동 적용 설명을 반복하지 않습니다.
- 버튼, 체크박스, 토글, 슬롯 UI를 만들지 않습니다.
- 장비 시스템이 추가되어도 연계 카드에는 장착 버튼을 추가하지 않습니다.

## 8. 업적 탭

### 8.1 상단 도구

- 필터: `전체`, `달성`, `미달성`
- 등급: `전체`, `Bronze`, `Silver`, `Gold`
- 정렬: `기본 순`, `달성 최근 순`, `진행도 높은 순`

### 8.2 업적 항목

업적은 카드 그리드보다 세로 목록을 사용합니다. 조건과 진행도를 한 줄 폭 안에서 읽기 쉽기 때문입니다.

항목에 표시:

- 업적 이름
- 등급 배지
- 조건 설명
- 현재 진행도
- 보상
- 달성 시각 또는 미달성 상태

```text
[SILVER] 반격 전문가
카운터 성공 10회
[███████░░░] 7 / 10
보상: 빌드 유연성 +5
```

- 진행도를 계산할 수 없는 일회성 조건은 `달성 전`으로 표시합니다.
- 달성 후에도 조건과 보상을 숨기지 않습니다.
- 수령 버튼은 만들지 않습니다. 보상은 업적 해금 시 자동 지급합니다.
- `rewardClaimed`는 중복 방지를 위한 내부 상태일 뿐 사용자 조작 UI가 아닙니다.

## 9. 미래 탭

### 장비

- 장비만 슬롯, 장착, 해제, 비교 UI를 가집니다.
- 연계 탭과 시각적으로 구분합니다.
- 장비 미구현 상태에서는 탭 자체를 등록하지 않습니다.

### 보관함

- 사냥터에서 얻은 상자와 열쇠 조각을 확인합니다.
- 상자 개봉과 설계도 해금은 보관함 탭이 소유합니다.
- 보관함은 전투 중 능력 변경을 하지 않고, 다음 사냥터/토너먼트 시작 시 적용 가능한 해금 상태만 보여줍니다.

### 기록

- 최근 토너먼트 목록, 캐릭터별 승률, 최고 도전 단계
- 대량 기록이 필요하면 localStorage가 아니라 IndexedDB 전환을 검토합니다.

### 도전

- 최고 해금 단계
- 단계별 최초 클리어
- 캐릭터별 최고 클리어 단계
- 실제 난이도 선택은 스탯 설정 팝업에서 유지하고 컬렉션 허브에서는 기록만 봅니다.

## 10. 반응형 규칙

### 데스크톱

- 너비: `min(920px, calc(100vw - 40px))`
- 높이: `min(760px, calc(100vh - 40px))`
- 도감과 연계 카드: 3열, 충분한 폭이면 4열
- 상세 화면: 얼굴/요약과 기록을 2열 grid로 배치 가능

### 모바일

- 팝업은 화면 가장자리에서 최소 8px 안전 여백을 둡니다.
- 너비는 가용 영역 100%를 사용합니다.
- 높이는 `calc(100dvh - 16px)` 이내로 제한합니다.
- 헤더와 탭은 고정하고 콘텐츠만 스크롤합니다.
- 카드 grid는 기본 2열, 360px 미만에서는 1열을 허용합니다.
- 탭 수가 화면 너비를 넘으면 탭 바만 가로 스크롤합니다.
- 탭 글자 크기는 뷰포트 비례로 커지지 않게 `rem`과 `em`을 사용합니다.
- 최소 터치 높이는 40px입니다.
- 브라우저 주소창 변화에 대응하기 위해 `100vh`보다 `100dvh`를 우선합니다.

## 11. 시각 규칙

- 기존 플랫 디자인을 유지합니다.
- 그라데이션, 그림자, 블러, 글로우를 사용하지 않습니다.
- 팝업 배경은 밝은 단색, 외곽선은 진한 단색을 사용합니다.
- 탭 활성 상태는 밑줄, 배경색, 굵기 중 두 가지 이상으로 구분합니다.
- Bronze, Silver, Gold는 채도가 다른 단색 배지로 표현하고 광택 금속 효과를 넣지 않습니다.
- 카드 모서리는 기존 시스템과 같은 8px 이하를 사용합니다.
- 카드 안에 다시 카드 형태의 프레임을 중첩하지 않습니다.
- 진행도 바는 단색 채움과 숫자를 함께 표시합니다.

## 12. 접근성 및 입력

- 팝업 컨테이너: `role="dialog"`, `aria-modal="true"`
- 탭 바: `role="tablist"`
- 탭: `role="tab"`, `aria-selected`, `aria-controls`
- 탭 콘텐츠: `role="tabpanel"`
- `Escape`: 팝업 닫기
- 좌우 방향키: 탭 이동
- 팝업이 열리면 닫기 버튼 또는 활성 탭으로 포커스 이동
- 닫으면 컬렉션 진입 버튼으로 포커스 복귀
- 배경 콘텐츠는 포커스할 수 없게 처리

전투 캔버스의 pointer 이벤트가 팝업 클릭을 받지 않도록 overlay가 이벤트를 차단해야 합니다.

## 13. Alpine 구조

컬렉션 허브는 `PopupService`를 사용하지 않습니다.

이유:

- 현재 `PopupService`는 `bodyHtml`과 `x-html`을 사용합니다.
- 탭, 검색, 필터, 카드 선택처럼 반응형 상태가 많은 UI와 맞지 않습니다.
- 개발 규칙은 문자열 HTML 조립 대신 Alpine 템플릿을 요구합니다.

`index.html`에 전용 템플릿을 둡니다.

```html
<div
    class="collection-hub-overlay"
    x-show="collectionHub.visible"
    role="dialog"
    aria-modal="true"
>
    <section class="collection-hub">
        <header>...</header>
        <nav role="tablist">...</nav>
        <div class="collection-hub-content">
            <section x-show="collectionHub.activeTab === 'roster'">...</section>
            <section x-show="collectionHub.activeTab === 'links'">...</section>
            <section x-show="collectionHub.activeTab === 'achievements'">...</section>
        </div>
    </section>
</div>
```

데이터 흐름:

```text
player profile + roster + definitions
  -> collection view-model formatter
  -> UIController.renderCollectionHub(viewModel)
  -> appStore.collectionHub
  -> Alpine template
```

- 템플릿이 도메인 규칙을 계산하지 않습니다.
- 숙련도, 진행도, 적용 중 연계 효과는 view-model 생성 단계에서 계산합니다.
- UIController는 전달과 상태 갱신만 담당합니다.

## 14. 파일 구조

```text
src/
  collection/
    collection-view-model.js
  ui.js
  styles.css
index.html
```

| 파일 | 책임 |
| --- | --- |
| `collection-view-model.js` | 프로필과 정의를 화면용 데이터로 변환, 검색·필터·정렬 |
| `ui.js` | Alpine 컬렉션 상태와 열기/닫기/탭 액션 연결 |
| `index.html` | 전용 Alpine 탭 및 카드 템플릿 |
| `styles.css` | 팝업, 탭, 그리드, 목록, 모바일 반응형 스타일 |

## 15. 빈 상태와 오류 상태

- 프로필 로드 실패: `기록을 불러오지 못했습니다`와 재시도 버튼
- 검색 결과 없음: `검색 결과가 없습니다`
- 필터 결과 없음: `이 조건에 맞는 항목이 없습니다`
- 업적 정의 없음: `등록된 업적이 없습니다`
- 연계 효과 없음: `아직 해금한 연계 효과가 없습니다`
- 기록 초기 상태: 캐릭터 카드는 유지하고 수치만 `0` 또는 `미기록`

로딩은 localStorage 기반이라 일반적으로 즉시 끝나지만, 향후 서버 저장 전환을 위해 `loading`, `error` 상태를 둡니다.

## 16. 구현 순서

1. 컬렉션 view-model 순수 함수 구현
2. `appStore.collectionHub` 상태와 액션 추가
3. 컬렉션 진입 버튼 추가
4. 전용 Alpine 팝업과 탭 템플릿 구현
5. 도감 grid와 캐릭터 상세 구현
6. 연계 목록 구현
7. 업적 목록과 진행도 구현
8. 데스크톱과 모바일 스타일 구현
9. 키보드 및 포커스 처리
10. 빈 상태와 오류 상태 구현

## 17. 필수 회귀 조건

- 팝업을 열어도 토너먼트와 전투 상태가 초기화되지 않아야 합니다.
- 탭 이동으로 검색과 선택 상태가 다른 탭에 잘못 공유되지 않아야 합니다.
- 카드 수와 업적 수를 하드코딩하지 않아야 합니다.
- 미기록 캐릭터도 도감에 표시되어야 합니다.
- 연계 탭에 장착, 해제, 슬롯 UI가 없어야 합니다.
- 업적 보상 수령 버튼이 없어야 합니다.
- 모바일에서 페이지 전체 스크롤이 생기지 않아야 합니다.
- 팝업 콘텐츠가 작은 화면에서 잘리지 않고 내부 스크롤되어야 합니다.
- 팝업 뒤 캔버스가 클릭을 받지 않아야 합니다.
- 닫은 뒤 키보드 포커스가 진입 버튼으로 돌아와야 합니다.
- 새 탭 정의를 추가해도 기존 탭 ID와 상태가 깨지지 않아야 합니다.
