# 보상 밸런스 설정

> 성장 책임 기준은 [성장 책임 기준](progression-responsibilities.md)을 따른다. 업적은 직접 전투 스탯 대신 수집 보상을 제공하며, 레벨·스탯 배분·장비·마스터리의 전투 책임을 혼합하지 않는다.

플레이어가 획득하거나 소비하는 수치는 `src/rewardBalanceConfig.js`가 단일 원본입니다.
경험치, 사냥터 전리품, 상자, 상인, 장비 성장과 환원, 업적, 숙련도 수치를 조정할 때는 이 파일만 수정합니다.

## 범위

- `experience`: 대전/사냥터 경험치, 레벨 비용, 레벨업 보상
- `rebirth`: 환생 후보 수·희귀 장비 기준 영구 기초 수치·액션/패시브 카드 장착·등급 상한·후보 가중치·불꽃 단계
- `hunting.chest`: 상자 비용, 파손 우선순위, 상자 보상 확률과 수량. 일반 상자만 파편 보상이 가능하고 고급 이상 상자는 장비를 확정 지급한다.
- `hunting.shards`: 전투 파편, 깊이 보정, 패배 보존, 전투 보상 배율
- `hunting.events`: 축복, 상인, 휴식, 상자 방, 저주 제단 등 이벤트 보상
- `equipment`: 장비 요구 레벨, 인벤토리, 강화, 판매, 분해, 융합, 기본·특수 옵션 수치와 특수 효과 쿨다운
- `progression`: 숙련도 조건과 효과, 업적 보상, 성장 보너스 상한

## 수정 원칙

기존 도메인 모듈은 보상 지급과 표시만 담당하고 수치의 원본을 소유하지 않습니다. 수치 변경 후에는 `npm test`, `npm run check`, `npm run format:check`, `node scripts/huntingUserScenario.mjs`를 실행해 보상 결과와 순환을 함께 확인합니다.

## 레벨 보상: 기본 스탯과 대표 행동 강화

레벨 보상은 캐릭터 ID별 `characterLevelProgressions`와 공통 `levelStatTargetMultiplier`에서 관리한다. Lv.2~10의 모든 행은 HP·공격·속도·방어력 중 하나 이상을 올리고, Lv.3·Lv.6·Lv.9 행은 대표 행동 tier를 추가한다. 기본 스탯은 사용자 스탯 배율보다 먼저 적용되며 장비 고정 수치는 그 뒤에 더한다. 수치형 행동 강화는 기준값 대비 5% 단위로 두고, 개수처럼 정수가 필요한 결과는 `Math.round`를 사용한다.

`levelStatTargetMultiplier`는 `1.5`다. 따라서 Lv.10의 HP·공격·속도·방어력 최종 수치는 각 캐릭터의 Lv.1 값의 정확히 150%다. 각 캐릭터의 50% 증가분을 Lv.2~10의 9개 구간에 누적 분배하며, 표시와 적용은 0.5 단위까지 보존한다. 쿨타임 스탯(`skill`)은 이 표의 기초 수치 대상이 아니며, 스탯 배분·장비·숙련도·패시브의 별도 modifier로만 조정한다.

### Spin Ball: 충전 · 나선 충격 · 오버스핀

Spin Ball의 회전 공격은 별도 피해 공식을 만들지 않는다. 충돌 없는 시간으로 기준 `4.5초` 동안 회전 충전을 쌓고, 충전 상태가 목표 각속도 `20πrad/s`, 즉 초당 10회전을 정한다. 실제 각속도는 목표를 향한 `applyTorque()`로만 높아지며, 충돌 뒤에는 각충격으로 낮아진다. 공통 충돌 피해식은 `선형 속도 + 접점 회전 속도`를 피해 속도로 사용한다. 따라서 회전은 별도 `+60%` 보너스가 아니라 기존 선형 속도와 같은 단위로 피해에 환산된다.

기본 충돌은 회전 충전을 모두 소비해 링과 실제 회전이 함께 크게 줄어든다. Lv.3은 충전의 25%를 남기고, Lv.6은 충전 50% 이상 충돌에 `210 × 충전 비율`의 나선 넉백을 더한다. Lv.9는 만충 충돌 자체를 오버스핀으로 바꿔 같은 충전 소비와 함께 최종 충돌 피해를 `1.45배`로 강화한다. 회전 속도는 직접 덮어쓰지 않고 토크와 각충격만 사용한다.

Spin Ball은 전용 `spinGrip` 재질(마찰 16)을 사용한다. 공통 충돌 solver는 실제 각속도를 접점 속도 `v + ω×r`에 포함하고, 재질 조합 마찰로 제한한 접선 충격을 상대 선형 속도에 적용한다. 기본 러버끼리의 마찰 `0.2`와 달리 Spin Ball-러버 조합은 약 `1.789`다. 초당 10회전 스핀볼의 실제 전투 충돌 계측에서 상대 횡발사량은 80px/s 정면 접근 시 `15.5px/s → 138.5px/s`, 220px/s 접근 시 `42.6px/s → 380.9px/s`로 커진다. 이는 별도 스핀 넉백이 아니라 기존 물리 충돌 응답이다.

## 환생 보상

환생의 수치 원본은 `rebirth` 설정이다. 후보는 3장이고 영구 기초 수치 보상은 가중치 72, 액션 카드는 가중치 18, 패시브 카드는 가중치 10으로 같은 공용 풀에서 비복원 추첨한다. 후보 안의 보상 ID는 중복하지 않는다.

- 영구 기초 수치 보상은 HP·공격·속도·방어력 중 서로 다른 두 항목을 무작위로 골라 즉시 누적한다. 각 수치는 `equipment.statRanges.rare`의 기본 범위에 같은 `equipment.statValueRatios`를 곱해 만들므로 희귀 장비의 가치 기준을 그대로 따른다. 후보를 만들 때 선택된 두 수치와 식별자를 함께 저장해, 새로고침 뒤에도 다시 추첨하거나 재계산하지 않는다. `radius`, `mass`, 충돌·이동 판정과 쿨타임은 포함하지 않으며, 카드 rank·장착 한도와 무관하다.
- 액션·패시브 카드의 rank와 장착 한도는 각각 10, 3이다. 중복 카드는 rank만 1 올린다.
- 액션 카드는 원래 Ability 인스턴스를 `abilityTier: 0`으로 생성하므로 해당 캐릭터의 XP 대표 행동 tier를 상속하지 않는다. 발생 Ability가 보유한 rank modifier만 적용한다.
- 패시브 `냉각 회로`는 rank 1에서 모든 능력 쿨타임을 30% 줄이고, rank마다 2%p씩 강화해 rank 10에서 48% 감소를 상한으로 한다. 이 감소는 스탯 배분·숙련도·장비 쿨타임 보정 뒤에 곱한다.
- 불꽃 visual stage는 누적 환생 횟수만 입력으로 사용한다. 사각 셀 밀도는 환생 1회의 56개와 10환생의 최종 92개만 정의하고, 중간 환생은 정수 반올림 선형 보간한다. 셀의 크기·투명도·난류 흔들림도 같은 1~10 구간을 선형으로 보간하므로, 시각 stage 경계에서 강도가 튀지 않는다. `WeakMap` 플룸은 새 셀을 공 외곽 전체의 독립 표면 앵커에 강하게 결합했다가 수명에 따라 결합을 푼다. 새 셀의 초기 속도는 해당 앵커의 외곽 법선 방향이고, 위쪽 부력은 매 프레임 작용하면서 수명에 따라 강해져 초기 궤적을 위로 굽힌다. 결합이 풀린 셀은 공통 후류 방향으로 강제 합류하지 않고 독립적으로 위로 타오르며, 공의 실제 이동과 남은 이전 위치가 자연스러운 후류를 만든다. 곡선 윤곽이나 외곽 점선은 그리지 않는다. 셀은 수명에 따라 밝은 중심·주황 몸통·적색 끝으로 바뀌며 `source-over` 알파 합성으로 겹친다. 물리 수치·저장 데이터는 바꾸지 않는다.

같은 220px/s 정면 충돌에서 만충 접점 회전 속도는 약 `3140px/s`다. 피해 속도는 `220 + 3140 = 3360px/s`로 환산되며, 현재 스핀볼 기본 수치·방향 조건의 충돌 피해는 기존 보너스 방식 `4 → 6` 대신 선형 속도 환산 방식 `4 → 67`이 된다.

### Hero Ball: 귀환 자석 · 축적 · 방출

Hero의 오브는 멀리서 효과를 바로 획득하지 않는다. 모든 효과 적용은 Hero 본체와 오브가 실제로 충돌했을 때만 발생한다.

| 구간 | 강화      | 수치와 판정                                                                                                                                                                                                                                          |
| ---- | --------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Lv.3 | 귀환 자석 | 발사하거나 방출한 자신의 오브는 생성 뒤 1초 동안 자석 대상이 아니다. 이후 `Hero 반지름 × 2.5 + Orb 반지름` 안에 들어오면 Hero 방향으로 물리적으로 가속한다. 자석은 위치를 옮기거나 즉시 획득하지 않는다.                                             |
| Lv.6 | 축적      | 실제로 적용된 자신의 오브 효과마다 축적을 1개 얻고 최대 20개까지 보관한다. 적대 전투원과의 충돌에서 실제 피해가 0보다 클 때, `최종 충돌 피해 × (1 + 축적 × 0.03)`을 적용한 뒤 축적을 모두 소비한다. 투사체·벽·지형·피해 0 충돌은 소비 조건이 아니다. |
| Lv.9 | 방출      | Lv.6 축적을 소비한 같은 적대 전투원 충돌이 끝난 뒤, 소비한 축적의 절반인 `Math.round(축적 / 2)`개를 Hero 몸 바깥에서 새 Hero Orb로 방출한다. 방출 오브는 기존 Hero Orb의 종류 가중치와 `owner 전투 속도 × 1.2~1.5` 속도 산식을 그대로 사용한다.      |

방출 오브는 몸 바깥을 향해 시작하고 Hero보다 빠른 기존 속도 산식을 사용하므로, 같은 프레임에 다시 획득되지 않는다. 이후 되돌아와 실제 충돌하면 일반 오브와 같은 방식으로 획득할 수 있다. 방출분도 `HERO_ORB_MAX_ACTIVE_PER_OWNER`의 기존 owner별 활성 오브 상한을 공유한다.

## 숙련도 수치

숙련도 등급 수치는 `progression.masteryTiers`가 단독 소유한다. 동일 키에 대한 누적 상한은 없으며, 여러 캐릭터가 같은 키를 제공하면 단순 가산된다. 캐릭터별 정의나 전투 코드는 수치를 소유하지 않는다.

숙련도는 장비의 고정 수치까지 합친 fighter spec에 마지막으로 퍼센트 적용한다. 충돌 피해 증감도 장비 충돌 보정과 액션 처리 뒤의 최종 피해 단계에서 적용한다. 따라서 장비와 숙련도의 수치를 조정할 때는 서로의 효과가 곱해지는 순서를 바꾸지 않는다.

수치 변경 전후에는 능력이 없는 표준 볼로 같은 시드와 좌우 시작 위치를 교차 비교한다. GOLD 단일 효과의 목표는 일반 스탯·충돌·회복 계열에서 대체로 `+3~5%p` 승률 변화다. 속도 복귀율과 쿨다운은 승률 외에 HP 잔량과 능력 회전율도 함께 본다.

갈증은 `masteryRuntime.vampire`의 4초 준비 시간과 최대 결손 HP 배율 2를 사용한다. 회복식은 `실제 충돌 피해 × 등급 흡혈률 × (1 + 결손 HP 비율)`이며, 표준 충돌과 고피해 충돌의 회복 격차를 함께 측정한다.

## 장비 스탯 환산

`scripts/equipmentStatBalance.mjs`는 능력이 없는 표준 볼(HP 100, 공격력 10, 방어력 1, 속도 300)로 장비형 고정 스탯의 상대 가치를 측정합니다. 같은 시드를 좌우 시작 위치에 교차 적용하고, 장비 없음 전투와의 승률 상승폭만 계산해 시작 위치 편향을 제거합니다. 제한 시간 안에 끝나지 않은 전투는 최대 HP 대비 남은 HP 비율로 판정해 표본에서 빠지지 않게 합니다.

기본 실행은 `node scripts/equipmentStatBalance.mjs`입니다. 정밀 측정은 PowerShell에서 `$env:BALANCE_SAMPLES=120; node scripts/equipmentStatBalance.mjs`처럼 표본 수를 늘려 실행합니다.

장비 기본 옵션은 등급별 `statRanges`의 숫자를 공통 가치 포인트로 사용한다. 실제 고정 스탯은 `equipment.statValueRatios`를 곱해 생성하며, 현재 기준은 HP 10, 공격력 1, 방어력 1, 속도 5다. 이 비율은 장비 스탯 부여의 유일한 기준값이며, common 장비의 1~3포인트 옵션은 HP +10~30, 공격 +1~3, 방어 +1~3, 속도 +5~15 중 하나가 된다.

새 장비명은 가장 높은 가치 단위의 기본 스탯 하나를 대표 접두사로 사용한다. HP는 `튼튼한` 계열, 공격은 `맹공의` 계열, 방어는 `수호자의` 계열, 속도는 `질풍의` 계열을 베이스 장비명 앞에 붙인다. 특수 옵션이 있으면 이름과 구분되는 점 접미사로 붙인다. 예: `질풍의 철검 • 갈망`.

## 장비 특수 옵션

현재 특수 옵션은 같은 종류를 여러 장비에서 얻어도 가장 높은 수치 하나만 적용한다. 서로 다른 종류는 함께 적용한다.

- `파쇄`: 적대 전투원 간 충돌 거래에서 자신이 주는 모든 실제 충돌 피해의 원본 수치를 증가시킨다. 기본 충돌과 충돌 처리 중 발생한 접촉형 스킬 피해가 대상이다.
- `순환`: 장착자의 스킬 쿨다운을 비율만큼 줄인다. 스킬 스탯과 숙련도 쿨다운 보정 뒤에 함께 곱해진다.
- `갈망`: 적대 전투원 간 한 번의 충돌 거래가 끝난 뒤, 기본 충돌과 접촉형 스킬로 준 실제 총 피해의 비율만큼 회복한다. HP가 부족하고 장비 효과 쿨다운이 준비됐을 때만 발동하며, 현재 쿨다운은 `2.5초`다.
- `중량`: 실제 질량을 증가시켜 rigid-body 충돌과 회전 관성 결과를 바꾼다. Gunner 숙련도의 중량 장전과 곱으로 결합된다.
- `반향`: 벽 반사 뒤 법선 속도를 비율만큼 증가시킨다. Orbit 숙련도의 반사 궤도와 곱으로 결합된다.
- `소용돌이`: collision solver가 소유자에게 전달하는 각충격을 비율만큼 증가시킨다.

특수 옵션 변경은 `src/rewardBalanceConfig.js`에서 수치만 조정하고, 충돌 거래 기준이 유지되는지 회귀 테스트와 시뮬레이션으로 확인한다.
