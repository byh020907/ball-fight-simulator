# 보상 밸런스 설정

> 성장 책임 기준은 [성장 책임 기준](progression-responsibilities.md)을 따른다. 업적은 직접 전투 스탯 대신 수집 보상을 제공하며, 레벨·스탯 배분·장비·마스터리의 전투 책임을 혼합하지 않는다.

플레이어가 획득하거나 소비하는 수치는 `src/rewardBalanceConfig.js`가 단일 원본입니다.
경험치, 사냥터 전리품, 상자, 상인, 장비 성장과 환원, 업적, 숙련도 수치를 조정할 때는 이 파일만 수정합니다.

## 범위

- `experience`: 대전/사냥터 경험치, 레벨 비용, 레벨업 보상
- `hunting.chest`: 상자 비용, 파손 우선순위, 상자 보상 확률과 수량. 일반 상자만 파편 보상이 가능하고 고급 이상 상자는 장비를 확정 지급한다.
- `hunting.shards`: 전투 파편, 깊이 보정, 패배 보존, 전투 보상 배율
- `hunting.events`: 축복, 상인, 휴식, 상자 방, 저주 제단 등 이벤트 보상
- `equipment`: 장비 요구 레벨, 인벤토리, 강화, 판매, 분해, 융합, 기본·특수 옵션 수치와 특수 효과 쿨다운
- `progression`: 숙련도 조건과 효과, 업적 보상, 성장 보너스 상한

## 수정 원칙

기존 도메인 모듈은 보상 지급과 표시만 담당하고 수치의 원본을 소유하지 않습니다. 수치 변경 후에는 `npm test`, `npm run check`, `npm run format:check`, `node scripts/huntingUserScenario.mjs`를 실행해 보상 결과와 순환을 함께 확인합니다.

## 장비 스탯 환산

`scripts/equipmentStatBalance.mjs`는 능력이 없는 표준 볼(HP 100, 공격력 10, 방어력 1, 속도 300)로 장비형 고정 스탯의 상대 가치를 측정합니다. 같은 시드를 좌우 시작 위치에 교차 적용하고, 장비 없음 전투와의 승률 상승폭만 계산해 시작 위치 편향을 제거합니다. 제한 시간 안에 끝나지 않은 전투는 최대 HP 대비 남은 HP 비율로 판정해 표본에서 빠지지 않게 합니다.

기본 실행은 `node scripts/equipmentStatBalance.mjs`입니다. 정밀 측정은 PowerShell에서 `$env:BALANCE_SAMPLES=120; node scripts/equipmentStatBalance.mjs`처럼 표본 수를 늘려 실행합니다.

장비 기본 옵션은 등급별 `statRanges`의 숫자를 공통 가치 포인트로 사용한다. 실제 고정 스탯은 `equipment.statValueRatios`를 곱해 생성하며, 현재 기준은 HP 10, 공격력 1, 방어력 1, 속도 5다. 이 비율은 장비 스탯 부여의 유일한 기준값이며, 일반 장비의 1~3포인트 옵션은 HP +10~30, 공격 +1~3, 방어 +1~3, 속도 +5~15 중 하나가 된다.

새 장비명은 가장 높은 가치 단위의 기본 스탯 하나를 대표 접두사로 사용한다. HP는 `튼튼한` 계열, 공격은 `맹공의` 계열, 방어는 `수호자의` 계열, 속도는 `질풍의` 계열을 베이스 장비명 앞에 붙인다. 특수 옵션이 있으면 그 효과를 접미사로 붙인다. 예: `질풍의 철검 갈망`.

## 장비 특수 옵션

현재 특수 옵션은 같은 종류를 여러 장비에서 얻어도 가장 높은 수치 하나만 적용한다. 서로 다른 종류는 함께 적용한다.

- `파쇄`: 적대 전투원 간 충돌 거래에서 자신이 주는 모든 실제 충돌 피해의 원본 수치를 증가시킨다. 기본 충돌과 충돌 처리 중 발생한 접촉형 스킬 피해가 대상이다.
- `순환`: 장착자의 스킬 쿨다운을 비율만큼 줄인다. 스킬 스탯과 숙련도 쿨다운 보정 뒤에 함께 곱해진다.
- `갈망`: 적대 전투원 간 한 번의 충돌 거래가 끝난 뒤, 기본 충돌과 접촉형 스킬로 준 실제 총 피해의 비율만큼 회복한다. HP가 부족하고 장비 효과 쿨다운이 준비됐을 때만 발동하며, 현재 쿨다운은 `2.5초`다.
- `중량`: 실제 질량을 증가시켜 rigid-body 충돌과 회전 관성 결과를 바꾼다.
- `반향`: 벽 반사 뒤 법선 속도를 비율만큼 증가시킨다.
- `소용돌이`: collision solver가 소유자에게 전달하는 각충격을 비율만큼 증가시킨다.

특수 옵션 변경은 `src/rewardBalanceConfig.js`에서 수치만 조정하고, 충돌 거래 기준이 유지되는지 회귀 테스트와 시뮬레이션으로 확인한다.
