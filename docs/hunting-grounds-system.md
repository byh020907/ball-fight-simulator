## 1. 시스템 정의

사냥터는 토너먼트와 분리된 로그라이크식 파밍 모드입니다. 토너먼트가 우승, 도전 단계, 디펜딩 챔피언, 숙련도 같은 경쟁/증명 루프를 담당한다면, 사냥터는 연전, 위험 감수, 랜덤 이벤트, 장비 파밍, 상자와 재화 파밍을 담당합니다.

플레이어가 토너먼트를 이기기 어렵다고 느낄 때 사냥터에서 경험치와 장비를 파밍해 캐릭터를 성장시킨 후 다시 토너먼트에 도전하는 선순환 구조입니다. 경험치 시스템(레벨 보상)과 통합되어 있으며, 장비 시스템은 별도 문서([`docs/equipment-system.md`](equipment-system.md))에서 정의합니다.

핵심 질문은 포탈을 발견했을 때 반복됩니다.

```text
지금 포탈로 귀환해서 보상을 확정할까?
다음 10층으로 가서 더 좋은 보상을 노릴까?
(단, 다음 포탈을 찾을 때까지 귀환 불가)
```

## 2. 설계 원칙

### 2.1 토너먼트와 역할 분리

사냥터에서는 토너먼트 우승, 도전 단계 해금, 디펜딩 챔피언 streak를 처리하지 않습니다. 사냥터 보상은 상자, 열쇠 조각, 사냥터 전용 해금, 소량 XP 중심으로 둡니다.

### 2.2 스테이지 선택

사냥터 입장 전에 원정지를 선택할 수 있습니다. 동굴(1000×1000 기준), 숲(1280×1280 기준), 사막(1440×1280 기준) 세 스테이지가 있으며, 100층 보스를 처치하면 다음 스테이지가 해금됩니다. 각 스테이지는 고유한 기준 전장 크기와 시각적 테마(암석/숲/모래)를 가집니다. 전투 가독성을 위해 모든 stage 배경은 이름표·HP 바·이펙트가 묻히지 않도록 밝은 팔레트를 사용합니다. 선택한 스테이지는 프로필에 저장되어 다음 입장 시에도 유지됩니다.

### 2.3 실제 지형 (Terrain)

stage는 단순 배경 외에 실제 충돌 지형을 가질 수 있습니다. 현재 동굴(cave) stage는 기본 지형을 폴리곤 암벽 장애물로 생성하며, 원형 장애물은 사냥터 생성에서 사용하지 않습니다. fighter와 충돌 시 arena 벽 반사와 동일한 방식으로 밀려납니다. 충돌 계층은 레거시 호환을 위해 원형 entity vs circle/polygon terrain을 모두 지원하지만, 신규 사냥터 배치는 polygon terrain을 기준으로 합니다 (SAT 기반 polygon 충돌). 지형은 `stageId`와 `floor`에 기반해 결정론적으로 생성되므로 같은 조건에서는 항상 같은 배치가 나옵니다.

모든 `BattleBall`(원형 캐릭터 포함)은 기본적으로 `RotationalBody` 믹스인을 통해 회전각과 각속도를 가집니다. 기본 초기 각속도는 `randomSpin()` 헬퍼로 생성되며 최소 절대값이 0.9 rad/s로 보장되어 화면에서 눈에 띄게 회전합니다. 다각형 몹은 `appearance.angle`/`appearance.angularVelocity`가 명시되면 그 값을 우선 사용하고, 없으면 동일한 최소 회전 속도를 받습니다. `spec.rotationEnabled === false`로 캐릭터별 회전을 비활성화할 수 있습니다. `angularDamping`은 초당 유지율로 `Math.pow(angularDamping, delta)`로 적용되어 프레임레이트에 무관하게 일관된 감쇠를 보장합니다.

몹의 얼굴과 장착 장비도 `this.angle` 기준으로 회전하며, 이름표·HP바·movement ring 같은 UI 요소는 회전하지 않습니다. WallSlam 효과는 물리 `applyAngularImpulse()`를 통해 각운동량을 전달하므로 회전이 활성화된 볼의 각속도가 물리적으로 변화합니다. 회전이 비활성화된 볼은 영향을 받지 않습니다. 비활성 상태에서는 `update()`에서 `clearAngularForces()`가 호출되어 accumulator가 초기화됩니다.

회전값은 `generateMobAppearance(rng)`에서 rng 기반으로 생성되어 재현 가능합니다. 몹-vs-플레이어 충돌도 원형 radius가 아닌 polygon shape 기반으로 처리됩니다: `resolveFighterShapeCollision`이 SAT 알고리즘으로 polygon-polygon, circle-polygon 충돌을 감지하고, SAT normal 방향으로 충돌 반응(impulse)을 주며 분리합니다. 분리 시 bounding circle overlap을 하한으로 사용하여 완전한 분리를 보장합니다.

투사체 충돌, 몹 AI의 장애물 우회, concave polygon, 동적 회전 장애물은 후속 과제입니다.

### 2.2 포탈 귀환이 핵심

귀환은 포탈 이벤트에서만 가능합니다. 일반 전투 승리 후에는 귀환할 수 없고, 반드시 10층 전진을 눌러 다음 층으로 이동해야 합니다. 포탈을 지나치면 귀환 권한이 사라집니다.

사냥터의 재미는 포탈을 발견했을 때 "지금 귀환할까, 더 깊이 갈까" 판단하는 데 있습니다. 귀환하지 않고 전진하면 더 높은 층에서 더 좋은 보상을 노릴 수 있지만, 다음 포탈을 찾을 때까지 귀환할 수 없습니다.

### 2.3 보상은 확률보다 축적

상자는 등급별 기대감을 주되, 핵심 해금은 적 처치 재화로 확정 진행되게 합니다. 상자에서 무작위 보상을 얻더라도, 장기적으로는 `열쇠 조각`을 모아 상자를 열거나 설계도를 해금하는 구조를 우선합니다.

### 2.4 다수 전투 확장 전제

사냥터는 장기적으로 `1 대 n`, `n 대 n`, 웨이브 전투를 필요로 합니다.

MVP는 기존 1v1 전투를 연속으로 이어붙여 시작할 수 있습니다. 전투 시스템은 팀 기반으로 리팩터링되어 있으며, 같은 팀끼리는 선형/회전 충돌 물리(분리, collision impulse, angular impulse)는 모두 동일하게 적용되지만, 피해와 적대 효과(actionContext collision hooks, ability onCollision, sound/spark)는 발생하지 않습니다.

## 4. 기본 루프

### 4.1 입장 조건

플레이어가 토너먼트에서 **한 번이라도 우승한 캐릭터**만 사냥터에 입장할 수 있습니다. 우승 기록이 없는 캐릭터는 사냥터 선택 UI에 표시되지 않습니다.

### 4.2 HP 규칙

사냥터 내 캐릭터 HP는 **층간 완전 누적**됩니다. 전투가 끝나도 HP는 회복되지 않으며, 휴식지 이벤트나 특별 효과를 통해서만 회복할 수 있습니다. HP가 0이 되면 사냥터 패배로 처리됩니다.

원정이 첫 전투 전에 이벤트를 만나도 HP 상태는 비어 있지 않습니다. 이벤트와 다음 층 판정 전에 현재 캐릭터의 레벨·스탯 배분·장비·숙련도·사냥터 보정·Hero Ball 이월 HP를 반영한 최대 HP를 확정하고, 현재 HP는 그 범위 안으로 동기화합니다. 따라서 휴식지, 함정, 상인 수리와 저체력 이벤트 확률은 같은 실제 HP를 사용합니다.

전투 계산·저체력 이벤트 확률·업적 판정에는 원본 실수 HP를 유지한다. 반면 준비 화면, 이벤트 결과, 상인 수리, 물약 결과처럼 사용자에게 보이는 HP는 `getHuntingDisplayHealth()`/`getHuntingDisplayHp()`를 통해 항상 올림 정수로 표시한다. 양수 HP가 0으로 보이지 않게 하면서도 표시 때문에 실제 원정 체력이 바뀌지 않도록 한 경계다.

#### HP 물약

HP 물약은 사냥터의 층간 누적 HP를 보완하는 영구 소모품입니다. 파편으로 미리 사서 프로필에 보관하며, 현재는 사냥터 안에서 즉시 구매할 수 없고 전투 중에는 사용할 수 없습니다.

- 가격과 보유: 한 병은 파편 100, 최대 10개까지 보유한다.
- 회복: 전투 준비 화면에서만 한 병으로 최대 HP의 25%를 회복한다. 현재 HP가 최대 HP면 사용하지 못하며, 회복 후 HP는 최대 HP를 넘지 않는다.
- 원정 한도: 기본 1회, 영구 확장으로 최대 5회다. 확장 비용은 100 → 200 → 400 → 800 파편이며, 프로필에 저장한다.
- 전투 한도: 한 전투 준비 화면에서는 전투 준비용 물약을 합쳐 최대 한 병만 마신다. 원정 런은 물약을 미리 옮기지 않고 그 원정에서 실제로 사용한 횟수만 저장한다. 병 수는 마시는 순간 프로필 보유 수량에서 차감하므로, 쓰지 않은 물약은 그대로 남는다.
- 확장성: 소모품은 정의별 보유 한도·사용 시점·효과를 가진다. HP 물약은 `사냥터 전투 전` 사용 규칙을 갖고, 경험치 물약 등은 이후 같은 인벤토리 구조에 별도 사용 규칙으로 추가할 수 있다.

### 4.3 진행 루프

```text
사냥터 입장 (캐릭터 선택, stage 선택)
  -> 1층에서 시작
  -> 첫 이동 UI가 그려진 뒤 자동으로 10층 전진
  -> 최대 10층을 1층씩 이동하며 층별 판정 수행:
      - 빈층: 메시지 출력 후 계속 진행
      - 전투: 이동 정지 → 전투 준비 → 전투 시작 → 승리 시:
        - 상자 드롭 시: 상자 UI(확인) → "10층 전진" 버튼
        - 상자 없음: 바로 "10층 전진" 버튼 표시
      - 이벤트: 유형에 따라 처리 (아래 §7 참고)
      - 최종보스(100층): 전투 준비 → 전투 시작 → 승리 시:
        - 상자 드롭 시: 상자 UI(확인) → 스테이지 클리어 결과
        - 상자 없음: 바로 스테이지 클리어 결과
  -> 포탈 이벤트에서만 "포탈 귀환" 가능
  -> 패배 시 일부 보상 손실 후 종료
```

100층 전진 루프:

```text
HUNTING_MAX_FLOOR = 100
HUNTING_ADVANCE_STEPS = 10

advance() 루프:
  for step 0..9:
    1층 전진 → rollHuntingFloorOutcome(floor)
    empty     → 로그 + 애니메이션 → continue
    combat    → 전투 준비 화면 → _startFloorBattle() → return (정지)
    event     → _handleAdvanceEvent() → portal/상인/챔피언 → 정지
                                      → boon/mishap/rest/chest/altar → continue
    final_boss → 전투 준비 화면 → _startFloorBattle() → return (정지)

모든 10단계 소진 시 → "X층 전진 완료" + 전진 버튼 다시 표시

이동 중 UI의 좌우 층 표시(`7F → 17F`)는 각 1층 이동의 출발/도착이 아니라
이번 전진 액션의 시작층과 목표층을 고정해서 보여준다. 진행 바의 max도
실제 이동할 층 수에 맞춰진다(95층 시작 시 5).
```

### 4.4 전장 루팅

일반 사냥터 몬스터가 처치되면 전장에 `일반 파편` 3~7개를 반드시 떨어뜨리고, 각 파편은 층수에 따라 변하는 독립 보상값을 가진다. `소형 힐팩`, `파편 꾸러미`, `일반 상자`, `고급 상자`는 추가로 떨어뜨릴 수 있다. 아이템은 처치 즉시 보상으로 합산하지 않으며, 플레이어가 전장에서 실제로 닿아 회수한 뒤에만 이번 전투의 임시 보상에 기록한다. 패배하면 이 임시 보상은 폐기되고, 승리하면 **회수한 전리품만** 사냥터 런의 pending loot으로 병합한다. 전투 승리 뒤 별도로 계산해 지급하는 파편 보상은 없다.

- 일반 몬스터는 층수와 무관하게 랜덤한 3~7개의 일반 파편을 확정으로 생성한다. 각 일반 파편은 해당 층의 랜덤 보상값을 독립적으로 가지며, 처치 지점에서 각자 360도 무작위 방향으로 발사된다. 그 뒤 별도의 보너스 드롭 판정을 한 번 가진다. 보너스 기본 확률은 15%이고, 플레이어의 잃은 체력 비율에 따라 선형으로 증가해 체력이 0에 가까울수록 최대 30%(기본의 2배)가 된다. 고등급 몬스터에 추가 독립 보상 판정을 더하지 않는다.
- 보너스 드롭 판정을 통과한 뒤에는 몬스터 태그의 `rarity:common|rare|unique|epic`에 맞는 하나의 보상 테이블을 고른다. common은 힐팩과 일반 상자만 사용하고, 파편 꾸러미와 고급 상자는 rare부터 나온다.

| 몬스터 등급 | 파편 꾸러미 | 고급 상자 | 일반 테이블 잔여 |
| --- | ---: | ---: | ---: |
| common | 0% | 0% | 100% |
| rare | 15% | 5% | 80% |
| unique | 30% | 10% | 60% |
| epic | 45% | 20% | 35% |

- 보너스 테이블은 잃은 체력 비율 `m`에 따라 소형 힐팩 `20 + 20m`, 일반 상자 `10`의 비율을 쓴다. 등급별 특별 보상이 먼저 차지한 뒤에는 이 비율을 남은 확률에 맞게 비례 축소한다. 만피여도 힐팩 확률을 파편으로 치환하지 않으며, 회복할 체력이 없는 힐팩은 회수되지 않는다.
- 소형 힐팩은 회수 순간의 잃은 체력 25%를 회복한다. 일반 파편은 층수와 무관하게 물리적으로 3~7개를 무작위로 생성한다. 각 파편의 보상값은 1층에서 3~7로 무작위이며, 25층마다 값 범위 전체를 5씩 올린다. 따라서 깊은 층도 등급보다 층수의 경제 성장을 크게 반영한다. 파편 꾸러미는 이 층수 기준값에 rare `1~2배`, unique `1~2.5배`, epic `1~3.5배`를 무작위로 적용하고 중앙값이 양끝보다 자주 나오게 하며, 꾸러미 결과만 5 단위로 정규화한다.
- 고급 상자는 rare에서 uncommon 확정, unique에서 uncommon 70%/rare 30%, epic에서 rare 70%/epic 30%다. legendary는 일반 몬스터 드롭에 넣지 않고 상자방·업적 같은 특별 경로에 남긴다.
- 일반 전투 중간 보스는 `rare`·`unique`·`epic` 일반 몬스터를 강화한 적이다. 원본의 몬스터 태그·처치 통계·기본 파편/추가 전리품을 유지하고, 강화석 물리 드롭을 추가한다.
- 챔피언 난입과 최종 보스는 로스터 기반 보스라 일반 몬스터 태그와 일반 몬스터 전장 루팅은 받지 않는다. 세 보스 역할 모두 강화석을 전장에 물리적으로 드롭하며, 각 강화석은 값 1로 1층 1~3개에서 100층 4~12개까지 선형 증가한 수량만큼 나온다.
- 상자는 기존 전투 보상 상자 UI를 재사용한다. 전장에서 실제로 회수한 상자만 승리/최종 보스 결과보다 먼저 `확인` 흐름으로 표시하며, 여러 개면 회수 순서대로 모두 확인한다.
- 루팅 아이템은 기본적으로 플레이어 반지름의 4배 범위 안에서 자석으로 끌려온다. 일반 몬스터를 모두 처치해 승리하면 기존 결과 연출 안에서 약 1초 동안 남은 아이템을 전장 전체 자석으로 회수한 뒤 결과 흐름으로 넘어간다. 이 물리 보정은 Hero Orb의 자석 효과와 같은 공통 유틸리티를 쓰며, Hero Orb만 보유자의 능력 기반 범위 배율을 추가로 적용한다.
- 전리품은 Hero Orb보다 분명히 큰 반지름 16~22로 표시한다. 회수 순간에는 보상 색상의 큰 펄스와 24개 입자, 보상 문구, 짧은 상승 수집음이 공통으로 재생된다. 따라서 파편·힐팩·상자는 서로 다른 보상 색과 문구를 유지하면서도 실제 회수 여부를 즉시 알 수 있다.
- 모든 전리품은 `RotationalBody` 믹스인으로 공통 회전각과 각속도를 가지며, 기존 `randomSpin()` 초기값과 `integrateRotation()` 갱신을 사용한다. 종류별 `drawItem()`은 그대로 두고 공통 기반이 아이템 중심을 기준으로 회전 좌표계를 적용한다.
- 모든 전리품은 처치한 몬스터의 유효 이동 속도에 Hero Orb와 같은 1.2~1.5배 계수를 적용해 360도 무작위 방향으로 발사된다. 생성 뒤 1초 동안은 자석과 직접 회수가 모두 유예되어 처치 위치에서 잠시 흩어진 뒤 기존 자석 회수 경로를 탄다. 이 시간은 `CollectionGrace` 믹스인을 통해 Hero Orb와 같은 상태·갱신 규칙을 공유한다.
- 전리품은 벽과 blocking 지형에 충돌해 반사한다. 회수 대상이 아닌 몬스터와 닿으면 Hero Orb와 같은 반발 처리로 전리품만 튕겨 나가며, 몬스터의 위치·속도에는 영향을 주지 않는다. 회수 대상 플레이어와의 접촉은 유예가 끝난 뒤 기존 회수 규칙을 우선한다.

`HuntingLootItem`이 수명, 이동, 자석, 회수 피드백과 임시 보상 기록을 소유한다. `SmallHealPack`, `ShardDrop`, `ShardBundleDrop`, `ChestDrop`은 각 보상 적용과 렌더링만 구현하고, `HuntingLootDropController`가 일반 몬스터 처치 hook에서 아이템을 생성한다. 새 루팅 종류는 이 공통 기반 클래스를 상속해 등록한다.

### 4.5 경험치 지급

사냥터 전투에서도 매치 XP 공식이 동일하게 적용됩니다. 단, 사냥터 전투는 토너먼트 단계가 없으므로 `stageMultiplier = 1.0` 고정입니다. Lv.3·Lv.6·Lv.9의 대표 행동 강화도 사냥터 전투에 동일하게 적용되어, 토너먼트보다 느리지만 꾸준히 캐릭터가 성장합니다.

확장 흐름:

```text
한 층에 적 여러 명 연속 등장
일부 층은 1 대 n 전투
정예/챔피언 난입
상자방/상인/제단/휴식지 노드
```

## 5. 전투 구조 요구사항

사냥터 구현 전투 시스템은 팀 기반이어야 하며, 현재 기본 `BattleSimulation`은 명시적 `teamId`가 없을 때 fighter별 고유 팀을 부여해 기존 개인전 동작을 유지합니다.

### 5.1 팀 모델

각 fighter는 `teamId`를 가집니다.

```js
{
    id: "archer",
    teamId: "player"
}
```

기본 토너먼트 1v1:

```text
player fighter -> teamId: "player"
AI fighter     -> teamId: "rival"
```

사냥터 예시:

```text
player team:
  - player fighter

enemy team:
  - enemy fighter 1
  - enemy fighter 2
  - elite fighter
```

미래 n 대 n 예시:

```text
player team:
  - player fighter
  - ally fighter

enemy team:
  - enemy fighter 1
  - enemy fighter 2
```

### 5.2 아군 피해 방지

같은 팀끼리는 다음 효과가 발생하지 않아야 합니다.

- 충돌 피해
- 투사체 피해
- 폭발 피해
- 흡혈/삼키기/그림자 공격 같은 적대 능력
- 클릭 액션의 적대 효과

같은 팀끼리도 물리 충돌과 밀어내기는 유지할 수 있습니다. 다만 플레이 감각이 나쁘면 사냥터 전용으로 아군끼리 통과하거나 약한 분리만 적용하는 옵션을 검토합니다.

### 5.3 타겟 선택

`simulation.getOpponent(ball)`은 기존 1v1 호환을 위해 유지하되, 내부적으로는 같은 팀이 아닌 생존 fighter 중 가장 적절한 대상을 반환해야 합니다.

권장 API:

```js
simulation.isHostile(a, b);
simulation.getEnemiesOf(fighter);
simulation.getNearestEnemy(fighter);
simulation.getOpponent(fighter); // getNearestEnemy 별칭으로 유지
```

투사체와 광역 효과는 `getOpponent()` 하나만 쓰지 말고 필요에 따라 `getEnemiesOf()`를 순회해야 합니다.

## 6. 난이도 상승

### 동굴 몬스터 풀

동굴 몬스터는 14종의 데이터 정의를 사용한다. 모든 종은 1층부터 낮은 확률의 후보이며, 각 `주력 전환 층`에 가까워질수록 확률이 올라간다. 전환 층에 도달한 최신 몬스터는 3배 가중치로 해당 구간의 주력이 되고, 이전 주력은 낮은 누적 확률로 계속 조우 후보에 남는다. 일반 전투 편성은 첫 두 몬스터를 서로 다른 종으로 정해, 초반부터 최소 두 행동을 조합한다. 몬스터 행동은 플레이어 캐릭터 Ability를 재사용하지 않고 전용 행동으로 처리한다. 전기·회복·사슬·충격파·방벽·흡수·레이저 같은 연결선과 파동 이펙트는 행동을 읽게 하는 핵심 신호다.

일반 전투 적 수는 모든 층에서 2~10명이 후보이며, 층이 깊어질수록 높은 편성의 가중치가 커진다. 저층은 2~3명이 주력이지만 10명도 낮은 확률로 등장하고, 고층은 7~10명 편성의 비중이 높아진다. 전장은 해당 전투의 실제 적 수를 기준으로 `1 + (적 수 - 2) / 8`만큼 면적을 확대해 최대 2배까지만 커진다. 따라서 동굴은 2명일 때 1000×1000, 6명일 때 약 1225×1225, 10명일 때 약 1414×1414 정사각 전장을 사용한다. 중간 보스와 최종 보스도 최종 적 편성 수를 기준으로 같은 전장 규칙을 적용한다.

|---|---|---:|---|---|

전기 마법사는 사거리 330 안에서 0.5초 동안만 번개를 시전한 뒤 3초 쿨다운에 들어간다. 시전 중 대상이 사거리를 이탈해도 즉시 피해·연결선을 끄고 같은 3초 쿨다운을 시작하며, 재충전 중 사거리 재진입은 새 시전을 시작시키지 않는다. 번개는 `src/effects/electricArc.js`가 시작점과 끝점 사이를 거리 기반 절점으로 나누고, 각 절점을 시간에 따라 수직으로 흔들어 그린다. 공통 렌더러는 외곽 광선·색 본선·흰 심선을 겹치므로, 다음 연결형 공격도 같은 아크 표현을 재사용할 수 있다. 전기·힐·사슬·흡수와 방벽 교대 같은 연결형 효과는 대상이 처치·파괴·삼킴 상태가 되면 즉시 상태를 해제하며, 같은 프레임 렌더링에서도 선을 남기지 않는다.

레이저 몬스터는 충전 중 현재 목표 방향을 향해 초당 제한된 회전 속도로 조준선을 돌린다. 발사가 시작되면 그 순간의 각도를 고정하고, 해당 방향의 전장 벽까지 충전선과 발사선을 유지한다. 피해 판정도 같은 벽 끝 범위 안에서만 적용된다. 힐러는 반경 안 아군 한 명에게만 초록 회복 빔을 연결해 자기 HP 100/s를 대상 HP 300/s로 전환한다. 힐러는 별도로 최대 HP의 5%/s를 자연 회복한다.

방벽 볼은 4초마다 방벽을 켠 뒤 1.5초 동안 아군과 실제로 충돌했을 때만 위치를 교환한다. 충돌한 아군이 플레이어에 더 가까울 때만 방벽 볼이 그 자리로 나서고, 뒤쪽 아군과의 충돌은 그대로 밀려난다. 방벽 볼끼리는 서로 위치를 바꾸지 않으며, 한 번 교체된 아군은 같은 방벽 턴이 끝날 때까지 다시 교체하지 않는다. 그래서 멀리 있는 아군을 순간이동시키지 않고, 아군이 진로를 막는 순간에만 방벽 볼이 전열을 맡는다. 방벽 시간에는 기존처럼 방어력이 상승하고, 교체 순간은 파란 연결선과 양쪽 파동으로 표시한다.

층이 깊어질수록 적은 강해지고, 일반 파편과 파편 꾸러미의 층수 기준값도 커집니다. 전투 승리 자체에 고정 파편 보상은 없습니다.

```text
enemyPowerMultiplier = 1 + (floor - 1) * 0.08
elitePowerMultiplier = enemyPowerMultiplier + 0.12
```

적 보정 대상은 `hp`, `damage`, `defense`입니다. `speed`, `skill`, `radius`, `mass`는 전투 안정성을 위해 제외합니다.

### 6.1 몬스터 태그와 처치 기록

동굴 몬스터 14종은 모두 `monster` 태그와 하나의 희귀도 태그(`rarity:common`, `rarity:rare`, `rarity:unique`, `rarity:epic`)를 정의 데이터에 함께 둡니다. 사냥터 전투에서 플레이어가 승리하면 실제 편성된 일반 몬스터의 태그를 처치 통계에 누적합니다. 같은 태그는 전장 루팅 테이블도 선택하므로, 새 몬스터는 행동 구현과 함께 태그만 추가하면 기존 희귀도 처치 업적과 등급별 보상에 자동 포함됩니다.

게임 UI에서 몬스터 희귀도는 태그의 소문자 영문 값 그대로 `common`, `rare`, `unique`, `epic`으로 표시한다. 여기서 `일반 몬스터`는 보스가 아닌 사냥터 몬스터를 뜻하는 분류이며, `common` 희귀도와 혼용하지 않는다.

일반 전투 중간 보스는 원본 몬스터 태그를 그대로 받아 같은 등급 전리품 표와 몬스터 처치 통계에 포함됩니다. 반면 챔피언 난입과 최종 보스는 일반 몬스터 태그를 받지 않습니다. 챔피언 난입 승리는 별도 통계로 기록하고, 최종 보스는 스테이지 클리어 통계로 기록합니다. 세 보스 역할의 강화석 드롭은 공통 보스 보상으로 별도 처리하므로, 챔피언/최종 보스가 일반 몬스터 처치 업적을 부풀리지는 않습니다.

### 6.2 몬스터 도감과 지역 기록

사냥터 런을 시작하면 선택한 지역을 `visitedStageIds`에 즉시 기록한다. 따라서 전투 중 컬렉션을 열어도 이미 진입한 지역은 몬스터 상세의 지역 선택지에 나타난다. 실제 일반 몬스터가 전투 편성에 포함되는 순간에는 런의 `monsterCodexByType`에 조우를 기록하고, 해당 전투에서 승리할 때만 처치 수를 더한다. 패배하거나 귀환하지 않은 런의 조우·처치 기록은 영구 프로필에 병합하지 않는다.

각 행동 타입은 도감 카드 하나만 가지며, 기록은 타입 전체와 지역별로 함께 보관한다. 카드가 처음 조우되기 전에는 실루엣과 `미조우`만 표시한다. 조우한 타입의 상세에서는 방문한 지역을 선택할 수 있고, 그 지역에서 아직 해당 타입을 만나지 않았다면 지역 이름만 보여 주고 외형·행동·기본 스탯을 잠근다. 지역별 정의가 공개된 경우에는 그 정의의 행동 설명과 정확한 기본 `HP · 공격 · 방어 · 속도`를 표시한다.

상세는 타입 전체 처치 수와 현재 선택한 지역의 처치 수, 타입 전체 첫·마지막 조우 층을 함께 표시한다. 처치 업적은 지역별로 나누지 않고 타입 전체 처치 수를 사용한다. `hunting_monster_codex_complete`는 현재 등록된 일반 몬스터 타입을 모두 한 번 이상 조우하면 달성하며, 타입 수는 정의 레지스트리에서 계산한다.

## 7. 층 판정과 이벤트

각 층은 `rollHuntingFloorOutcome()`으로 판정됩니다.

```text
층 판정 유형:
  empty      — 빈 통로, 자동으로 다음 층 진행
  combat     — 일반/정예 적 조우, 이동 정지 → 전투
  event      — 이벤트 발생, 유형에 따라 정지 또는 자동 진행
  final_boss — 100층 고정, 챔피언급 보스 전투 → 승리 시 스테이지 클리어

층별 확률 (floor 1 기준):
  combat: ~35%  (깊을수록 ↑)
  event:  ~35%  (깊을수록 ↓)
  empty:  ~30%  (깊을수록 ↓)
```

### 6.1 전투 직후 완충 (Combat Relief)

전투 승리 직후에는 다음 3층 동안 전투 확률이 감소하고 이벤트 확률이 증가합니다.
이는 연속 전투로 인한 피로감을 줄이기 위한 장치입니다.

```text
HUNTING_COMBAT_RELIEF.INITIAL_FLOORS = 3

전투 승리 시 combatReliefFloors = 3
  relief=3: combat × 0.35  event + (감소분 × 0.7)
  relief=2: combat × 0.55  event + (감소분 × 0.65)
  relief=1: combat × 0.75  event + (감소분 × 0.55)
  relief=0: 기본 확률

층 이동마다 relief는 1씩 감소.
100층 final_boss는 완충 영향을 받지 않음.
```

### 7. 층 판정과 이벤트

| 이벤트 | 코드 | 동작 | 정지 여부 |
| --- | --- | --- | --- |
| 포탈 | `portal` | 귀환 또는 계속 전진 선택 | 이벤트 화면에서 선택 |
| 떠돌이 상인 | `wandering_merchant` | 파편으로 장비·강화석 구매 | 이벤트 화면에서 상품 선택 |
| 축복 | `boon` | 6~10 파편에 1층 1배~100층 5배 선형 보상 적용 | 결과 확인 후 전진 |
| 함정 | `mishap` | 현재 HP 5~10% 손실 (최대 HP 20% 이하에서는 등장하지 않음) | 결과 확인 후 전진 |
| 상자방 | `chest_room` | 전투 없이 등급 상자 1개 획득, 미확보 전리품에 보관 | 확인 후 전진 |
| 휴식지 | `rest_site` | HP 25% 회복 | 결과 확인 후 전진 |
| 저주받은 제단 | `cursed_altar` | 스탯 교환 (1~3회 전투 지속) | 결과 확인 후 전진 |
| 챔피언 난입 | `champion_intrusion` | 강화 로스터 보스 등장, 동반 일반 몬스터 전리품과 보스 강화석 회수 | 전투 준비 화면에서 `전투 시작` |

포탈은 유일하게 귀환을 허용하는 이벤트입니다. 포탈을 지나치고 다음 층으로 전진하면 귀환 권한이 사라집니다.

축복 파편 보상은 `round(random(6..10) × (1 + 4 × min(floor - 1, 99) / 99))`입니다. 기본 평균은 8 파편이고, 1층은 6~10, 100층부터는 30~50 파편으로 5배 상한을 유지합니다.

### 7.1 이벤트 결과 확인

모든 사냥터 이벤트는 토스트만 띄우고 지나가지 않습니다. 이벤트명, 효과, 실제 변화량을 오버레이에 표시하고, 다음 행동이 필요한 시점까지 경로 진행을 멈춥니다.

- 축복·함정·휴식지·저주받은 제단: 결과를 확인한 뒤 `계속 전진`합니다.
- 챔피언 난입: 강화 적의 등장은 전투 준비 화면에서 보여 주고, 같은 화면의 `전투 시작`으로 전투에 진입합니다. 챔피언 본체는 일반 몬스터 전리품 대상이 아니지만 강화석을 물리적으로 드롭합니다.
- 포탈·방랑 상인·상자방: 기존의 귀환/상품/상자 전용 상호작용을 유지하면서 같은 이벤트 헤더에 발견 결과를 표시합니다.
- 저주받은 제단은 올리는 스탯·내리는 스탯·각 퍼센트·지속 전투 횟수를 그대로 표시합니다.

### 7.2 저HP 포탈 보정과 거부 억제

캐릭터 HP가 낮을수록 포탈 이벤트가 더 자주 등장합니다. 이벤트 선택 시 포탈에 가중치를 부여하는 방식입니다.

```text
HUNTING_PORTAL_DECLINE.HP_MULT:
  HP ≥ 50%: portal weight × 1.0 (기본)
  30% ≤ HP < 50%: portal weight × 1.8
  HP < 30%: portal weight × 3.0
```

포탈에서 귀환하지 않고 `10층 전진`을 선택하면 `portalDeclineFloors = 5`가 설정되고, 이후 5층 동안 HP 기반 포탈 보정이 억제됩니다(×1.0 고정). 층 이동마다 1씩 감소하며, 0이 되면 다시 HP 기반 보정이 활성화됩니다. 이는 포탈을 거부했을 때 바로 다음 층마다 포탈이 반복되는 것을 방지하기 위함입니다.

## 8. 상자와 개봉 비용

상자는 사냥터 중 획득하지만, 개봉은 보관함에서 열쇠 조각을 써서 진행합니다. 상자방에서는 등급별 색상과 공통 상자 도형을 보여 준 뒤 `계속 전진`을 눌러 다음 층 이동을 재개합니다. 상자 도형은 보관함 카드에서도 같은 컴포넌트를 사용해 등급 표현을 일치시킵니다.

전장에서 상자를 실제로 회수한 전투는 승리 결과 화면 대신 먼저 상자 UI(`확인` 버튼)를 표시합니다. 확인을 누른 후에야 일반 전투 승리 선택(10층 전진) 또는 최종 보스 스테이지 클리어 결과로 진행됩니다. 회수한 상자가 여러 개면 모두 순서대로 확인한 뒤 결과 흐름으로 넘어갑니다. 이 전투 보상 상자는 상자방과 동일한 `<chest-icon>` 컴포넌트를 재사용하지만, `huntingChestConfirmLabel` 상태로 버튼 문구를 구분(`확인` vs `계속 전진`)하고, `HUNTING_CHEST_CONTINUE_HANDLERS` 매핑으로 phase별 후속 handler(`_continueChestRoom` / `_continueCombatRewardChest`)를 분리합니다.

| 등급 | 코드 | 개봉 비용 | 주요 보상 |
| --- | --- | ---: | --- |
| 낡은 상자 | `common` | 20 | 열쇠 조각 환급, 소량 XP, 일반 색상 |
| 튼튼한 상자 | `uncommon` | 50 | 색상, 낮은 등급 모자, 사냥터 임시 아이템 |
| 빛나는 상자 | `rare` | 120 | 스킨, 설계도, 칭호 |
| 챔피언 상자 | `epic` | 250 | 챔피언 모자 변형, 고급 설계도 |
| 전설 상자 | `legendary` | 500 | 희귀 스킨, 전용 연출, 고급 칭호 |

MVP는 `common`, `uncommon`, `rare` 3등급만 구현합니다.

## 9. 열쇠 조각 획득

사냥터 전투 파편은 층 클리어·정예·챔피언 보너스로 정산하지 않습니다. 일반 몬스터가 전장에서 실제로 떨어뜨리고 회수한 파편과 파편 꾸러미만 `pendingLoot`에 들어갑니다. 이벤트, 상자 개봉, 상점 등 전투 밖 보상 경로는 기존 규칙을 유지합니다.

## 10. 패배와 귀환

귀환하면 모든 pendingLoot를 securedLoot로 옮기고 프로필에 저장합니다.

### 10.1 상자 파손 (연쇄 확률)

패배 시 pendingLoot의 상자가 연쇄적으로 파손됩니다. 각 상자는 등급에 따라 가중치가 부여되며, 가중치가 높을수록 파손될 확률이 높습니다.

```text
파손 가중치:
  common:   1
  uncommon: 2
  rare:     3
  epic:     4
  legendary: 5

파손 확률 (연쇄):
  첫 번째 파손: 100% (가중치 랜덤 선택, 높은 등급이 더 잘 깨짐)
  두 번째 파손: 50%
  세 번째 파손: 25%
  네 번째 파손: 12.5%
  ...
  (매 단계 50%씩 감소)
```

의사 코드:

```text
candidates = pendingLoot.chests shuffled
sort by weight descending (pick order)
index = 0
probability = 1.0
while probability > random() AND index < candidates.length:
    destroyed = candidates[index]
    remove destroyed from pendingLoot
    index += 1
    probability *= 0.5
```

상자가 3개(common, uncommon, rare)이고, rare가 파손된 후 50% 확률로 uncommon, 다시 25% 확률로 common이 파손될 수 있습니다. 운이 나쁘면 모든 상자를 잃을 수도 있습니다.

### 10.2 열쇠 조각 보존

```text
열쇠 조각: 50% 보존 (반올림)
```

### 10.3 XP 보존

```text
캐릭터 XP: 70% 지급
```

매치 XP의 70%는 패배해도 지급됩니다. 이로 인해 레벨 자체는 패배해도 천천히 오릅니다.

## 12. 설계도 방식

상자에서 완성 보상 대신 설계도를 얻고, 열쇠 조각으로 확정 해금하는 구조를 권장합니다.

```text
빛나는 상자
  -> "흡혈 배지 설계도" 발견
  -> 보관함에서 열쇠 조각 150개로 해금
  -> 이후 사냥터 보상 선택지에 흡혈 배지 등장
```

설계도 발견은 랜덤, 해금은 확정 재화 지불입니다.

### 13.1 파편 상점

장비 탭 우측 상단의 `상점` 버튼은 컬렉션을 닫지 않는 파편 상점 팝업을 엽니다. 상점은 사냥터 보상으로 쌓인 파편의 반복 소모처이며, 일반 사냥터 상자보다 낮은 품질 풀(common 70%, uncommon 25%, rare 5%)만 제안합니다.

- 상자 가격: 파편 150
- 구매 제한: 2개, 구매가 한 번이라도 있으면 마지막 성공 구매 시각부터 6시간 후 초기화
- 상자 리롤: 파편 30부터 매회 30씩 올라 최대 300(기본 비용의 10배), 마지막 성공 리롤 시각부터 1시간 30분 후 비용 단계 초기화
- 구매와 리롤은 각각 실제로 사용한 뒤에만 남은 시간을 표시합니다. 구매에 성공하면 지급한 상자는 기존 제안 등급을 유지하고, 다음 제안만 같은 확률표로 즉시 자동 리롤합니다. 이 자동 리롤은 수동 리롤 횟수·비용·초기화 타이머에 영향을 주지 않으며, 수동 리롤은 제안 중인 상자의 등급을 즉시 다시 뽑습니다.
- 제안 상자는 기존 `<chest-icon>`의 common(회색), uncommon(초록), rare(금색) 표현을 재사용합니다. 팝업에서 동적으로 생성되는 상자에는 `x-component="'chest-icon'"`을 명시해 템플릿 마운트를 보장합니다. 리롤 시 등급이 같아도 상자 자체가 튀어 오르고 뚜껑이 열렸다 닫히는 교체 효과를 재생합니다.
- 리롤이 성공하면 세 음이 짧게 올라가는 효과음을 재생합니다. 파편이 부족해 리롤하지 못한 클릭에는 재생하지 않습니다.
- 상점에서 구매한 상자는 보관함에 추가되고, 상점 팝업은 닫히지 않습니다.

상점 상품은 상자 전용 분기가 아니라 상자·장비·소모품을 등록하는 목록으로 구성합니다. 현재 소모품 영역은 아래 두 행을 제공하며, 최대 보유량 또는 최대 사용 한도에서는 해당 버튼을 비활성화합니다.

- `HP 물약`: `보유 n/10 · 구매 100`
- `원정 물약 사용 한도`: `n/5 · +1 확장` (100 → 200 → 400 → 800)

현재 HP 물약은 이 원정 밖 파편 상점에서만 보충합니다. 방랑 상인 이벤트는 이후 같은 상품 정의를 사용해 HP 물약을 판매할 수 있도록 열어 두되, 현재 상품 풀에는 넣지 않습니다.

## 16. MVP 구현 범위

1. 사냥터 입장 버튼 (메인 메뉴)
2. 캐릭터 선택 UI (한 번이라도 우승한 캐릭터만 표시)
3. 최대 5층 1v1 연속 전투, HP 층간 누적
4. 승리 후 귀환/전진 선택
5. 층별 적 강화
6. 랜덤 이벤트 3종: 상자방, 휴식지, 저주받은 제단
7. 열쇠 조각 지급과 보관 (패배 시 50% 보존)
8. 상자 3등급: common/uncommon/rare
9. 상자 파손 연쇄 확률 (100%→50%→25%→...)
10. 보관함 탭에서 상자 개봉
11. 사냥터 매치 XP 지급 (stageMultiplier=1.0 고정)

**MVP 제외** (별도 문서, 이후 구현):
- 장비 시스템 ([`docs/equipment-system.md`](equipment-system.md))
- 사냥터 전용 임시 아이템
- 동시 다수 전투 / n 대 n
- 상인 이벤트
- 설계도 시스템
- 챔피언 난입

## 17. 필수 회귀 조건

- 토너먼트 결과와 사냥터 결과가 서로의 진행 상태를 오염하지 않아야 합니다.
- 사냥터 입장 시 우승 기록이 없는 캐릭터는 선택 UI에 표시되지 않아야 합니다.
- 귀환 시 pendingLoot가 프로필에 정확히 저장되어야 합니다.
- 패배 시 상자 파손 확률이 연쇄적으로 정확히 적용되어야 합니다.
- 패배 시 파손 가중치가 등급별로 달라야 합니다(높은 등급일수록 먼저 파손).
- 패배 시 열쇠 조각 50% 보존이 정확히 계산되어야 합니다.
- 패배 시 XP 70% 지급이 한 번만 적용되어야 합니다.
- 사냥터 HP는 층간 리셋되지 않아야 합니다 (이벤트 회복만).
- HP가 0이 되면 사냥터가 즉시 패배 처리되어야 합니다.
- 귀환 후에도 프로필 HP는 사냥터 시작 전 상태로 복원되어야 합니다(사냥터 HP는 전투 중에만 누적).
- 같은 상자 ID를 두 번 개봉할 수 없어야 합니다.
- 열쇠 조각이 부족하면 상자를 열 수 없어야 합니다.
- 상자 개봉 비용은 등급별로 달라야 합니다.
- 사냥터 전투에서 같은 팀끼리는 피해를 주지 않아야 합니다.
- 사냥터 전투에서 적 팀이 모두 패배하면 조우 승리로 처리해야 합니다.
- 플레이어 팀이 모두 패배하면 사냥터 패배로 처리해야 합니다.
- 사냥터 매치 XP는 `stageMultiplier=1.0`으로 계산되어야 합니다.
- 사냥터 XP도 `processedReportIds` 중복 방지 대상에 포함되어야 합니다.
- 모바일 보관함 UI에서 상자 카드와 개봉 버튼이 화면 밖으로 잘리지 않아야 합니다.

## 20. 2026-07-04 추가 구현 — 상자 보상 테이블과 이벤트 확장

사냥터 상자는 더 이상 등급/id만 가진 더미 객체가 아닙니다. `createHuntingChest()`는 등급별 개봉 비용, 보상 테이블 버전, 미리보기 문자열을 함께 생성합니다.

### 상자 보상 테이블

구현 파일:

- `src/hunting/huntingConfig.js`: 상자 5등급, 비용, 보상 타입 정의
- `src/hunting/huntingRewards.js`: 등급별 보상 테이블, 보상 미리보기, 보상 롤링
- `src/hunting/chestRewards.js`: 보관함 개봉 가능 여부, 실제 개봉, 즉시 적용 가능한 보상 반영

보상 타입:

| 타입 | 적용 |
|---|---|
| `key_shards` | 개봉 즉시 `profile.hunting.keyShards`에 반영 |
| `instant_heal` | 후속 런 적용용 deferred effect로 반환 |
| `temporary_stat` | 후속 런 적용용 deferred effect로 반환 |

현재 MVP에서는 보관함에서 즉시 적용 가능한 해조각 보상만 프로필에 직접 반영합니다. HP 회복과 임시 스탯 증가는 런 시작/진행 UI에 연결할 후속 payload로 반환합니다. 새 저장 스키마를 성급히 늘리지 않기 위한 선택입니다.

### 이벤트 확장

`src/hunting/huntingEncounters.js`의 이벤트 풀은 아래 타입을 포함합니다.

| 이벤트 | 효과 |
|---|---|
| `rest_site` | carried HP를 최대 HP의 25%만큼 회복하고 결과 확인 뒤 전진 |
| `chest_room` | 층수 기반 희귀도 상자를 pending loot에 추가하고 상자 전용 화면 표시 |
| `cursed_altar` | 다음 1~3회 전투에 적용되는 능력치 교환 효과를 표시한 뒤 전진 |
| `champion_intrusion` | 강화 적을 전투 준비 화면에서 표시하고 `전투 시작` 뒤 전투 진입 |

저주받은 제단은 `applyHuntingCursedAltar()`를 통해 `run.statModifiers`에 gain/loss modifier를 추가합니다. 전투 클리어 시 `recordHuntingFloorResult()`가 modifier 지속 층수를 소모합니다. 상자방처럼 전투가 아닌 이벤트 보상은 `consumeStatModifiers: false`를 사용해 지속 층수를 소모하지 않습니다.

챔피언 난입은 `HUNTING_ENEMY_TYPES.CHAMPION`과 `HUNTING_SCALING.CHAMPION_POWER_BONUS`를 사용합니다. `HuntingManager`는 해당 이벤트가 걸린 층의 적을 champion으로 스케일합니다. 챔피언 본체는 일반 몬스터 전리품 대상에서 제외되지만 강화석을 드롭하고, 동반 일반 몬스터가 남긴 전리품도 함께 회수할 수 있습니다.
