# 경험치 및 캐릭터 레벨 시스템

> 현재 책임 기준: 레벨은 해당 캐릭터 자신의 기본 수치와 대표 행동을 강화한다. 캐릭터 간 지원 효과는 마스터리, 계정 수집 보상은 업적이 소유한다. 상세 기준은 [성장 책임 기준](progression-responsibilities.md)을 따른다.

> 상태: 설계 초안, 구현 전
> 기준 코드: 2026-06-28 `main`
> 관련 문서: [`meta-progression-system.md`](meta-progression-system.md), [`collection-achievements-system.md`](collection-achievements-system.md), [`character-mastery-system.md`](character-mastery-system.md), [`collection-hub-ui.md`](collection-hub-ui.md), [`player-data-storage-security.md`](player-data-storage-security.md)

## 1. 시스템 정의

경험치 시스템은 매 경기 또는 토너먼트 종료 직후 플레이어에게 즉각적인 성장 보상을 제공하는 장치입니다.

현재 성장 구조는 업적과 숙련도 중심이라 “특정 목표를 달성했을 때” 보상이 크게 발생합니다. 경험치는 이와 달리 패배해도 쌓이고, 작은 행동도 보상으로 환산되어 한 판 한 판이 남는 느낌을 만듭니다.

- 업적은 장기 목표입니다.
- 숙련도는 우승 기록 기반의 캐릭터/계정 수집 성장입니다.
- 경험치는 플레이 직후 확정 지급되는 단기 보상입니다.
- 경험치 레벨 보상은 작고 자주 주되, 전투 밸런스를 무너뜨리지 않게 상한을 둡니다.
- 플레이어 성장으로 게임이 지나치게 쉬워지지 않도록 상대도 파생 성장 보정을 받습니다.

## 2. 설계 원칙

### 2.1 패배해도 성장

플레이어가 패배해도 참가 경험치를 지급합니다. 패배한 판도 “아무것도 얻지 못했다”로 끝나지 않아야 합니다.

### 2.2 즉시 보상 우선

경험치는 경기 결과 화면에서 바로 보여줍니다. 컬렉션 허브에서만 확인 가능한 숨은 수치가 되면 목적을 잃습니다.

### 2.3 캐릭터별 개성 강화

레벨업 보상은 전 캐릭터 공통 스탯 상승만 반복하지 않습니다. 각 캐릭터의 대표 행동이 XP 지급 이유와 레벨 보상에 반영되어야 합니다.

### 2.4 영구 성장은 작고 유한하게

레벨 보상은 같은 캐릭터를 플레이할 때만 적용되는 작은 보정으로 시작합니다. 계정 전체를 강하게 만드는 효과는 기존 메타 성장/업적 시스템의 책임으로 둡니다.

### 2.5 중복 처리 방지

같은 MatchReport 또는 TournamentReport를 다시 처리해도 XP가 두 번 지급되면 안 됩니다. 기존 `processedTournamentReportIds`와 같은 중복 방지 패턴을 재사용합니다.

### 2.6 성장 보상감과 난이도 유지의 균형

플레이어 레벨 보상은 체감되어야 하지만, 레벨이 오를수록 토너먼트가 자동 승리 구간이 되면 안 됩니다.

상대 성장은 플레이어의 레벨을 완전히 무효화하지 않는 보정이어야 합니다. 플레이어가 성장하면 여전히 조금 유리해지되, 상대도 일부 따라와서 경기 긴장감을 유지합니다.

## 3. 용어

| UI 용어 | 코드 용어 | 의미 |
| --- | --- | --- |
| 경험치 | `xp` | 다음 레벨까지 쌓이는 진행도 |
| 누적 경험치 | `totalXp` | 캐릭터가 지금까지 얻은 전체 XP |
| 캐릭터 레벨 | `experienceLevel` | 캐릭터별 XP로 계산되는 레벨 |
| 계정 경험치 | `accountXp` | 선택 사항. 모든 플레이를 합산하는 장기 진행도 |
| 대표 행동 XP | `signatureXp` | 캐릭터 고유 행동으로 얻는 보너스 XP |
| 상대 성장 보정 | `rivalExperienceScaling` | 플레이어 레벨에 따라 상대에게 적용되는 파생 난이도 보정 |
| 디펜딩 챔피언 | `defendingChampion` | 직전 토너먼트에서 우승한 AI 캐릭터 표시/현상금 대상 |
| 현상금 | `bountyXp` | 디펜딩 챔피언을 플레이어가 격파했을 때 지급하는 보너스 XP |

`masteryLevel`과 `experienceLevel`은 다른 개념입니다. `masteryLevel`은 우승 횟수 기반 숙련도이고, `experienceLevel`은 경기마다 쌓이는 XP 기반 레벨입니다.

## 4. MVP 범위

1차 구현은 다음 범위로 제한합니다.

1. 캐릭터별 `totalXp`와 `experienceLevel` 저장
2. 매치 종료 후 XP 전용 결과 패널 표시
3. 참가, 승리, 라운드 진출, 대표 행동 기반 XP 지급
4. 레벨업 시 작은 캐릭터 전용 보너스 적용
5. 플레이어 레벨에 따른 상대 성장 보정 적용
6. AI 우승자를 디펜딩 챔피언으로 표시하고 격파 보상 지급
7. 컬렉션 허브 도감 카드에 레벨과 다음 레벨 진행도 표시

계정 경험치, 스킨, 칭호, 레벨업 연출 강화는 2차 범위로 둡니다.

## 5. XP 지급 규칙

유저가 직접 선택한 캐릭터만 XP를 받습니다. AI끼리 진행된 매치와 플레이어가 참가하지 않은 매치는 XP 대상이 아닙니다.

XP는 고정 테이블 대신 **매치 성과 기반 공식**으로 계산합니다. 같은 승리라도 더 인상적인 플레이(많은 딜, 낮은 HP 생존, 역전)가 더 많은 XP를 줍니다.

### 5.1 매치 XP 공식

```
matchXP = (dealRatio + hpRemainRatio + comebackBonus) × stageMultiplier × XP_SCALE
```

| 변수 | 계산 | 범위 |
| --- | --- | ---: |
| `dealRatio` | `내가상대에게입힌피해 ÷ 상대최대HP` | 0~2+ |
| `hpRemainRatio` | `매치종료시내잔여HP ÷ 내최대HP` | 0~1 |
| `comebackBonus` | 승리 시 `max(0, (0.3 - minHpRatio) × 0.5)` | 0~0.15 |
| `XP_SCALE` | 20 | 고정 |
| `stageMultiplier` | 라운드별 배율 (아래 표) | 1.0~3.5 |

**comebackBonus 상세**: 매치 중 플레이어 HP가 30% 이하로 떨어진 구간이 있으면, 가장 낮았던 HP 비율과의 차이에 0.5를 곱합니다. HP 10%까지 떨어졌다가 이기면 `(0.3 - 0.1) × 0.5 = 0.1` 보너스. 패배 시 comebackBonus는 항상 0입니다.

#### stageMultiplier

| 단계 | 배율 |
| --- | ---: |
| 1라운드 (8강) | ×1.0 |
| 2라운드 (4강) | ×1.2 |
| 3라운드 (결승) | ×2.5 |
| 우승 보너스 | +1.0 (결승 배율에 추가) |

예시: 8인 토너먼트, 결승 진출 후 우승

| 매치 | dealRatio | hpRemainRatio | comeback | stageMul | XP |
| --- | ---: | ---: | ---: | ---: | ---: |
| 8강: 0.6딜 + 0.4생존 | 0.6 | 0.4 | 0 | 1.0 | 20 |
| 4강: 0.7딜 + 0.3생존 | 0.7 | 0.3 | 0 | 1.2 | 24 |
| 결승: 0.8딜 + 0.15생존 + 역전 | 0.8 | 0.15 | 0.1 | 3.5(2.5+1.0) | 74 |
| **토너먼트 총 XP** | | | | | **118** |

### 5.2 패배 시 XP

패배해도 위 공식이 동일하게 적용됩니다. `comebackBonus=0`, `stageMultiplier`는 현재 라운드까지 유지. 우승 보너스(+1.0)만 없습니다.

1라운드 패배: `(0.4 + 0 + 0) × 1.0 × 20 = 8 XP`
결승 패배: `(0.6 + 0 + 0) × 2.5 × 20 = 30 XP`

### 5.3 보정

모든 매치 XP 계산에는 다음 규칙이 추가로 적용됩니다.

- `dealRatio`는 상대 최대 HP를 초과할 수 있습니다(오버킬). 최대 2.0으로 클램프합니다.
- `hpRemainRatio`는 0 이상으로 클램프합니다(패배 시 0).
- 매치 종료 후 `BattleSimulation`의 `MatchReport`에서 `damageDealt`, `damageTaken`, `minHpRatio`(매치 중 최저 HP 비율)를 수집합니다.
- 이 값들은 `hooks.onDamageDealt`/`onDamageTaken`을 통해 실시간으로 추적합니다.
- XP 계산과 지급은 유저가 참가한 매치 종료 단계(`grantExperienceFromMatchReport`)가 소유합니다. 토너먼트 종료 단계에서는 업적/숙련도/도전 단계만 처리하고 XP를 다시 지급하지 않습니다.

## 6. 레벨 곡선

MVP는 10레벨 상한으로 시작합니다. 레벨당 필요 XP는 이전 레벨 × 1.35로 기하급수적으로 증가합니다. 토너먼트 1회 우승(약 110XP)으로 간신히 레벨 2에 도달할 수 있는 수준입니다.

| 레벨 | 구간 필요 XP | 누적 XP |
| ---: | ---: | ---: |
| 1 | 0 | 0 |
| 2 | 100 | 100 |
| 3 | 135 | 235 |
| 4 | 182 | 417 |
| 5 | 246 | 663 |
| 6 | 332 | 995 |
| 7 | 448 | 1,443 |
| 8 | 605 | 2,048 |
| 9 | 817 | 2,865 |
| 10 | 1,103 | 3,968 |

상한 이후 XP는 계속 기록할 수 있지만 전투 보상은 늘리지 않습니다. UI에는 `Lv.10 MAX`로 표시합니다.

`totalXp`에서 레벨을 계산할 때는 누적 XP 테이블을 이진 탐색합니다. 레벨 곡선을 변경해도 저장된 `totalXp`에서 재계산됩니다.

## 7. 레벨 보상

레벨 보상은 캐릭터별 대표 행동 강화로만 적용합니다. 다른 캐릭터를 플레이할 때 적용되는 효과는 숙련도/연계 시스템의 책임입니다.

직접 전투 보상 구간은 Lv.3, Lv.6, Lv.9뿐입니다. 각 구간은 `ability_modifier`로 해당 Ability에 전달하며, 수치와 충돌·투사체 판정의 최종 기준은 [보상 밸런스 설정](reward-balance.md#레벨-보상-대표-행동-강화)에서 관리합니다. 나머지 레벨은 장비 요구 레벨과 도전 관문 진행도만 유지합니다.

전투 보정은 `collectActiveExperienceEffects(profile, fighterId)` 같은 계산 함수에서 파생합니다. 레벨업 순간에 스탯 수치를 프로필에 직접 더하지 않습니다.

## 8. 상대 성장 보정

플레이어 캐릭터 레벨이 오르면 해당 토너먼트의 상대도 일부 성장합니다. 이 보정은 상대 프로필에 저장하지 않고, 토너먼트 생성 시 현재 플레이어 캐릭터 레벨에서 파생합니다.

### 8.1 적용 원칙

- 상대 성장 보정은 플레이어가 직접 참가하는 토너먼트에만 적용합니다.
- AI끼리 진행하는 경기에서도 같은 토너먼트 안의 상대 spec은 이미 생성된 값을 사용하되, XP나 레벨을 따로 얻지는 않습니다.
- 보정 대상은 `hp`, `damage`, `defense`로 제한합니다.
- `speed`, `skill`, `radius`, `mass`는 상대 성장으로 변경하지 않습니다. 물리 안정성과 전투 리듬이 흔들리기 때문입니다.
- 도전 단계 보정과 중첩되지만 별도 상한을 둡니다.
- UI에는 “상대 성장 보정 +N%”처럼 선택한 토너먼트 난이도 설명에 표시합니다.

### 8.2 권장 공식

MVP에서는 플레이어 캐릭터 레벨만 사용합니다.

```text
rivalExperienceScaling =
    min((playerExperienceLevel - 1) * 0.008, 0.06)
```

예시:

| 플레이어 레벨 | 상대 성장 보정 |
| ---: | ---: |
| 1 | +0.0% |
| 3 | +1.6% |
| 5 | +3.2% |
| 8 | +5.6% |
| 10 | +6.0% |

상대 보정은 플레이어 레벨 보상의 약 60~70% 정도만 따라오게 조정합니다. 이렇게 하면 플레이어는 성장한 만큼 조금 강해지지만, 레벨만으로 전투가 무너질 정도의 격차는 생기지 않습니다.

### 8.3 도전 단계와의 관계

도전 단계는 명시적으로 더 어려운 토너먼트를 선택하는 시스템입니다. 상대 성장 보정은 플레이어 레벨 때문에 기본 난이도가 붕괴하지 않게 하는 자동 보정입니다.

적용 순서:

```text
base rival spec
  -> challengeLevel 보정
  -> rivalExperienceScaling 보정
  -> tournament roster 고정
```

두 보정은 모두 토너먼트 시작 시 한 번만 계산합니다. 토너먼트 진행 중 레벨업이 발생해도 현재 토너먼트 상대에게는 소급 적용하지 않습니다.

### 8.4 확장안

2차 구현에서는 계정 전체 최고 레벨이나 평균 레벨을 일부 반영할 수 있습니다.

```text
effectivePressureLevel =
    max(selectedCharacterLevel, floor(accountTopCharacterLevel * 0.5))
```

다만 계정 레벨을 강하게 반영하면 새 캐릭터를 키울 때 초반이 답답해질 수 있습니다. MVP에서는 선택 캐릭터 레벨만 사용합니다.

## 9. 디펜딩 챔피언과 현상금

AI 우승 캐릭터를 영구 성장시키지는 않습니다. 대신 직전 토너먼트의 AI 우승자를 디펜딩 챔피언으로 표시하고, 플레이어가 해당 캐릭터를 격파하면 보너스 XP를 지급합니다.

이 시스템의 목적은 AI 우승에 기억과 목표를 부여하는 것입니다. 전투력 누적 성장은 제한하거나 1회성으로만 사용합니다.

### 9.1 챔피언 등록

토너먼트 종료 시 우승자가 플레이어가 아니면 해당 캐릭터를 디펜딩 챔피언으로 등록합니다.

```js
defendingChampion: {
    fighterId: "rage",
    sinceTournamentId: "uuid",
    title: "디펜딩 챔피언",
    streak: 1,
    expiresAfterTournament: false
}
```

같은 AI 캐릭터가 연속 우승하면 `streak`를 1 증가시킵니다. 다른 AI가 우승하면 챔피언을 교체하고 `streak`를 1로 초기화합니다. 플레이어가 우승하면 디펜딩 챔피언은 유지하되 `streak`는 증가하지 않습니다.

### 9.2 현상금 지급

플레이어가 디펜딩 챔피언을 직접 격파하면 보너스 XP를 지급합니다.

```text
bountyXp = min(10 + defendingChampion.streak * 5, 30)
```

예시:

| 챔피언 연승 | 격파 보너스 |
| ---: | ---: |
| 1 | +15 XP |
| 2 | +20 XP |
| 3 | +25 XP |
| 4 이상 | +30 XP |

현상금은 해당 매치에서 플레이어가 직접 챔피언을 이겼을 때만 지급합니다. AI가 챔피언을 탈락시키면 현상금은 지급하지 않고, 토너먼트 종료 후 새 우승자를 기준으로 챔피언 상태를 갱신합니다.

### 9.3 1회성 챔피언 버프

디펜딩 챔피언에게 전투 보정을 준다면 영구 누적이 아니라 다음 토너먼트의 해당 캐릭터 등장분에만 적용합니다.

```text
championAura =
    min(0.01 + defendingChampion.streak * 0.005, 0.025)
```

적용 대상:

- `hp`
- `damage`
- `defense`

제외 대상:

- `speed`
- `skill`
- `radius`
- `mass`

이 버프는 챔피언이 플레이어에게 패배하면 즉시 제거합니다. 플레이어와 만나지 않은 채 토너먼트가 끝나면 새 우승 결과에 따라 유지, 교체, 또는 streak 갱신을 처리합니다.

### 9.4 승률 편향 감쇠

AI 우승 결과에 따라 어떤 성장값이나 챔피언 버프를 주는 경우, 최근에 이미 강한 캐릭터가 더 강해지는 양의 피드백을 막아야 합니다.

MVP에서는 챔피언 버프 자체가 작고 1회성이므로 감쇠는 선택 사항입니다. 다만 구현 시 승률 데이터가 있으면 다음 공식을 사용합니다.

```text
growthDampener =
    clamp(1 - max(0, recentWinRate - baselineWinRate) * 2, 0.35, 1)

effectiveChampionAura =
    championAura * growthDampener
```

기준값:

```text
baselineWinRate = 1 / rosterSize
recentWinRate = 최근 N개 토너먼트에서 해당 캐릭터 우승 비율
```

예시: 8인 토너먼트 기준 기대 우승률은 12.5%입니다. 최근 우승률이 30%인 캐릭터는 `1 - (0.30 - 0.125) * 2 = 0.65` 감쇠를 받아 챔피언 버프가 65%만 적용됩니다.

이 감쇠는 “강한 캐릭터를 벌준다”가 아니라 “이미 많이 이기는 캐릭터가 시스템 보너스로 더 치고 나가는 것을 막는다”는 목적입니다.

### 9.5 전장 시각 표시

디펜딩 챔피언은 전장 안에서 즉시 구분되어야 합니다. 텍스트 로그만으로는 전투 중 목표가 충분히 살아나지 않습니다.

기본 표식은 **챔피언 모자**입니다.

- 챔피언 볼 머리 위에 작은 금색 모자를 그립니다.
- 모자는 물리 충돌 판정에 포함하지 않습니다.
- 모자는 볼 회전과 함께 돌지 않고 화면 기준으로 항상 똑바로 표시합니다.
- 모자는 fighter보다 위, floating text보다 아래 레이어에 그립니다.
- 모자 크기는 `ball.radius`에 비례하되 최소/최대 크기를 둡니다.
- 색상만으로 구분하지 않도록 검은 외곽선, 작은 보석, 카드/대진표 텍스트 표식을 함께 사용합니다.

권장 canvas 구성:

```text
hatWidth = clamp(ball.radius * 1.25, 14, 26)
hatHeight = clamp(ball.radius * 0.55, 7, 13)
hatY = ball.position.y - ball.radius - hatHeight * 0.6
```

시각 형태:

- 아래쪽 얇은 챙
- 위쪽 사다리꼴 crown
- 중앙 작은 붉은 보석
- 1px 또는 2px 어두운 외곽선

전투 중 챔피언이 플레이어에게 패배하면 모자는 즉시 사라져도 되고, KO 연출이 끝날 때까지 남겨도 됩니다. 단, 다음 라운드 spec에는 더 이상 챔피언 버프가 적용되지 않아야 합니다.

### 9.6 UI 표시

```text
Rage Ball
디펜딩 챔피언 · 현상금 +20 XP
```

전투 시작 로그:

```text
디펜딩 챔피언 Rage Ball 등장! 격파 시 +20 XP
```

격파 성공 결과:

```text
챔피언 격파 +20 XP
```

표시는 강하게 하되, 챔피언 버프 수치가 작다면 UI에서 수치보다 현상금과 타이틀을 우선 보여줍니다.

### 9.7 PC UI 설계

PC에서는 오른쪽 정보 패널과 대진표를 활용해 챔피언 정보를 넉넉하게 보여줍니다.

#### 토너먼트 시작 전

대진표 상단에 챔피언 배너를 표시합니다.

```text
┌────────────────────────────────────┐
│ 디펜딩 챔피언                      │
│ [모자] Rage Ball   현상금 +20 XP   │
│ 연승 2 · 챔피언 보정 +1.5%         │
└────────────────────────────────────┘
```

원칙:

- 배너는 대진표 위에 한 번만 표시합니다.
- 챔피언이 현재 토너먼트에 없으면 배너를 표시하지 않습니다.
- 챔피언 보정을 사용하지 않는 설정이면 `챔피언 보정` 문구를 숨깁니다.
- 현상금은 항상 표시합니다.

#### 대진표

챔피언이 포함된 bracket row는 다음 세 요소를 가집니다.

- 이름 앞 모자 아이콘
- 행 왼쪽 3px 금색 강조선
- 오른쪽 `+20 XP` 현상금 chip

예시:

```text
│ [모자] Rage Ball              +20 XP │
```

행 높이는 기존 bracket row와 같게 유지합니다. chip 때문에 행이 커져 대진표가 밀리면 안 됩니다.

#### 전투 HUD

플레이어가 챔피언과 매치할 때만 경기장 상단 중앙에 작은 HUD pill을 표시합니다.

```text
[모자] 디펜딩 챔피언 Rage Ball · 격파 +20 XP
```

원칙:

- 경기장 내부 오브젝트를 가리지 않도록 상단 안전 여백에 배치합니다.
- 전투 시작 후 3초가 지나면 반투명 상태로 줄이거나 숨깁니다.
- 전투 로그에도 같은 내용을 한 줄 기록합니다.

#### 결과 팝업

챔피언 격파 성공 시 XP 요약 안에 별도 줄을 표시하고, 상단 결과 문구에 짧은 타이틀을 추가합니다.

```text
챔피언 격파!
Vampire Ball +92 XP

토너먼트 참가 +10
...
챔피언 격파 +20
```

### 9.8 모바일 UI 설계

모바일에서는 가로 공간이 좁으므로 대진표 행을 크게 늘리지 않습니다. 챔피언 정보는 전투장 아래 또는 하단 패널 상단에 compact strip으로 표시합니다.

#### 토너먼트 시작 전

스탯/도전 단계 팝업 하단에 한 줄 champion strip을 추가합니다.

```text
[모자] Rage Ball · 현상금 +20 XP
```

원칙:

- strip은 한 줄 높이로 유지합니다.
- 긴 캐릭터명은 말줄임 처리합니다.
- 현상금 수치는 오른쪽에 고정합니다.
- 페이지 전체 스크롤을 만들지 않고 팝업 내부 스크롤에 포함합니다.

#### 대진표

모바일 bracket row는 아이콘과 짧은 chip만 표시합니다.

```text
[모자] Rage       +20
```

`디펜딩 챔피언` 전체 문구는 row 안에 넣지 않습니다. row를 탭하거나 길게 누를 수 있는 UI가 생기면 상세 tooltip/bottom sheet에서 설명합니다.

#### 전투 HUD

모바일 전투 중에는 경기장 위에 긴 pill을 올리지 않습니다. 대신 경기장 바로 아래 상태 영역 첫 줄에 compact strip을 표시합니다.

```text
[모자] 챔피언 · 격파 +20 XP
```

원칙:

- 경기장 canvas를 가리지 않습니다.
- 하단 액션/상태 버튼과 겹치지 않습니다.
- 세로 공간이 부족하면 전투 시작 3초 뒤 `챔피언 +20`으로 축약합니다.
- 모자 자체는 canvas 안에서 계속 표시하므로 strip이 축약되어도 목표 식별은 유지됩니다.

#### 결과 팝업

모바일 결과 팝업은 XP 항목을 3~5줄로 제한합니다. 챔피언 격파는 일반 대표 행동 XP보다 우선 표시합니다.

```text
+92 XP
챔피언 격파 +20
매치 승리 2회 +24
흡혈 7회 +14
```

PC/모바일 레이아웃은 같은 필드를 사용하고 표시 밀도만 다르게 가져갑니다.

## 10. 플레이어 프로필

권장 저장 구조:

```js
{
    experience: {
        currentXp: 0, // 캐릭터별 XP 합계/레거시 호환용
        byCharacter: {
            vampire: {
                currentXp: 0
            }
        },
        processedReportIds: [],
        defendingChampion: {
            fighterId: null,
            sinceTournamentId: null,
            streak: 0
        }
    }
}
```

`experienceLevel`, 다음 레벨까지 필요한 XP, 적용 중인 보상은 저장하지 않고 캐릭터별 `currentXp`와 정의 테이블에서 계산합니다. 이렇게 해야 레벨 곡선이나 보상 수치를 조정했을 때 저장 데이터를 마이그레이션하지 않고 재계산할 수 있습니다.

레거시 세이브에 전역 `experience.currentXp`만 있고 `byCharacter`가 비어 있으면, 최초 로드 시 최근 플레이 기록이 있는 캐릭터로 한 번 귀속합니다. 기록이 없으면 현재 선택 캐릭터로 귀속합니다. 이후 지급되는 XP는 항상 `byCharacter[characterId].currentXp`에 쌓입니다.

중복 방지는 `processedReportIds` 배열 대신 `Set<tournamentId + ":" + characterId>` 복합 키로 체크합니다. 64개 제한이 없으므로 오래된 보고서를 버리지 않아도 중복 지급이 발생하지 않습니다. 이미 처리된 조합이 들어오면 XP를 지급하지 않고 `alreadyProcessed: true`를 반환합니다.

상대 성장 보정은 저장하지 않습니다. 매 토너먼트 시작 시 선택 캐릭터의 `totalXp`에서 레벨을 계산하고, 그 레벨로 보정값을 파생합니다.

디펜딩 챔피언은 저장합니다. 단, 챔피언의 영구 스탯이나 XP는 저장하지 않습니다.

## 11. 결과 화면 UX

### 11.1 매치 종료 XP 결과 패널

매치 종료 직후 결과 오버레이 안에 XP 전용 패널을 표시합니다. 패널은 유저 입력 없이 현재 흐름처럼 자동으로 다음 경기로 넘어가며, 표시되는 동안 XP 바가 실제 획득량만큼 차오릅니다.

```text
Dash Ball
+32 XP

Lv.2
[■■■■■■□□□□] 65/135 XP
다음 레벨까지 70XP
다음 보상 · 대표 행동 강화 I
```

패배 시:

```text
+8 XP
Lv.1
[■□□□□□□□□□] 8/100 XP
다음 레벨까지 92XP
```

레벨업 시에는 같은 패널에서 `LEVEL UP` 배지를 표시하고, 바가 이전 레벨 끝까지 찬 뒤 새 레벨 진행도로 이어지는 짧은 애니메이션을 사용합니다.

### 11.2 초기 캐릭터 화면 XP 미터

초기 `내 캐릭터` 패널에는 선택된 캐릭터의 캐릭터별 XP를 표시합니다.

```text
Lv.3 · 235 XP
[■■□□□□□□□□] 0/182 XP
다음 레벨까지 182XP
다음 보상 · Lv.6 · 대표 행동 강화 II
```

### 11.3 도감 XP 표시

컬렉션 허브 도감 카드에는 `Lv.N · NXP`를 작은 chip으로 표시하고, 도감 상세에는 레벨, 누적 XP, 진행 바, 다음 레벨까지 남은 XP, 다음 보상 한 줄을 표시합니다.

표시 원칙:

- 총 XP 증가량을 가장 먼저 보여줍니다.
- 결과 패널은 자동 진행을 막지 않습니다.
- 패배 시에도 XP 패널을 같은 위치에 보여줍니다.
- 디펜딩 챔피언을 격파했다면 기본 XP와 분리된 보상 줄로 보여줍니다.
- 컬렉션 허브에는 캐릭터별 `Lv.N`, 진행 바, 다음 보상 한 줄을 표시합니다.

## 12. 프로젝트 구조

```text
src/
  experience/
    experienceConfig.js
    experienceState.js
    experienceRewards.js
    experienceScaling.js
    championState.js
    index.js
```

| 파일 | 책임 |
| --- | --- |
| `experienceConfig.js` | XP 지급량, 레벨 곡선, 대표 행동 정의 |
| `experienceState.js` | XP 지급, 중복 처리 방지, 레벨 계산 |
| `experienceRewards.js` | 레벨 보상 파생 계산 |
| `experienceScaling.js` | 플레이어 레벨 기반 상대 성장 보정 계산 |
| `championState.js` | 디펜딩 챔피언 등록, 현상금, 1회성 챔피언 버프 계산 |
| `ui.js` | 챔피언 배너, strip, 결과 XP 패널/캐릭터 XP 미터 상태 연결 |
| `styles.css` | PC/모바일 챔피언 표시 반응형 스타일 |
| `playerProfile.js` | 기본값과 마이그레이션 |
| `collectionViewModel.js` | 컬렉션 허브용 레벨/진행도 표시 데이터 |
| `app.js` | 매치 종료 직후 XP 지급 및 결과 오버레이 UI 연결 |

전투 중 엔티티나 Ability가 직접 `profile.experience`를 수정하지 않습니다. `BattleApp.finishMatch()`가 `MatchReport`를 완성한 뒤 유저 참가 매치에 한해 XP를 지급합니다.

## 13. 처리 흐름

```text
BattleApp.startTournament()
  -> 선택 캐릭터 experienceLevel 계산
  -> rivalExperienceScaling 계산
  -> defendingChampion가 있으면 해당 fighter spec에 championAura 계산
  -> 상대 fighter spec에 성장 보정 적용 후 토너먼트 고정
BattleSimulation
  -> 대표 행동 이벤트를 MatchReport에 기록
BattleApp.finishMatch()
  -> 플레이어 참가 매치 결과 누적
  -> grantExperienceFromMatchReport(profile, matchReport) // matchReport.playerFighterId 기준
  -> 매치 결과 오버레이에 XP 패널 표시
  -> savePlayerProfile()
BattleApp.showTournamentChampion()
  -> TournamentReport 완성
  -> 디펜딩 챔피언 격파 여부에 따라 bountyXp 지급
  -> 우승자가 AI면 defendingChampion 등록/갱신
  -> 최종 토너먼트 결과를 UI에 표시
  -> savePlayerProfile()
```

XP 지급은 매치 종료 시 유저 캐릭터 기준으로 한 번만 실행합니다. 도감 기록, 업적 판정, 도전 단계 해금은 토너먼트 종료 흐름에서 별도로 처리하며 XP를 다시 지급하지 않습니다.

## 14. 필수 회귀 조건

- 패배한 토너먼트도 매치 XP를 지급해야 합니다(`comebackBonus`는 0).
- 같은 `tournamentId:characterId` 조합을 두 번 처리해도 XP가 중복 증가하지 않아야 합니다.
- AI끼리 진행한 매치는 XP에 포함되지 않아야 합니다.
- 캐릭터 A로 얻은 레벨 보상이 캐릭터 B에게 적용되지 않아야 합니다.
- 캐릭터 A의 레벨로 생성한 상대 성장 보정이 캐릭터 B 선택 토너먼트에 재사용되지 않아야 합니다.
- 레벨 곡선 변경 후에도 저장된 `totalXp`에서 레벨이 재계산되어야 합니다.
- Lv.10 이후에는 전투 보상이 추가로 증가하지 않아야 합니다.
- 상대 성장 보정은 `hp`, `damage`, `defense`에만 적용되어야 합니다.
- 상대 성장 보정은 상한을 넘어 증가하지 않아야 합니다.
- 토너먼트 도중 레벨업해도 이미 생성된 상대 spec은 변경되지 않아야 합니다.
- `matchXP` 계산 시 `dealRatio`는 최대 2.0으로 클램프되어야 합니다.
- `matchXP` 계산 시 `hpRemainRatio`는 패배 시 0이어야 합니다.
- `comebackBonus`는 승리 시에만 계산되어야 하며, 패배 시 0이어야 합니다.
- `MatchReport`는 `damageDealt`, `damageTaken`, `minHpRatio`를 정확히 기록해야 합니다.
- `minHpRatio`는 매치 중 가장 낮았던 `hp/maxHp` 시점을 기록해야 합니다(회복 효과 고려).
- AI 우승자가 디펜딩 챔피언으로 저장되어야 합니다.
- 같은 AI가 연속 우승할 때만 챔피언 `streak`가 증가해야 합니다.
- 플레이어가 디펜딩 챔피언을 직접 격파했을 때만 현상금 XP를 받아야 합니다.
- AI가 디펜딩 챔피언을 탈락시킨 경우 현상금 XP를 지급하지 않아야 합니다.
- 챔피언 버프는 영구 저장되지 않아야 하며, 토너먼트 생성 spec에만 반영되어야 합니다.
- 챔피언 버프와 승률 편향 감쇠를 적용해도 `speed`, `skill`, `radius`, `mass`는 변하지 않아야 합니다.
- 디펜딩 챔피언 볼에는 전투 중 챔피언 모자가 표시되어야 합니다.
- 챔피언 모자는 볼 회전과 무관하게 화면 기준으로 똑바로 보여야 합니다.
- 모바일에서 챔피언 strip이 canvas, 액션 버튼, 결과 팝업 주요 문구와 겹치지 않아야 합니다.
- PC에서 챔피언 chip 때문에 bracket row 높이가 달라지지 않아야 합니다.
- 색상만 제거해도 챔피언 여부를 모자 형태 또는 텍스트로 구분할 수 있어야 합니다.
- 컬렉션 허브의 레벨/진행도는 프로필의 `totalXp`와 일치해야 합니다.

## 15. 후속 결정

- 계정 경험치를 별도 레벨/칭호 시스템으로 확장할지 결정해야 합니다.
- 레벨 10 칭호와 시각 효과 문구는 캐릭터별 콘텐츠 작업에서 확정합니다.
- 상대 성장 보정 비율과 상한은 balanceSim으로 레벨별 승률을 측정한 뒤 확정합니다.
- 챔피언 버프를 아예 보정 없음으로 둘지, 최대 2.5% 1회성 보정까지 허용할지 balanceSim으로 결정합니다.
- 승률 편향 감쇠의 `recentWinRate` 집계 기간은 토너먼트 30~100회 시뮬레이션 기준으로 정합니다.
- 경험치 시스템의 `minHpRatio` 추적을 위해 `BattleSimulation.hooks`에 `onHpChanged` 이벤트를 활용할지, 별도 추적기가 필요할지 결정해야 합니다.
